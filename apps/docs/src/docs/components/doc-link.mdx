# DocLink

DocLink is a specialized component for linking to internal documentation pages
within the Nimbus documentation site. It provides a consistent way to handle
internal document linking with optional code formatting and utilizes the
`activeRouteAtom` for seamless navigation.

## Usage

The DocLink component handles proper routing to documentation pages using the
application's state management (Jotai's `activeRouteAtom`) and offers options
for styling the link content.

```tsx
import { DocLink } from "../components/navigation/doc-link";

// Basic usage
<DocLink docRoute="/components/button">Button Component</DocLink>

// With code formatting
<DocLink docRoute="/components/button" asCode={true}>Button</DocLink>

// Just text without a link
<DocLink>Text without link</DocLink>

// With custom click handler
<DocLink
  docRoute="/components/button"
  onClick={(e) => console.log('Button clicked')}
>
  Button Component
</DocLink>
```

## Best Practices

- Use DocLink for all internal documentation links to ensure consistent
  navigation behavior
- DocLink automatically updates the application's route state via
  `activeRouteAtom`
- Use the `asCode` prop when referencing component names, props, or code
  elements
- Ensure that routes begin with a forward slash for proper navigation
