# Documentation Site (Docs App)

Vite-powered documentation site rendering component docs from MDX files with interactive code examples and auto-generated prop tables.

## RFC 2119 Compliance

**This file and all docs app documentation MUST be interpreted according to [RFC 2119](https://www.ietf.org/rfc/rfc2119.txt).**

Key words: **MUST** / **MUST NOT** / **SHOULD** / **SHOULD NOT** / **MAY** (see root CLAUDE.md for definitions)

## Quick Commands

```bash
# Development
pnpm start:docs                 # Start dev server (http://localhost:5173)

# Building
pnpm build:docs                 # Build docs-build + app (from root)

# Regenerate data after MDX/type changes
pnpm --filter @commercetools/nimbus-docs-build build
```

## App Overview

**Purpose:** Interactive documentation site for Nimbus components

**Key Technologies:**
- Vite (build + dev server)
- React 18 + React Router
- MDX (markdown with JSX)
- `@commercetools/nimbus-docs-build` (data generation)

**Features:**
- Component docs from MDX files
- Live code examples (jsx-live)
- Auto-generated prop tables from TypeScript
- Search functionality
- Multi-view documentation (main, dev, api)

## Architecture

```
MDX files → Docs Build → Generated Data → Docs App → User
(Nimbus)    (Scanner)    (routes.json)    (React)
                         (props.json)
```

**YOU MUST regenerate data after:**
- Adding/modifying MDX files
- Changing component TypeScript interfaces
- Adding new components

## Key Concepts

### 1. Multi-View Documentation

Documentation structure is flexible and adapts to the type of item (components, hooks, utils, etc.).

Established pattern for React components:
- **Overview** (`.mdx`) - Main documentation page (required)
- **Guide** (`.guide.mdx`) - Guidance for designers/UX professionals (optional)
- **Dev** (`.dev.mdx`) - Developer-focused implementation details (optional)
- **Accessibility** (`.a11y.mdx`) - Accessibility documentation (optional)

**Location:** Colocated with components in `packages/nimbus/src/components/`

### 2. jsx-live Blocks

Interactive code examples executed in browser:

````markdown
```jsx-live
const App = () => <Button>Click me</Button>
```
````

**All Nimbus components are globally available** - no imports needed.

See [jsx-live Blocks Guide](../../docs/docs-app/jsx-live-blocks.md)

### 3. PropsTable Component

Auto-generated prop documentation from TypeScript:

```jsx
<PropsTable id="Button" />
```

**YOU MUST use exact component export name** - checks TypeScript definitions.

See [PropsTable Guide](../../docs/docs-app/props-table.md)

## MDX File Format

### Required Frontmatter

```markdown
---
id: button
title: Button
description: Button component for user actions
order: 1
menu: Components
tags: [interactive, form]
---

# Button

Component content here...
```

**YOU MUST include:**
- `id` - Unique identifier (kebab-case)
- `title` - Display name
- `description` - Short summary
- `order` - Sort order in menu
- `menu` - Menu category
- `tags` - Search tags (array)

See [MDX Format Guide](../../docs/docs-app/mdx-format.md)

## Development Workflow

### Adding New Documentation

1. Create MDX file in component directory (e.g., `button.mdx`)
2. Add required frontmatter
3. **YOU MUST regenerate data:** `pnpm --filter @commercetools/nimbus-docs-build build`
4. Start dev server: `pnpm start:docs`
5. Navigate to `/components/button` (based on `id`)

### Modifying Existing Documentation

1. Edit MDX file
2. **YOU MUST regenerate data:** `pnpm --filter @commercetools/nimbus-docs-build build`
3. Dev server auto-reloads

### Adding Interactive Examples

Use jsx-live blocks:

````markdown
```jsx-live
const Demo = () => {
  const [count, setCount] = useState(0);

  return (
    <Button onPress={() => setCount(count + 1)}>
      Clicked {count} times
    </Button>
  );
};
```
````

**Available globals:**
- All Nimbus components
- `useState`, `useEffect` (React hooks)
- `Icons` namespace

## Build System Integration

**Data generation:** `@commercetools/nimbus-docs-build` package

**Scans:**
- `packages/nimbus/src/**/*.mdx` - Documentation files
- `packages/nimbus/src/**/*.types.ts` - TypeScript interfaces

**Generates:**
- `apps/docs/src/data/routes.json` - Navigation routes
- `apps/docs/src/data/props.json` - Component prop tables

**YOU MUST rebuild after changes to:**
- MDX frontmatter
- Component TypeScript types
- File additions/removals

See [Build System Guide](../../docs/docs-app/build-system.md)

## Route Generation

Routes auto-generated from MDX `id` and `menu` frontmatter:

```yaml
---
id: button
menu: Components
---
```

**Generates route:** `/components/button`

**Menu categories:**
- `Components` → `/components/`
- `Foundations` → `/foundations/`
- `Guides` → `/guides/`

## Search Functionality

Search indexes MDX content via frontmatter `tags`:

```yaml
---
tags: [button, interactive, form, click]
---
```

**YOU SHOULD include relevant tags** for discoverability.

## Common Issues

### Routes Not Showing
**YOU MUST:**
1. Verify MDX has required frontmatter
2. Regenerate data: `pnpm --filter @commercetools/nimbus-docs-build build`
3. Restart dev server
4. Check console for errors

### PropsTable Empty/Wrong
**YOU MUST:**
1. Verify component export name matches `id` prop
2. Check TypeScript types exist in `*.types.ts`
3. Regenerate data: `pnpm --filter @commercetools/nimbus-docs-build build`
4. Inspect `src/data/props.json` for generated props

### jsx-live Example Not Working
**YOU SHOULD:**
1. Verify syntax is valid JSX
2. Check component is exported from Nimbus
3. Review browser console for errors
4. Ensure proper React hook usage

### Build Fails
**YOU MUST:**
1. Check MDX syntax validity
2. Validate frontmatter YAML format
3. Ensure all referenced components exist
4. Review build console errors
