## Overview

PublicPageLayout is a pattern that composes a centered, full-viewport-height
 layout for public-facing pages. It replaces Merchant Center Application Kit's
 `public-page-layout` component.

The layout vertically stacks: an optional brand logo, an optional welcome
heading, the main content area, and an optional legal footer — all
horizontally centered with a constrained max-width.

## Anatomy

- A `<main>` landmark with an accessible label (default: "Public page")
- Optional logo area, centered horizontally
- Optional welcome heading rendered as an `<h2>`
- Content area with a max-width controlled by `contentWidth`
- Optional legal/footer message area

## Usage

### When to use

- Login, registration, password reset, and other unauthenticated pages
- Any public-facing page that needs a centered, single-column layout with
  branding

### When not to use

- Authenticated app pages — use `DefaultPage` or other app layout patterns
- Pages that need a sidebar or multi-column chrome — compose Flex and Stack
  directly

### Do

- Pass a brand logo via the `logo` prop for consistent branding
- Use `contentWidth="wide"` when rendering two-column content
- Provide a descriptive `aria-label` when the default "Public page" is too
  generic (e.g., `aria-label="Login page"`)

### Don't

- Don't nest another `<main>` landmark inside children — PublicPageLayout
  already renders one
- Don't use this for authenticated layouts that need navigation or sidebars
