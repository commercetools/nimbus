# Localized field

A localized field is a single component that enables users to input and manage content for multiple language versions of a piece of data within one unified form area.

## Overview

The localized field component serves as a container that displays and manages content across various defined locales. It minimizes clutter by allowing users to toggle between language versions using a control while ensuring that the data for each specified locale is correctly captured, validated, and stored in the application.

## Variables

Get familiar with the features.



### Size

There are two supported sizes for localized fields, the default (md) and a smaller version for more compact needs (sm).


```jsx-live
const App = () => {
  const [smValues, setSmValues] = useState({ en: 'Hello world', de: 'Hallo Welt' });
  const [mdValues, setMdValues] = useState({ en: 'Hello world', de: 'Hallo Welt' });

  return (
    <Box>
      <LocalizedField
        size="md"
        defaultLocaleOrCurrency="en"
        valuesByLocaleOrCurrency={mdValues}
        onChange={(e) => {
          setMdValues({
            ...mdValues,
            [e.target.locale]: e.target.value
          });
        }}
      />
       <LocalizedField
        size="sm"
        defaultLocaleOrCurrency="en"
        valuesByLocaleOrCurrency={smValues}
        onChange={(e) => {
          setSmValues({
            ...smValues,
            [e.target.locale]: e.target.value
          });
        }}
      />
    </Box>
  );
};
```

### Expand and hide

Allow users to be able to expand and minimize localized fields to simplify a page, or expand to see everything when needed.


```jsx-live
const App = () => {
  const [values, setValues] = useState({ en: 'Hello world', de: 'Hallo Welt', es: 'Hola Mundo', nl: 'Hallo Wereld' });

  return (
    <LocalizedField
      size="md"
      defaultLocaleOrCurrency="en"
      defaultExpanded
      valuesByLocaleOrCurrency={values}
      onChange={(e) => {
        setValues({
          ...values,
          [e.target.locale]: e.target.value
        });
      }}
    />
  );
};
```


### Localized field combinations

Combine with rich text, text, multiline text, richttext, and moneyinput to give users more availability for localized input details.


```jsx-live
const App = () => {
  const [enValues, setEnValues] = useState({
    en: 'hello',
  });

  const [deValues, setDeValues] = useState({
    de: 'hallo',
  });

  return (
    <Box>
      <LocalizedField
        type="richText"
        defaultLocaleOrCurrency="en"
        valuesByLocaleOrCurrency={enValues}
        onChange={(e) => {
          setEnValues({
            ...enValues,
            [e.target.locale]: e.target.value
          });
        }}
      />
      <LocalizedField
        type="richText"
        defaultLocaleOrCurrency="de"
        valuesByLocaleOrCurrency={deValues}
        onChange={(e) => {
          setDeValues({
            ...enValues,
            [e.target.locale]: e.target.value
          });
        }}
      />
    </Box>
  );
};
```

## Accessibility

Accessibility ensures that digital content and functionality are usable by everyone, including people with disabilities, by addressing visual, auditory, cognitive, and physical limitations.

### Accessibility standards

 - **Focus management:** Ensure the Tab order moves logically between the active input field and the language switching controls. When the language is switched, focus should return to the newly visible input field.
 - **Keyboard operability:** The language switching controls (tabs, buttons) must be fully navigable and operable via the keyboard.
 - **ARIA roles:** Ensure the active input field correctly inherits all necessary ARIA attributes (e.g., aria-required, aria-invalid) from the FormField component.
 - **Screen reader communication:** Ensure screen readers announce the change of language context when the user activates a different locale input, often by announcing the new tab or language code.
