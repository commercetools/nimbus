## Accessibility

Accessibility ensures rendered Markdown is usable by everyone, including people
who rely on assistive technology. The default renderers enforce WCAG 2.1 AA for
the content they produce; a consumer override owns the accessibility of whatever
it renders.

### Headings (1.3.1, 2.4.6)

- Headings render as semantic `<h1>`–`<h6>`. Use `headingOffset` to nest
  rendered headings beneath the host page's outline.
- The component never adds a heading-level skip, and it renders author skips
  faithfully — a skip emits a development-mode warning rather than silently
  rewriting structure.

### Links (2.4.4, 1.4.1, 1.4.11)

- External links get `rel="noopener noreferrer"`, `target="_blank"`, an i18n
  "(opens in new tab)" label for assistive technology, and a visible icon
  indicator that is conveyed by shape (not color alone) and meets ≥3:1 non-text
  contrast.

### Images (1.1.1)

- Author `alt` text is preserved. A missing alt renders `alt=""` (decorative)
  and warns in development. Images carry `loading="lazy"` and
  `referrerpolicy="no-referrer"`.

### Tables (1.3.1)

- Tables use real `<table>` / `<thead>` / `<th scope="col">` semantics.

### Lists & code

- Real `<ul>` / `<ol>` and semantic `<pre><code>` are produced. GFM task-list
  checkboxes are read-only with an accessible name derived from the item text.

### Streaming (4.1.3)

- While `isStreaming` is set, the root carries `aria-busy="true"`. When the
  stream settles, `aria-busy` clears and a single coalesced completion message
  is announced via a polite live region — no per-token announcement spam, and no
  ARIA wiring required from the consumer.

### Color & motion

- All default styles use AA-compliant tokens, and no streaming reveal animation
  is introduced.
