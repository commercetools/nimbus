## Getting started

### Import

```tsx
import { ScrollableRegion, type ScrollableRegionProps } from '@commercetools/nimbus';
```

### Basic usage

The simplest implementation wraps content in a fixed-height container:

```jsx live-dev
const App = () => (
  <ScrollableRegion aria-label="Log output" h="150px">
    {Array.from({ length: 20 }, (_, i) => (
      <Text key={i}>Log entry {i + 1}: Application started successfully.</Text>
    ))}
  </ScrollableRegion>
)
```

## Usage examples

### Role region

Use `role="region"` for major page landmarks. This enforces an accessible name
via TypeScript — either `aria-label` or `aria-labelledby` must be provided:

```jsx live-dev
const App = () => (
  <ScrollableRegion role="region" aria-label="Terms and conditions" h="150px">
    {Array.from({ length: 20 }, (_, i) => (
      <Text key={i}>Clause {i + 1}: Lorem ipsum dolor sit amet.</Text>
    ))}
  </ScrollableRegion>
)
```

## Component requirements

- An accessible name (`aria-label` or `aria-labelledby`) is strongly
  recommended for all scrollable regions. It is enforced at the TypeScript
  level when `role="region"`.
- In development mode, a `console.warn` is logged when `role="region"` is used
  without an accessible name.

## Accessibility

The ScrollableRegion component handles accessibility as follows:

- ARIA `role` and accessible name are always applied to the container
- `tabIndex={0}` is added only when content overflows, making the container
  keyboard-focusable
- A focus ring is shown only during keyboard navigation (not mouse clicks) via
  Chakra's `focusVisibleRing`
- Detects both vertical and horizontal overflow

If your use case requires tracking and analytics for this component, it is good
practice to add a **persistent**, **unique** id to the component:

```tsx
const PERSISTENT_ID = "example-scrollable-region";

export const Example = () => (
  <ScrollableRegion id={PERSISTENT_ID} aria-label="Example">
    {/* content */}
  </ScrollableRegion>
);
```

#### Keyboard navigation

| Key | Action |
|---|---|
| `Tab` | Focuses the scrollable region (when overflowing) |
| `Arrow Up` / `Arrow Down` | Scrolls content vertically |
| `Arrow Left` / `Arrow Right` | Scrolls content horizontally |
| `Page Up` / `Page Down` | Scrolls by page |
| `Home` / `End` | Scrolls to beginning / end |
