## Overview

ScrollableRegion makes scrollable containers accessible to keyboard-only and
screen reader users. It always applies ARIA `role` and accessible name for
screen readers, and when content overflows it adds `tabIndex` and a
keyboard-only focus ring so users can scroll without a mouse.

### Resources

- [WCAG 2.1 SC 2.1.1 Keyboard](https://www.w3.org/WAI/WCAG21/Understanding/keyboard.html)
- [ARIA Scrollable Region Practice](https://www.w3.org/WAI/ARIA/apd/practices/scrollable-region/)

## Variables

Get familiar with the features.

### Default (overflowing)

When content overflows, the container becomes keyboard-focusable with a
`role="group"` landmark and visible focus ring on keyboard navigation.

```jsx live
const App = () => (
  <ScrollableRegion aria-label="Log output" h="150px">
    {Array.from({ length: 20 }, (_, i) => (
      <Text key={i}>Log entry {i + 1}: Application started successfully.</Text>
    ))}
  </ScrollableRegion>
)
```

### Non-overflowing

When content fits within the container, the ARIA `role` and accessible name
remain applied but `tabIndex` is not set — the container is not
keyboard-focusable when there is nothing to scroll.

```jsx live
const App = () => (
  <ScrollableRegion aria-label="Short content" h="150px">
    <Text>This content fits and the container is not focusable.</Text>
  </ScrollableRegion>
)
```

### Scroll axis control

By default, scrolling is enabled on both axes. Use the `scrollable` prop to
control which axis is scrollable or to disable scrolling entirely.

```jsx live
const App = () => (
  <ScrollableRegion aria-label="Vertical only" scrollable="y-auto" h="150px">
    {Array.from({ length: 20 }, (_, i) => (
      <Text key={i}>Entry {i + 1}: Data row content.</Text>
    ))}
  </ScrollableRegion>
)
```

## Labeling patterns

### Role region

Use `role="region"` for major page sections. This requires either `aria-label`
or `aria-labelledby` to satisfy WCAG landmark naming requirements.

```jsx live
const App = () => (
  <ScrollableRegion role="region" aria-label="Terms and conditions" h="150px">
    {Array.from({ length: 20 }, (_, i) => (
      <Text key={i}>Clause {i + 1}: Lorem ipsum dolor sit amet.</Text>
    ))}
  </ScrollableRegion>
)
```

### With aria-labelledby

When a visible heading labels the scrollable region, use `aria-labelledby`
instead of `aria-label` to avoid duplicating text.

```jsx live
const App = () => (
  <Box>
    <Heading id="logs-heading" fontSize="md">Application Logs</Heading>
    <ScrollableRegion aria-labelledby="logs-heading" h="150px">
      {Array.from({ length: 20 }, (_, i) => (
        <Text key={i}>Log entry {i + 1}: Request processed.</Text>
      ))}
    </ScrollableRegion>
  </Box>
)
```
