## Getting started

### Import

```tsx
import { Kbd } from '@commercetools/nimbus';
```

### Basic usage

The Kbd component displays keyboard keys and shortcuts with consistent styling:

```jsx live-dev
const App = () => (
  <Text>
    Press <Kbd>Enter</Kbd> to submit the form.
  </Text>
)
```

## Usage examples

### Single keys

Display individual keyboard keys:

```jsx live-dev
const App = () => (
  <Stack direction="column" gap="300">
    <Text>Press <Kbd>A</Kbd> to select all</Text>
    <Text>Press <Kbd>Esc</Kbd> to cancel</Text>
    <Text>Press <Kbd>Tab</Kbd> to move focus</Text>
    <Text>Press <Kbd>Space</Kbd> to activate</Text>
    <Text>Press <Kbd>Delete</Kbd> to remove</Text>
  </Stack>
)
```

### Key combinations

Display multiple keys pressed together using separate Kbd components:

```jsx live-dev
const App = () => (

    <Text>

    </Text>

    </Text>
    <Text>
      Press <Kbd>Ctrl</Kbd> + <Kbd>Z</Kbd> to undo
    </Text>
    <Text>
      Press <Kbd>Ctrl</Kbd> + <Kbd>Shift</Kbd> + <Kbd>Z</Kbd> to redo
    </Text>
  </Stack>
)
```

## Component requirements

## Accessibility

The Kbd component renders semantic `<kbd>` HTML elements which are recognized by screen readers as keyboard input. This helps assistive technology users understand that the content represents keys to press.

**Best practices:**
- Always use Kbd within descriptive text that explains the action
- Provide clear context about what the keyboard shortcut does
- Use consistent patterns for key combinations (e.g., modifier + key)
- Consider providing alternative methods for users who cannot use keyboards

**Example with clear context:**
```jsx live-dev
const App = () => (
  <Text>
    To save your changes, press <Kbd>Ctrl</Kbd> + <Kbd>S</Kbd> or use the Save button.
  </Text>
)
```

#### Keyboard navigation

The Kbd component does not have interactive elements and does not require keyboard navigation. It is a display-only component used to visually represent keyboard keys.
