## Getting started

### Import

```tsx
import { ProgressBar, type ProgressBarProps } from '@commercetools/nimbus';
```

### Basic usage

The simplest implementation requires a `value` and a `label`. The default range is 0 to 100.

```jsx-live-dev
const App = () => (
  <ProgressBar value={40} label="Loading..." />
)
```

## Usage examples

### Size options

The `2xs` and `md` (default) size variants are available to match your interface density:

```jsx-live-dev
const App = () => (
  <Stack direction="column" gap="400">
    <ProgressBar value={60} size="2xs" label="Extra small (2xs)" />
    <ProgressBar value={60} size="md" label="Medium (md)" />
  </Stack>
)
```

### Visual variants

Choose between `solid` (default) and `contrast` variants to match your design context:

```jsx-live-dev
const App = () => (
  <Stack direction="column" gap="400">
    <ProgressBar value={75} variant="solid" label="Solid variant" />
    <Box bg="primary.9" p="400" borderRadius="200">
      <ProgressBar value={75} variant="contrast" label="Contrast variant" />
    </Box>
  </Stack>
)
```

### Layout options

The `layout` prop controls the positioning of the label and value text:

```jsx-live-dev
const App = () => (
  <Stack direction="column" gap="600">
    <ProgressBar
      value={50}
      layout="stacked"
      label="Stacked (default)"
    />
    <ProgressBar
      value={50}
      layout="inline"
      label="Inline"
    />
    <ProgressBar
      value={50}
      layout="minimal"
      aria-label="Minimal (no visual label)"
    />
  </Stack>
)
```

> [!TIP]\
> When using `layout="minimal"`, the visual label is hidden. You **must** provide an `aria-label` for accessibility.

### Indeterminate state

Use the `isIndeterminate` prop when the progress duration is unknown. The bar will show an animated loading state:

```jsx-live-dev
const App = () => (
  <ProgressBar
    isIndeterminate
    label="Processing..."
  />
)
```

### Dynamic vs. Static progress

The `isDynamic` prop indicates whether the progress represents an active, ongoing process (like a download) or a static indicator (like a step in a wizard).

```jsx-live-dev
const App = () => (
  <Stack direction="column" gap="600">
    <Stack gap="200">
      <ProgressBar
        value={65}
        isDynamic={true}
        label="Dynamic progress"
      />
      <Text fontSize="sm" color="gray.600">
        Used for active processes (uploads, downloads).
      </Text>
    </Stack>

    <Stack gap="200">
      <ProgressBar
        value={60}
        isDynamic={false}
        label="Static progress"
      />
      <Text fontSize="sm" color="gray.600">
        Used for static indicators (wizard steps, capacity).
      </Text>
    </Stack>
  </Stack>
)
```

### Custom formatting

Use `formatOptions` to customize how the value is displayed. This accepts standard `Intl.NumberFormatOptions`.

```jsx-live-dev
const App = () => (
  <Stack direction="column" gap="400">
    <ProgressBar
      value={75}
      label="Percentage (default)"
      formatOptions={{ style: "percent" }}
    />
    <ProgressBar
      value={1500}
      maxValue={2000}
      label="Currency"
      formatOptions={{ style: "currency", currency: "USD" }}
    />
    <ProgressBar
      value={42}
      label="Storage"
      formatOptions={{ style: "unit", unit: "gigabyte" }}
    />
  </Stack>
)
```

### Color palettes

The `colorPalette` prop allows you to theme the progress bar using the system color tokens:

```jsx-live-dev
const App = () => (
  <Stack direction="column" gap="400">
    <ProgressBar value={80} colorPalette="primary" label="Primary" />
    <ProgressBar value={60} colorPalette="green" label="Green" />
    <ProgressBar value={40} colorPalette="red" label="Red" />
    <ProgressBar value={90} colorPalette="orange" label="Orange" />
  </Stack>
)
```

## Component requirements

### Accessibility

The ProgressBar handles most accessibility requirements internally using `role="progressbar"`. However, you must always ensure the component has an accessible label.

- **Visual Label**: Use the `label` prop. This automatically associates the label with the progress bar.
- **Hidden Label**: If using `layout="minimal"` or hiding the label, use `aria-label` to describe the progress bar.

```tsx
// ✅ Accessible with visible label
<ProgressBar value={50} label="Loading files" />

// ✅ Accessible with hidden label
<ProgressBar value={50} layout="minimal" aria-label="Loading files" />
```

If your use case requires tracking and analytics for this component, it is good practice to add a **persistent**, **unique** id to the component:

```tsx
const PERSISTENT_ID = "upload-progress-bar";

export const Example = () => (
  <ProgressBar id={PERSISTENT_ID} value={45} label="Upload" />
);
```

## Common patterns

### Simulated file upload

This example demonstrates how to update the progress bar value dynamically to simulate a file upload process.

```jsx-live-dev
const App = () => {
  const [progress, setProgress] = useState(0);
  const [isRunning, setIsRunning] = useState(false);

  useEffect(() => {
    let timer;
    if (isRunning && progress < 100) {
      timer = setInterval(() => {
        setProgress((prev) => Math.min(prev + Math.random() * 10, 100));
      }, 500);
    } else if (progress >= 100) {
      setIsRunning(false);
    }
    return () => clearInterval(timer);
  }, [isRunning, progress]);

  const handleStart = () => {
    setProgress(0);
    setIsRunning(true);
  };

  return (
    <Stack direction="column" gap="400">
      <ProgressBar
        value={progress}
        label={progress < 100 ? "Uploading..." : "Upload Complete"}
        colorPalette={progress < 100 ? "primary" : "green"}
      />
      <Button
        onClick={handleStart}
        disabled={isRunning}
        variant="secondary"
        width="fit-content"
      >
        {progress === 100 ? "Restart Upload" : "Start Upload"}
      </Button>
    </Stack>
  );
}
```
