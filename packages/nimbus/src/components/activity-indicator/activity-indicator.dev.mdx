## Getting started

### Import

```tsx
import {
  ActivityIndicator,
  type ActivityIndicatorProps,
} from "@commercetools/nimbus";
```

### Basic usage

The ActivityIndicator signals that an agent or system is actively working. By
default it is decorative and scales with the surrounding `font-size`, so it can
sit inline next to a visible status label.

```jsx live-dev
const App = () => (
  <Text>
    Thinking <ActivityIndicator />
  </Text>
);
```

## Usage examples

### Size options

The `size` prop defaults to `inherit`, which ties the indicator's box to the
surrounding `font-size` (`1em`) so it scales with the text. The fixed sizes
(`2xs`, `xs`, `sm`, `md`, `lg`) reserve a square icon-box footprint — the same
scale points `LoadingSpinner` uses — so the indicator drops into input start/end
icon slots interchangeably with a spinner.

```jsx live-dev
const App = () => (
  <Stack direction="row" gap="400" alignItems="center">
    <ActivityIndicator size="2xs" />
    <ActivityIndicator size="xs" />
    <ActivityIndicator size="sm" />
    <ActivityIndicator size="md" />
    <ActivityIndicator size="lg" />
  </Stack>
);
```

### Color palette

The `colorPalette` prop accepts any Nimbus color palette (default `primary`). The
dots are filled from the palette's `11` shade.

```jsx live-dev
const App = () => (
  <Stack direction="row" gap="400">
    <ActivityIndicator colorPalette="primary" />
    <ActivityIndicator colorPalette="positive" />
    <ActivityIndicator colorPalette="info" />
  </Stack>
);
```

### On a solid colored surface

When the dots sit on a solid colored fill (e.g. an agent bubble), use
`variant="contrast"`. This fills the dots from the palette's `contrast` step,
which resolves to black or white automatically per palette so the dots stay
legible on the surface.

```jsx live-dev
const App = () => (
  <Stack direction="row" gap="400">
    <Box colorPalette="primary" bg="colorPalette.9" padding="400" borderRadius="200">
      <ActivityIndicator colorPalette="primary" variant="contrast" />
    </Box>
    <Box colorPalette="ctyellow" bg="colorPalette.9" padding="400" borderRadius="200">
      <ActivityIndicator colorPalette="ctyellow" variant="contrast" />
    </Box>
  </Stack>
);
```

## Accessibility

The ActivityIndicator is **decorative by default**. When you render visible text
that already conveys the state (for example "Thinking…"), leave the indicator
unlabeled — it is marked `aria-hidden` so assistive technology hears the text,
not the animation.

```tsx
// Visible text conveys the state; the indicator is decorative.
<Text>
  Thinking <ActivityIndicator />
</Text>
```

When the indicator is used **standalone** (no adjacent visible text), provide an
`aria-label`. This upgrades it to a polite live region (`role="status"`,
`aria-live="polite"`) so screen readers announce the activity without
interrupting.

```tsx
// Standalone: announce the activity politely.
<ActivityIndicator aria-label="Agent is typing" />
```

Passing an empty string (`aria-label=""`) opts into the live region using the
component's localized default label.

This component is **not** a progress indicator and does not use
`role="progressbar"`. For indeterminate progress ("please wait"), use
[LoadingSpinner](/components/feedback/loadingspinner) instead.

## Reduced motion

When the user has `prefers-reduced-motion: reduce` enabled, the vertical bounce
is replaced by a gentle opacity pulse, so the "working" state remains
perceivable without large motion.
