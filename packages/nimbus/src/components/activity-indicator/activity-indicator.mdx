## Overview

The activity indicator communicates that an agent or system is actively working
— for example composing a reply, thinking, or processing a request. It is most
commonly placed inline next to a short status message ("Thinking…", "Processing
request") in chat and AI surfaces.

Unlike the [Loading spinner](/components/feedback/loadingspinner), which
represents indeterminate progress and signals "please wait", the activity
indicator is a lightweight, presentational hint. Reach for the loading spinner
when something is loading and for the activity indicator when an agent is
active.

## Variables

Get familiar with the features.

### Visuals

#### Inline with text

By default the indicator scales with the surrounding `font-size`, so it sits
naturally beside a status label.

```jsx live
const App = () => (
  <Text fontSize="md">
    Thinking <ActivityIndicator />
  </Text>
);
```

#### On a solid colored surface

When the indicator sits on a solid colored fill, use `variant="contrast"`. The
dots take the palette's `contrast` step (black or white, chosen automatically per
palette) so they stay legible on the surface.

```jsx live
const App = () => (
  <Stack
    colorPalette="primary"
    backgroundColor="colorPalette.9"
    padding="400"
  >
    <Text color="colorPalette.contrast">
      Thinking{" "}
      <ActivityIndicator colorPalette="primary" variant="contrast" />
    </Text>
  </Stack>
);
```

### Size

The default `inherit` size tracks the surrounding text. Fixed sizes (`2xs`–`lg`)
reserve a square icon-box footprint so the indicator can sit in input icon slots
alongside other icons.

```jsx live
const App = () => (
  <Stack direction="row" gap="400" alignItems="center">
    <ActivityIndicator size="lg" />
    <ActivityIndicator size="md" />
    <ActivityIndicator size="sm" />
    <ActivityIndicator size="xs" />
    <ActivityIndicator size="2xs" />
  </Stack>
);
```
