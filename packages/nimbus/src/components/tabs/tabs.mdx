# Tabs

Organizes content into distinct sections, enabling users to switch views within a single container.

## Overview

Tabs are a navigational component used to organize and switch between different sections of content within the same contextual view, allowing users to access multiple related pages or views without navigating away from the main screen.

### Direction for tabs

Tabs can be oriented horizontally or vertically to suit different layout requirements.

#### Horizontal tabs

Organize and display distinct, related sections of content immediately beneath a common navigation bar or header, serving primarily to group content contextually while conserving vertical screen space.

## Guidelines

Provides a structured, predictable pattern for displaying multiple panes of related content in the same area. It minimizes screen clutter by allowing content sections to be easily hidden and revealed, typically used for navigation within a product or configuration panel.

### Best practices

- **Consistency and context:** Ensure all content within the tab panels is closely related to the parent context (e.g., all tabs relate to "User Settings" or "Product Details").
- **Concise labels:** Keep tab labels short (one or two words is ideal) and descriptive, clearly indicating the content of the corresponding panel.
- **Maintain visibility:** Ensure the current active tab is clearly distinguishable from inactive tabs using strong visual cues (e.g., accent color, bold text, contrasting underline).
- **Maintain content position:** Do not use tabs to reorganize or filter the content of a single continuous list. The content associated with each tab should be unique and distinct.
- **Limit number of tabs:** Avoid creating a large number of tabs (ideally 3â€“7 visible tabs) to prevent horizontal scrolling or overcrowding, which harms usability. Use a dropdown menu or alternative navigation pattern for overflow.
- **Avoid sub-navigation:** Do not place secondary navigation elements (like another row of tabs) directly inside the tab panel unless absolutely necessary.

### Usage

Tabs are used to organize related content into distinct, switchable sections, allowing users to efficiently access different views within the same area without leaving the context of the main page.

> [!TIP]\
> When to use

- **Organizing related views:** To switch between different views of the same object (e.g., "Details," "Inventory," "History" tabs on a Product page).
- **Configuration settings:** For grouping diverse settings categories (e.g., "General," "Billing," "Security" sections within a settings page).
- **Conserving vertical space:** To maximize vertical screen space by presenting options horizontally.

> [!CAUTION]\
> When not to use

- **Primary site navigation:** Do not use tabs for navigating between major, top-level sections of an application (use main navigation menus or links instead).
- **Too many options:** If the number of tabs causes horizontal scrolling or requires hiding options in an overflow menu on desktop.
- **Linear workflows:** Avoid using tabs to guide users through a mandatory, step-by-step process.

### Tab labeling

> [!TIP]\
> **Do**
>
> - Choose concise, descriptive labels (1-2 words) that accurately preview the content within the panel to aid quick visual scanning.
> - Ensure all tab content is closely related to the parent screen's context, maintaining a single-page feel.
> - Implement an overflow strategy (like a "More" menu) if the tab count exceeds 7, rather than allowing horizontal scrolling, which is poor usability.
> - Verify that content inside the hidden tab panels remains available to screen readers, aligning with accessibility best practices.

### Accessibility Standards

- **Keyboard navigation:** The Tab key should move focus to the entire tab list. The Left/Right Arrow keys must be used to navigate between tabs once the tab list is focused. Enter or Space selects the focused tab.
- **ARIA roles and states:**
  - The container must use `role="tablist"`.
  - Individual tabs must use `role="tab"` and `aria-selected="true/false"`.
  - The content area must use `role="tabpanel"` and be linked to its respective tab via `aria-labelledby`.
- **Focus management:** Ensure that when a tab is activated, focus moves logically to the associated tab panel or the first interactive element within it.
- **Contrast:** The contrast ratio between the active tab text and its background must be high, and the focus indicator must be easily visible.
