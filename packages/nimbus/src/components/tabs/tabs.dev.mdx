## Getting started

### Import

```tsx
import { Tabs, type TabsProps } from '@commercetools/nimbus';
```

### Basic usage

The Tabs component supports two approaches depending on your needs.

For maximum control over tab structure and content, use the **Compound API** with manual composition:

```jsx live-dev
const App = () => {
  return (
    <Tabs.Root>
      <Tabs.List>
        <Tabs.Tab id="tab1">First Tab</Tabs.Tab>
        <Tabs.Tab id="tab2">Second Tab</Tabs.Tab>
        <Tabs.Tab id="tab3">Third Tab</Tabs.Tab>
      </Tabs.List>
      <Tabs.Panels>
        <Tabs.Panel id="tab1">
          <Text>Content for the first tab.</Text>
        </Tabs.Panel>
        <Tabs.Panel id="tab2">
          <Text>Content for the second tab.</Text>
        </Tabs.Panel>
        <Tabs.Panel id="tab3">
          <Text>Content for the third tab.</Text>
        </Tabs.Panel>
      </Tabs.Panels>
    </Tabs.Root>
  );
};
```

For data-driven tabs that render from an array, use the **Dynamic rendering** approach:

```jsx live-dev
const App = () => {
  const tabs = [
    {
      id: 'tab1',
      tabLabel: 'First Tab',
      panelContent: <Text>Content for the first tab.</Text>,
    },
    {
      id: 'tab2',
      tabLabel: 'Second Tab',
      panelContent: <Text>Content for the second tab.</Text>,
    },
    {
      id: 'tab3',
      tabLabel: 'Third Tab',
      panelContent: <Text>Content for the third tab.</Text>,
    },
  ];

  return <Tabs.Root tabs={tabs} tabListAriaLabel="Tab sections" />;
};
```

When using the `tabs` prop, provide a `tabListAriaLabel` for accessibility. This label is applied to the tab list element for screen readers.

### Placement

The `placement` prop controls where the tab list appears relative to the panels:
- **horizontal orientation**: `start` (left-aligned) or `end` (right-aligned)
- **vertical orientation**: `start` (tabs on left) or `end` (tabs on right)

```jsx live-dev

  const tabs = [
    { id: '1', tabLabel: 'First', panelContent: 'First panel content' },
    { id: '2', tabLabel: 'Second', panelContent: 'Second panel content' },
    { id: '3', tabLabel: 'Third', panelContent: 'Third panel content' },
  ];

  return (
    <Stack direction="column" gap="600">
      <Stack direction="column" gap="200">
        <Text fontWeight="500">Horizontal with placement="start"</Text>
        <Tabs.Root orientation="horizontal" placement="start" tabs={tabs} />
      </Stack>

      <Stack direction="column" gap="200">
        <Text fontWeight="500">Horizontal with placement="end"</Text>
```

### Dynamic tabs with tabs prop

For dynamic content, pass an array of tab configurations to the `tabs` prop. This is useful when tab data comes from an API or changes at runtime.

```jsx live-dev
const App = () => {
  const [tabs, setTabs] = useState([
    {
      id: '1',
      tabLabel: 'Introduction',
      panelContent: 'Welcome to our product documentation.',
    },
    {
      id: '2',
      tabLabel: 'Getting Started',
      panelContent: 'Learn how to get started with our product.',
    },
    {
      id: '3',
      tabLabel: 'API Reference',
      panelContent: 'Explore our comprehensive API documentation.',
    },
  ]);

  const addTab = () => {
    const newId = String(tabs.length + 1);
    setTabs([
      ...tabs,
      {
        id: newId,
        tabLabel: `Tab ${newId}`,
        panelContent: `Dynamic content for tab ${newId}`,
      },
    ]);
  };

  const removeLastTab = () => {
    if (tabs.length > 1) {
      setTabs(tabs.slice(0, -1));
    }
  };

  return (
    <Stack direction="column" gap="400">
      <Stack direction="row" gap="300">
        <Button onClick={addTab} size="md">
          Add Tab
        </Button>
        <Button onClick={removeLastTab} size="md" variant="outline">
          Remove Last Tab
        </Button>
      </Stack>

      <Tabs.Root tabs={tabs} />
    </Stack>
  );
};
```

## Component requirements

## Accessibility

The Tabs component follows WCAG 2.1 AA guidelines and implements the WAI-ARIA tabs pattern.

#### Role

- `Tabs.Root` contains the tablist and tabpanels
- `Tabs.List` has `role="tablist"`
- `Tabs.Tab` has `role="tab"` with `aria-selected` indicating active state
- `Tabs.Panel` has `role="tabpanel"` and is associated with its tab via `aria-labelledby`

#### Labeling

- Each tab must have an `id` prop that matches its corresponding panel
- Tab content should be descriptive and concise
- Complex tab labels can include icons and text
- When using the `tabs` prop API, provide `tabListAriaLabel` to label the tab list for screen readers
- When using the compound API, apply `aria-label` directly to `Tabs.List`

#### Keyboard navigation

| Key | Action |
|-----|--------|
| `Tab` | Move focus into and out of the tab list |
| `ArrowRight` / `ArrowDown` | Move focus to the next tab (horizontal/vertical) |
| `ArrowLeft` / `ArrowUp` | Move focus to the previous tab (horizontal/vertical) |
| `Home` | Move focus to the first tab |
| `End` | Move focus to the last tab |
| `Enter` / `Space` | Activate the focused tab (when `keyboardActivation="manual"`) |

#### Persistent ID

If your use case requires tracking and analytics for this component, it is good practice to add a **persistent**, **unique** id to the component:

```tsx
const PERSISTENT_ID = "user-settings-tabs";

export const SettingsTabs = () => (
  <Tabs.Root id={PERSISTENT_ID}>
    <Tabs.List>
      <Tabs.Tab id="profile">Profile</Tabs.Tab>
      <Tabs.Tab id="security">Security</Tabs.Tab>
    </Tabs.List>
    <Tabs.Panels>
      <Tabs.Panel id="profile">Profile content</Tabs.Panel>
      <Tabs.Panel id="security">Security content</Tabs.Panel>
    </Tabs.Panels>
  </Tabs.Root>
);
```

### Router-integrated tabs

Use router-integrated tabs when tab state should be reflected in the URL â€” enabling shareable links, browser back/forward navigation, and deep linking to specific tabs.

**Use this pattern when:**
- Tab content represents distinct views users may want to bookmark or share
- The active tab should survive a page refresh
- Your app uses react-router for navigation (replaces MC's standalone `tab-header` component)

**Stick with standard tabs when:**
- Tabs are purely UI-local (no shareable state needed)
- Tab switches do not represent meaningful navigation steps

Sync tab state with the URL using react-router's `useSearchParams` hook: read the active tab from `?tab=` on mount and write back to it whenever the selection changes.

```tsx
import { useSearchParams } from 'react-router-dom';
import { Tabs } from '@commercetools/nimbus';

const DEFAULT_TAB = 'overview';

export const ProductDetailTabs = () => {
  const [searchParams, setSearchParams] = useSearchParams();
  const activeTab = searchParams.get('tab') ?? DEFAULT_TAB;

  const handleTabChange = (key: string | number) => {
    setSearchParams(
      (prev) => {
        const next = new URLSearchParams(prev);
        next.set('tab', String(key));
        return next;
      },
      { replace: true } // avoid polluting browser history on every tab click
    );
  };

  return (
    <Tabs.Root selectedKey={activeTab} onSelectionChange={handleTabChange}>
      <Tabs.List aria-label="Product details">
        <Tabs.Tab id="overview">Overview</Tabs.Tab>
        <Tabs.Tab id="variants">Variants</Tabs.Tab>
        <Tabs.Tab id="inventory">Inventory</Tabs.Tab>
      </Tabs.List>
      <Tabs.Panels>
        <Tabs.Panel id="overview">
          <p>Product overview content.</p>
        </Tabs.Panel>
        <Tabs.Panel id="variants">
          <p>Variant management content.</p>
        </Tabs.Panel>
        <Tabs.Panel id="inventory">
          <p>Inventory tracking content.</p>
        </Tabs.Panel>
      </Tabs.Panels>
    </Tabs.Root>
  );
};
```

For path-based routing (e.g. `/products/:id/overview`), use `useParams` or `useMatch` to determine the active segment, then call `navigate` instead of `setSearchParams`:

```tsx
import { useNavigate, useParams } from 'react-router-dom';
import { Tabs } from '@commercetools/nimbus';

export const ProductDetailTabs = () => {
  const { tab = 'overview' } = useParams<{ tab: string }>();
  const navigate = useNavigate();

  return (
    <Tabs.Root
      selectedKey={tab}
      onSelectionChange={(key) => navigate(`../${String(key)}`, { relative: 'path' })}
```
