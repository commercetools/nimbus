## Accessibility

Accessibility ensures that digital content and functionality are usable by
everyone, including people with disabilities, by addressing visual, auditory,
cognitive, and physical limitations.

```jsx-live
const App = () => (
  <ToggleButtonGroup.Root aria-label="Labels and icons example" selectedKeys={["li-shipped"]} colorPalette="primary">
    <Tooltip.Root delay={0} closeDelay={0}>
      <ToggleButtonGroup.Button id="li-processing">
        <Icons.RotateRight /> Processing
      </ToggleButtonGroup.Button>
      <Tooltip.Content>Processing Status</Tooltip.Content>
    </Tooltip.Root>
    <Tooltip.Root delay={0} closeDelay={0}>
      <ToggleButtonGroup.Button id="li-shipped">
        <Icons.LocalShipping /> Shipped
      </ToggleButtonGroup.Button>
      <Tooltip.Content>Shipped Status</Tooltip.Content>
    </Tooltip.Root>
    <Tooltip.Root delay={0} closeDelay={0}>
      <ToggleButtonGroup.Button id="li-cancelled">
        <Icons.Dangerous /> Cancelled
      </ToggleButtonGroup.Button>
      <Tooltip.Content>Cancelled Status</Tooltip.Content>
    </Tooltip.Root>
    <Tooltip.Root delay={0} closeDelay={0}>
      <ToggleButtonGroup.Button id="li-refunded">
        <Icons.CurrencyExchange /> Refunded
      </ToggleButtonGroup.Button>
      <Tooltip.Content>Refunded Status</Tooltip.Content>
    </Tooltip.Root>
  </ToggleButtonGroup.Root>
);
```

### Accessibility standards

#### 1. Perceivable:

- **1.1.1 Non-text Content (Level A):**
  - Provide text alternatives for any non-text content (e.g., icons within
    buttons) via `aria-label` on the button or surrounding `Tooltip`.
  - Ensure icons have sufficient visual clarity.
- **1.4.1 Use of Color (Level A):**
  - Avoid relying solely on color to indicate the selected state. Default styles
    typically use background color and potentially borders or font weight
    changes.
- **1.4.3 Contrast (Minimum) (Level AA):**
  - Ensure sufficient color contrast between button text/icons and their
    background, in both selected and unselected states.
  - Ensure sufficient contrast between adjacent buttons.

#### 2. Operable:

- **2.1.1 Keyboard (Level A):**
  - Make the entire group focusable using the Tab key.
  - Allow navigation between buttons within the group using arrow keys
    (Left/Right or Up/Down depending on orientation).
  - Allow selection using the Enter or Space key.
- **2.4.4 Link Purpose (In Context) (Level A):** (Applies conceptually to button
  purpose)
  - Button labels (text or `aria-label` for icons) should clearly describe the
    action or choice.
- **2.4.7 Focus Visible (Level AA):**
  - Provide a clear and visible focus indicator for the currently focused button
    within the group.

#### 3. Understandable:

- **3.2.1 On Focus (Level A):**
  - Ensure that moving focus between buttons does not trigger unexpected
    actions.
- **3.3.2 Labels or Instructions (Level A):**
  - Provide a clear `aria-label` for the `ToggleButtonGroup.Root` element to
    describe the purpose of the group to assistive technologies.

#### 4. Robust:

- **4.1.2 Name, Role, Value (Level A):**
  - Use appropriate ARIA roles (e.g., `radiogroup` for single selection, `group`
    for multiple selection, `radio` or `checkbox` for items depending on mode)
    to ensure compatibility with assistive technologies. Nimbus components built
    on React Aria generally handle this automatically.
