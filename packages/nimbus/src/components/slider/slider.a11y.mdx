## Accessibility

Accessibility ensures that digital content and functionality are usable by
everyone, including people with disabilities, by addressing visual, auditory,
cognitive, and physical limitations.

`Slider` and `RangeSlider` render no visible label of their own, so an
accessible name must always come from somewhere — either `FormField.Label`
when composed inside a form field, or an `aria-label` / `aria-labelledby` when
used standalone.

```jsx live
const App = () => (
  <Slider aria-label="Volume" defaultValue={30} minValue={0} maxValue={100} />
);
```

### Accessibility standards

- Always give the slider an accessible name: `aria-label` (or
  `aria-labelledby`) standalone, or `FormField.Label` when composed inside
  `FormField.Root`. A slider with no accessible name is unusable with a screen
  reader.
- For `RangeSlider`, give each handle its own name via `thumbLabels` (e.g.
  `["Minimum price", "Maximum price"]`); when omitted, the handles fall back
  to localized "Minimum" / "Maximum" labels so they're never anonymous.
- The current value is exposed to assistive technology through the handle's
  native value, not just the visual tooltip — screen readers announce it
  without needing the tooltip to be open.
- Support full keyboard interaction: `Tab` to focus a handle, `Arrow` keys to
  adjust by `step`, and `Home` / `End` to jump to the minimum / maximum.
- Ensure sufficient color contrast for the track, fill, and handle in every
  state (default, hover, focus, dragging, disabled, invalid).
- Communicate the invalid state with both a visual cue and a
  `FormField.Error` message — never color alone.
- Clearly indicate the focused handle with a visible focus ring.
