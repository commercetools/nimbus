## Overview

A slider lets people choose a value — or, with `RangeSlider`, a min–max range —
by dragging a handle along a track, instead of typing a number. It's a good fit
whenever the exact number matters less than its position relative to the whole
scale, such as volume, zoom, opacity, or a price range filter.

`Slider` selects a single value. `RangeSlider` selects two values at once (a
lower and an upper bound) using two handles that can't cross each other. Both
share the same track, handle, and tick styling, so a page that mixes single and
range sliders still looks consistent.

Neither component draws its own label or a static "current value" readout. The
current value appears in a small tooltip above the handle while it's hovered,
focused, or being dragged — the track stays visually quiet the rest of the time.
The label comes from the surrounding `FormField`, or from an `aria-label` when
the slider is used on its own.

### Key features

- **Tooltip value display:** the current value shows above the handle on hover,
  focus, or drag — there's no permanent number next to the track.
- **Range selection:** `RangeSlider` uses two handles that can't be dragged past
  each other, so the lower value never exceeds the upper one.
- **Tick marks:** an optional row of marks along the track for scales where the
  discrete steps matter (e.g. star ratings, quality levels).
- **Horizontal and vertical:** either orientation is available for layouts where
  a vertical control reads more naturally (e.g. a mixing-desk-style fader).

## Variables

### Size

`md` is the default size. `sm` is available for denser layouts.

```jsx live
const App = () => (
  <Stack direction="horizontal" gap="600" alignItems="center">
    <Slider aria-label="Small" size="sm" defaultValue={30} />
    <Slider aria-label="Medium (default)" size="md" defaultValue={30} />
  </Stack>
);
```

### Variants

`plain` is the default. One additional variant is available for contexts that
call for a different visual weight:

- **Plain** (default) — a standard filled track with a colored progress bar. The
  safest choice for most forms and settings panels.
- **Enclosed** — a thick, contained bar with the handle inset inside it (an
  iOS-style control). Good for touch-first contexts or standalone controls that
  benefit from a larger, more tactile target.

```jsx live
const App = () => (
  <Stack direction="horizontal" gap="600" alignItems="center">
    <Slider aria-label="Plain (default)" variant="plain" defaultValue={60} />
    <Slider aria-label="Enclosed" variant="enclosed" defaultValue={75} />
  </Stack>
);
```

### Orientation

`horizontal` is the default. `vertical` is useful for fader-style controls or
when horizontal space is limited.

```jsx live
const App = () => (
  <Stack direction="horizontal" gap="600" alignItems="flex-start">
    <Slider aria-label="Horizontal" defaultValue={40} />
    <Slider aria-label="Vertical" defaultValue={40} orientation="vertical" />
  </Stack>
);
```

### Ticks

Turn on `showTicks` to mark discrete steps along the track. `tickStep` (or
`step`, if `tickStep` is omitted) controls how far apart the ticks are.

```jsx live
const App = () => (
  <Slider
    aria-label="Rating"
    defaultValue={50}
    minValue={0}
    maxValue={100}
    step={25}
    showTicks
  />
);
```

### Range

`RangeSlider` selects a `[min, max]` pair with two independent handles. Pass
`thumbLabels` to give each handle its own accessible name (for example
`["Minimum price", "Maximum price"]`); when omitted, the handles fall back to
localized "Minimum" / "Maximum" labels.

```jsx live
const App = () => (
  <RangeSlider
    aria-label="Price range"
    thumbLabels={["Minimum", "Maximum"]}
    defaultValue={[20, 60]}
    minValue={0}
    maxValue={100}
  />
);
```
