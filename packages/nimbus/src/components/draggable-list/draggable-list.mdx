# Draggable list

The draggable list allows users to easily reorder or manage a sequence of items via a direct drag-and-drop interaction.

## Overview
The draggable list component presents structured, ordered content where the sequence itself is meaningful. It uses visual and functional cues to enable users to physically manipulate the list order by dragging an item to a new insertion point, providing immediate visual feedback throughout the process.

### Interactions
**Drag, drop and remove**

Items within draggable lists can be lifted and reordered, with an option to have items removable or disabled.

```jsx-live
const App = () => {
  const items = [
    { key: "1", label: "Food" },
    { key: "2", label: "Cosmetics" },
    { key: "3", label: "Furniture" },
    { key: "4", label: "Electronics" },
    { key: "5", label: "Clothing" },
  ];

  return (
    <DraggableList.Root
      aria-label="Task list"
      items={items}
      removableItems
    />
  );
}
```

**Cross list drag and drop**

With two sections allowing users to pass items between and allowing for reordering. While dragging the new location will highlight conveying to the user that they can drop this item in that section. 


```jsx-live
const App = () => {
  const [leftItems, setLeftItems] = useState([
    { key: "1", label: "Food" },
    { key: "2", label: "Cosmetics" },
    { key: "3", label: "Furniture" },
    { key: "4", label: "Electronics" },
    { key: "5", label: "Clothing" },
  ]);

  const [rightItems, setRightItems] = useState([
    { key: "6", label: "Sport" },
    { key: "7", label: "Home" },
  ]);

  return (
    <Flex gap="400">
      <DraggableList.Root
        aria-label="Left column"
        items={leftItems}
        removableItems
        onUpdateItems={setLeftItems}
      />
      <DraggableList.Root
        aria-label="Right column"
        items={rightItems}
        removableItems
        onUpdateItems={setRightItems}
      />
    </Flex>
  );
}
```

### Size

**Size variants**

Draggable list supports two sizes. Use the smaller size to show a more compact experience.


```jsx-live
const App = () => {
  const items = [
    { key: "1", label: "Small food" },
  ];

  const mediumItems = [
    { key: "1", label: "Medium food" },
  ];

  return (
    <Stack direction="horizontal" gap="800">
      <DraggableList.Root
        aria-label="Small food list"
        items={items}
        size="sm"
      />
      <DraggableList.Root
        aria-label="Medium food list"
        items={mediumItems}
        size="md"
      />
    </Stack>
  );
}
```

## Accessibility

### Accessibility standards

 - **Keyboard reordering is essential:** Since drag-and-drop is a complex pointer gesture, a keyboard-accessible method for reordering is mandatory (WCAG 2.5.1 Pointer Gestures). This usually involves providing "Move Up" and "Move Down" buttons or a position input field next to the list item.
 - **ARIA roles:** Use appropriate ARIA roles to convey the list and item structure to screen readers. For interactive lists, `role="listbox"` for the container and `role="option"` for items might be suitable, or use standard semantic lists (`<ul>`, `<li>`) enhanced with ARIA attributes.
 - **Focus management:** Ensure the drag handle or the dedicated keyboard buttons are included in the Tab order and have clear focus indicators.
 - **Status updates:** Announce the result of the reordering to screen reader users using an `aria-live` region (e.g., "Item moved to position 4").
