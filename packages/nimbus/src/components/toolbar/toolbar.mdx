# Toolbar

Toolbars organize related actions and controls in a compact bar for easy access.
They provide consistent placement for common or contextual tasks within an
application.

## Overview

A toolbar is a handy container that organizes a group of related buttons and
tools, using both horizontal and vertical layouts to fit the screen;
essentially, it gives users quick access to the most common actions they need
right now.

## Variables

Get familiar with the features.

### Visual options

#### Horizontal

Most commonly used, horizontal toolbars should sit above the content meant to be
altered.

```jsx-live
const App = () => (
  <Toolbar orientation="horizontal">
    <Group>
      <IconButton size="xs" variant="ghost">
        <Icons.FormatBold />
      </IconButton>
      <IconButton size="xs" variant="ghost">
        <Icons.FormatItalic />
      </IconButton>
      <IconButton size="xs" variant="ghost">
        <Icons.FormatUnderlined />
      </IconButton>
      <IconButton size="xs" variant="ghost">
        <Icons.MoreHoriz />
      </IconButton>
    </Group>
    <Separator orientation="vertical" />
    <Group>
      <IconButton size="xs" variant="ghost">
        <Icons.FormatListBulleted />
      </IconButton>
      <IconButton size="xs" variant="ghost">
        <Icons.FormatListNumbered />
      </IconButton>
    </Group>
  </Toolbar>
)
```

#### Vertical

Vertical toolbars are less common, but in the right interface it can be more
helpful than a horizontal placement.

```jsx-live
const App = () => (
  <Toolbar orientation="vertical">
    <IconButton size="xs" variant="ghost">
      <Icons.FormatBold />
    </IconButton>

    <IconButton size="xs" variant="ghost">
      <Icons.FormatUnderlined />
    </IconButton>
    <IconButton size="xs" variant="ghost">
      <Icons.MoreHoriz />
    </IconButton>

```

#### Plain and outline

Two versions are available, a plain version that has a slight shadow to lift,
and an outlined version that has more contrast from elements around it. The
plain version without an outline is most common.

```jsx-live

  <Stack direction="row" gap="600" alignItems="center">
    <Toolbar variant="plain" size="xs">
      <IconButton size="xs" variant="ghost">
```

### Groups

Grouping similar tools is a common best practice within all toolbars. Please
combine similar tools in groups to make it easier for users to make choices
quickly. If there are a lot of tools within a group, fold them under a menu
button and leave 2-3 out that are the most popular choices.

```jsx-live
const App = () => (
  <Toolbar size="xs">
    <Group>
      <IconButton size="xs" variant="ghost">
        <Icons.FormatBold />
      </IconButton>
      <IconButton size="xs" variant="ghost">
        <Icons.FormatItalic />
      </IconButton>
      <IconButton size="xs" variant="ghost">
        <Icons.FormatUnderlined />
      </IconButton>
      <IconButton size="xs" variant="ghost">
        <Icons.MoreHoriz />
```

## Accessibility

Accessibility ensures that digital content and functionality are usable by
everyone, including people with disabilities, by addressing visual, auditory,
cognitive, and physical limitations.

### Accessibility standards

- **Keyboard Navigation:** Ensure all interactive elements within the toolbar are navigable using the tab key in a logical order, following the order of items.
- **ARIA Roles:** Apply the ARIA role toolbar to the container element.
- **Labeling:** Provide descriptive, concise aria-labels for the toolbar container and for all icon buttons to ensure screen reader users understand the component's purpose and the action it performs.
- **Provide clear status:**
  - Use the disabled state for actions that are temporarily unavailable. Do not hide actions that are simply disabled, as this can cause shifting layouts and confuse users.
  - Clearly communicate the reason an action is disabled via tooltips or other contextual help.
