# Tag Group

A tag group is a compact, interactive element. It categorizes, labels, or adds attributes to objects for quick recognition and filtering.

## Overview

The tag group organizes a collection of individual tag components, allowing users to efficiently scan and understand multiple pieces of metadata or filter criteria associated with a product, order, or entity in a compact, manageable space.

### Visual options


#### Size

The large size of the tag within tag groups is the default, the small size is meant for more compact uses. The more tags present, consider smaller sizes to make the information more compact and easily read for the user.

  return (
    <Stack direction="row" gap="400" alignItems="flex-start">

### Toggle buttons vs clearable tag groups

Allow users to toggle the active state of a tag, or be able to remove tags by adding a "close" icon to each tag. Avoid mixing these interactions within the same group. 

```jsx-live
const App = () => {
  const [selectedToggleTags, setSelectedToggleTags] = useState(new Set(["vegetables"]));

  const removableTags = [
    { id: "remove1", name: "Remove me first" },
    { id: "remove2", name: "Remove me second" },
    { id: "remove3", name: "Remove me last" },
  ];

  const toggleTags = [
    { id: "fruits", name: "Fruits" },
    { id: "vegetables", name: "Vegetables" },
    { id: "meat", name: "Meat" },
  ];

  return (
    <Stack direction="column" gap="600" alignItems="flex-start">
      <Box>
        <Text as="p" textStyle="sm" marginBottom="200" fontWeight="bold">Removable tag group</Text>
        <TagGroup.Root aria-label="Removable tags" onRemove={() => {}}>
          <TagGroup.TagList items={removableTags}>
            {(item) => <TagGroup.Tag>{item.name}</TagGroup.Tag>}
          </TagGroup.TagList>
        </TagGroup.Root>
      </Box>

      <Box>
        <Text as="p" textStyle="sm" marginBottom="200" fontWeight="bold">Toggleable tag group</Text>
        <TagGroup.Root
          aria-label="Toggle tags"
          selectionMode="multiple"
          selectedKeys={selectedToggleTags}
          onSelectionChange={setSelectedToggleTags}
        >
          <TagGroup.TagList items={toggleTags}>
            {(item) => <TagGroup.Tag>{item.name}</TagGroup.Tag>}
          </TagGroup.TagList>
        </TagGroup.Root>
      </Box>
    </Stack>
  );
}
```


## Guidelines
These guidelines provide comprehensive direction for designing and developing tags within Nimbus.

### Best practices
 - **Limit the number of tags:** Avoid overwhelming users with too many tags.
 - **Maintain consistent styling:** Ensure tags look and behave uniformly across the product.
 - **Provide clear affordance for removal:** If tags are removable, make the close icon visible and interactive.
 - **Do not mix toggle-able and removable tags within the same area:** These are different types of selective interactions, avoid mixing to avoid confusion and clear use in your interface.
 - **Single context:** Ensure all tags within a single group are relevant to one entity or topic (e.g., all tags describe a single product's features, or all tags represent filters applied to a list).
 - **Conciseness:** Keep tag labels short and easily readable (ideally one or two words). If the text is too long, it should be truncated, and a tooltip should reveal the full context.
 - **Logical ordering:** Arrange tags in a predictable, logical sequence, such as alphabetically, by date added, or by importance.
 - **Color consistency:** Use color consistently to convey meaning. Define a clear palette for status tags (e.g., Green for "In Stock") and neutral colors for simple categorization.
 - **Handling overflow:** Implement a clear strategy for managing too many tags (e.g., collapsing the list and showing a "+X more" count or allowing horizontal scrolling).

### Usage


Tags are used to display keywords. They also help filter data, allowing users to narrow down lists of items, and show selected options, representing choices in multi-select inputs.



> [!TIP]\
> When to use

 - **Displaying metadata/attributes:** To showcase several non-hierarchical attributes of an item (e.g., product size, material, color, collection name).
 - **Filtering statuses (interactive tags):** To show the user which filters are currently applied, often allowing them to click an 'x' to remove the filter immediately.
 - **Categorization (non-interactive tags):** To quickly label and organize content pieces (e.g., "New," "Beta," "Requires Review").
 - **Highlighting key status:** To draw attention to critical, secondary status information (e.g., "Out of Stock," "Draft").


> [!CAUTION]\
> When not to use

 - **Primary navigation:** Do not use tags to replace main menu links or primary navigation structures.
 - **Long sentences/descriptions:** If you need to convey complex sentences or lengthy descriptions, use standard body text or a detail panel instead.
 - **Too many options:** If the list of possible tags is extremely long and variable (over 10-15 per group), consider using a multi-select dropdown with categorization instead.
 - **Destructive actions:** Tags should not be used as the sole trigger for complex, destructive, or irreversible actions.
 - **Mixing toggle-able and removable tags:** is not a good use of tooling, it creates a very confusing experience for users. 


### Tags and tag group in use

> [!TIP]\
> **Do**
>
> - Do keep tag phrases short and descriptive. 
> - Do keep removable tags easy to remove, and toggle-able tags easy to click. Avoid moving them or shuffling them while a user is interacting with them. Moving targets are harder to click. 
> - Do keep styling and sizing consistent within the same context. 
> - Do limit the amount of tags per context to not overwhelm the UI.

## Accessibility
Accessibility ensures that digital content and functionality are usable by everyone, including people with disabilities, by addressing visual, auditory, cognitive, and physical limitations.


```jsx-live
const App = () => {
  const vegetableOptions = [
  { id: "tomato", name: "Tomato" },
  { id: "zucchini", name: "Zucchini" },
  { id: "peppers", name: "Peppers" },
  ];

  return (
    <TagGroup.Root onRemove={() => {}}>
      <TagGroup.TagList items={vegetableOptions}>
        {(item) => <TagGroup.Tag>{item.name}</TagGroup.Tag>}
      </TagGroup.TagList>
    </TagGroup.Root>
  );
}
```

### Accessibility standards
- **Keyboard focus:** Ensure the user can navigate to and between individual tags (if interactive/removable) using the Tab key.
- **ARIA labels for removal:** For removable tags, the close button/icon must have a clear `aria-label` (e.g., "Remove filter: [Tag Label]") to inform screen readers exactly what is being dismissed.
- **Contrast:** Verify that the text contrast ratio against the tag's background color meets WCAG AA standards (4.5:1).
- **Logical reading order:** The order in which the tags are read by a screen reader must logically align with the visual order.
