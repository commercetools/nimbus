## Getting started

The `Tree` is a compound component for displaying hierarchical data such as file
trees and nested navigation structures. It wraps React Aria's `Tree` for
keyboard navigation, expand/collapse, selection, type-ahead and opt-in
drag-and-drop.

### Import

Import the Tree compound component and its TypeScript types:

```tsx
import { Tree, type TreeRootProps } from "@commercetools/nimbus";
```

### Anatomy

```tsx
<Tree.Root>
  <Tree.Item>
    <Tree.ItemContent>
      <Tree.Indicator />
      {/* label */}
    </Tree.ItemContent>
    {/* nested <Tree.Item>s */}
  </Tree.Item>
</Tree.Root>
```

- **`Tree.Root`** — the tree container. Owns size variant, selection mode,
  expanded keys and (optional) drag-and-drop. Requires an accessible name via
  `aria-label` or `aria-labelledby`.
- **`Tree.Item`** — a single node. Contains a `Tree.ItemContent` and,
  optionally, nested `Tree.Item`s.
- **`Tree.ItemContent`** — the node's content row. Renders the indicator, an
  optional selection checkbox (in multiple-selection mode) and the label, and
  applies level-based indentation.
- **`Tree.Indicator`** — the expand/collapse chevron. Visible only for items
  with children.

### Basic usage

Compose nested `Tree.Item` elements directly. Use `defaultExpandedKeys` to
control which branches start open.

```jsx live-dev
const App = () => {
  return (
    <Tree.Root
      aria-label="Files"
      defaultExpandedKeys={["documents", "project"]}
    >
      <Tree.Item id="documents" textValue="Documents">
        <Tree.ItemContent>
          <Tree.Indicator />
          Documents
        </Tree.ItemContent>
        <Tree.Item id="project" textValue="Project">
          <Tree.ItemContent>
            <Tree.Indicator />
            Project
          </Tree.ItemContent>
          <Tree.Item id="report" textValue="Weekly Report">
            <Tree.ItemContent>
              <Tree.Indicator />
              Weekly Report
            </Tree.ItemContent>
          </Tree.Item>
        </Tree.Item>
      </Tree.Item>
      <Tree.Item id="photos" textValue="Photos">
        <Tree.ItemContent>
          <Tree.Indicator />
          Photos
        </Tree.ItemContent>
      </Tree.Item>
    </Tree.Root>
  );
};
```

## Usage examples

### Selection

Set `selectionMode` to `"single"` or `"multiple"`. In multiple-selection mode,
`Tree.ItemContent` renders a selection checkbox automatically.

```jsx live-dev
const App = () => {
  return (
    <Tree.Root
      aria-label="Files"
      selectionMode="multiple"
      defaultExpandedKeys={["documents"]}
    >
      <Tree.Item id="documents" textValue="Documents">
        <Tree.ItemContent>
          <Tree.Indicator />
          Documents
        </Tree.ItemContent>
        <Tree.Item id="report" textValue="Weekly Report">
          <Tree.ItemContent>
            <Tree.Indicator />
            Weekly Report
          </Tree.ItemContent>
        </Tree.Item>
      </Tree.Item>
    </Tree.Root>
  );
};
```

### Sizes

`Tree.Root` accepts a `size` prop (`"sm"` or `"md"`, default `"md"`) that scales
row height, font size, chevron size and indentation step.

```jsx live-dev
const App = () => {
  return (
    <Tree.Root aria-label="Files" size="sm" defaultExpandedKeys={["documents"]}>
      <Tree.Item id="documents" textValue="Documents">
        <Tree.ItemContent>
          <Tree.Indicator />
          Documents
        </Tree.ItemContent>
        <Tree.Item id="report" textValue="Weekly Report">
          <Tree.ItemContent>
            <Tree.Indicator />
            Weekly Report
          </Tree.ItemContent>
        </Tree.Item>
      </Tree.Item>
    </Tree.Root>
  );
};
```

## Drag and drop

Drag-and-drop is opt-in and driven entirely through Nimbus — no
`react-aria-components` or `react-stately` imports required. The `useTree` hook
owns the hierarchical state and wires drag-and-drop for you: reorder between
siblings, and re-parent by dropping an item _onto_ a group. Spread its result
onto `Tree.Root`.

You don't author the drag handle. When the tree allows dragging,
`Tree.ItemContent` renders a keyboard- and screen-reader-operable drag handle
automatically (`Enter` to grab, arrow keys to move, `Enter` to drop), with a
consistent icon and a localized accessible name. Drop indicators for valid
positions are provided automatically. Your render function is identical to a
non-draggable tree — enabling drag-and-drop is the only change.

```tsx
import { Tree, useTree } from "@commercetools/nimbus";

function DraggableTree() {
  const tree = useTree({
    initialItems: [
      {
        id: "documents",
        title: "Documents",
        children: [{ id: "report", title: "Weekly Report" }],
      },
      { id: "photos", title: "Photos", children: [] },
    ],
    getKey: (item) => item.id,
    getChildren: (item) => item.children ?? [],
    dragAndDrop: true,
  });

  const renderItem = (node) => (
    <Tree.Item id={node.key} textValue={node.value.title}>
      <Tree.ItemContent>
        <Tree.Indicator />
        {node.value.title}
      </Tree.ItemContent>
      <Tree.SubTree items={node.children}>{renderItem}</Tree.SubTree>
    </Tree.Item>
  );

  return (
    <Tree.Root aria-label="Files" {...tree}>
      {renderItem}
    </Tree.Root>
  );
}
```

For custom drag data or accepted external drag types, pass an options object
instead of `true`: `dragAndDrop={{ getItems, acceptedDragTypes }}`. For
programmatic edits, the same `tree` object exposes `insert`, `remove`, `move`,
`update`, and more.

## Accessibility

The Tree component is built with React Aria and follows the WAI-ARIA `treegrid`
pattern. It provides:

**Roles:**

- The root exposes `role="treegrid"`; each item is a `role="row"` containing a
  `role="gridcell"`.

**Labeling:**

- Each row exposes `aria-level` reflecting its depth, `aria-expanded` when it
  has children, and `aria-selected` when selection is enabled.
- Always provide an accessible name on `Tree.Root` via `aria-label` or
  `aria-labelledby`.
- Provide a meaningful `textValue` on each `Tree.Item` (used for type-ahead)
  when the label is not a plain string.

**Keyboard navigation:**

- `Up` / `Down` — move between visible rows
- `Right` / `Left` — expand/collapse, or move to a child/parent
- `Home` / `End` — move to the first/last visible row
- _type characters_ — type-ahead to the next matching row

When drag-and-drop is enabled, `Tree.ItemContent` automatically renders a drag
handle per item so the reorder is operable by keyboard and screen reader, not
just pointer — you don't need to add one.
