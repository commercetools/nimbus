## Accessibility

Tooltip triggers must be focusable and hoverable in order to ensure that all
users can activate them. When displayed, the Tooltip automatically associates
the tooltip with the trigger element so that it is described by the tooltip
content.

Authors should ensure that elements with the
[role](https://developer.mozilla.org/en-US/docs/Web/Accessibility/ARIA/Reference/Roles)
tooltip are referenced through the use of
[aria-describedby](https://developer.mozilla.org/en-US/docs/Web/Accessibility/ARIA/Reference/Attributes/aria-describedby)
before or at the time the tooltip is displayed.

## Accessibility standards

### 1. Perceivable:

- **1.1.1 Non-text Content (Level A):** Provide a text alternative (alt text or
  equivalent) for non-text content within tooltips.
- **1.3.1 Info and Relationships (Level A):** Programmatically establish the
  relationship between a tooltip and its trigger element using ARIA attributes
  like `aria-describedby`. Assign an id to the trigger element and point
  `aria-describedby` to that id.
- **1.4.1 Use of Color (Level A):** Avoid relying solely on color to convey
  information within tooltips.
- **1.4.3 Contrast (Minimum) (Level AA):** Ensure tooltip text meets minimum
  contrast ratio requirements. Use a contrast checking tool.

#### 2. Operable:

- **2.1.1 Keyboard (Level A):** Make tooltips triggered by keyboard focus
  operable via keyboard. Allow users to navigate to the trigger using the Tab
  key. Make the tooltip appear on focus. Prevent tooltips from trapping keyboard
  focus. Make interactive tooltip content keyboard accessible.
- **2.4.7 Focus Visible (Level AA):** Provide a visible focus indicator for the
  tooltip trigger element when it receives keyboard focus.

#### 3.Understandable:

- **3.1.1 Language of Page (Level A):** Programmatically determine the tooltip
  content's language. Use the `lang` attribute on the tooltip element.
- **3.2.1 On Focus (Level A):** Prevent tooltips that appear on focus from
  unexpectedly changing the page context.
- **3.3.2 Labels or Instructions (Level A):** Write tooltip text that is clear,
  concise, and easy to understand.

#### 4. Robust:

- **4.1.2 Name, Role, Value (Level A):** Ensure assistive technologies correctly
  interpret the role of the tooltip trigger element. Use semantic HTML. Use
  `aria-describedby` to associate the tooltip with the trigger.
