## Overview

Alerts provide feedback to the user about the status of an action or a system
event. These specifications define the standard styles and variations for
alerts within our design system, covering different visual treatments,
layouts, and severities for diverse use cases.

### Resources

Deep dive into implementation details and access the Nimbus design library.

[Figma library](https://www.figma.com/design/AvtPX6g7OGGCRvNlatGOIY/NIMBUS-design-system?node-id=2702-5723)

## Variables

Get familiar with the features.

### Adjustable elements

Several of the features are flexible and optional for many use cases.
Non-optional elements are the message, and the icons that are tied to the type
of messaging.

#### Buttons and links

The alert component allows for two buttons and links within the paragraph
message. These buttons and links can be customized for any messaging needs.

#### Close and dismiss buttons

Features that may be used for closing or dismissing the alert message are
optional. Removing these elements will make this message persistent for the
user. There is no need to duplicate a close action, using the close icon in the
right corner or a dismiss button is enough for a user.

#### Title and description

The message is needed in all uses. Please always utilize messaging for the user.
Use of the title is optional, use if it gives greater clarity to the user.

```jsx live
const App = () => (
  <Stack direction="column">
    <Alert.Root colorPalette="critical" variant="subtle">
      <Alert.Title>Lorem Ipsum</Alert.Title>
      <Alert.Description>
        Lorem ipsum dolor sit amet, consectetur adipiscing elit. <Link>Read more</Link>
      </Alert.Description>
      <Alert.Actions>
        <Stack direction="row" alignItems="center">
          <Button variant="outline">Dismiss</Button>
          <Button variant="outline">View changes</Button>
        </Stack>
      </Alert.Actions>
      <Alert.DismissButton />
    </Alert.Root>

    <Alert.Root colorPalette="info" variant="subtle">
      <Alert.Title>Lorem Ipsum</Alert.Title>
      <Alert.Description>
        Lorem ipsum dolor sit amet, consectetur adipiscing elit. <Link>Read more</Link>
      </Alert.Description>
      <Alert.DismissButton />
    </Alert.Root>

    <Alert.Root colorPalette="warning" variant="subtle">
      <Alert.Title>Lorem Ipsum</Alert.Title>
      <Alert.Description>
        Lorem ipsum dolor sit amet, consectetur adipiscing elit. <Link>Read more</Link>
      </Alert.Description>
    </Alert.Root>

    <Alert.Root colorPalette="positive" variant="subtle">
      <Alert.Description>
        Lorem ipsum dolor sit amet, consectetur adipiscing elit. <Link>Read more</Link>
      </Alert.Description>
    </Alert.Root>
  </Stack>
)
```

### Style

The `variant` prop controls emphasis. `subtle` is the default; `outlined` is a
deprecated alias for `subtle` and renders identically — prefer `subtle` in new
usage.

#### Subtle (default)

Use when getting attention is more important. These can be used inline, but they
take up space and it is not as easy to slide into compact areas as much the flat
variant.

```jsx live
const App = () => (
  <Alert.Root colorPalette="positive" variant="subtle" width="8000">
    <Alert.Title>Success! Your Account is Verified</Alert.Title>
    <Alert.Description>
      Congratulations! Your account has been successfully verified. You now have full access to all features.
    </Alert.Description>
  </Alert.Root>
)
```

#### Outline

A border-only treatment with minimal fill — lighter-weight than `subtle` while
still standing apart from the surrounding content.

```jsx live
const App = () => (
  <Alert.Root colorPalette="info" variant="outline" width="8000">
    <Alert.Title>New Feature Available</Alert.Title>
    <Alert.Description>
      We've added a new 'Smart Suggestions' feature to help you find relevant content faster.
    </Alert.Description>
  </Alert.Root>
)
```

#### Solid

A high-contrast, filled treatment. Use sparingly for messages that must stand
out the most, such as a page-level banner.

```jsx live
const App = () => (
  <Alert.Root colorPalette="critical" variant="solid" width="8000">
    <Alert.Title>Payment Processing Error</Alert.Title>
    <Alert.Description>
      A payment processing error occurred for order #12345.
    </Alert.Description>
  </Alert.Root>
)
```

#### Flat

Use for inline messages specific to a field. You can also use this in dense
areas with popovers. Do not use as a global page alert. In these placements, use
links instead of buttons if a user needs to navigate to more information.

```jsx live
const App = () => (
  <Stack direction="column" alignItems="start">
    <Text fontWeight="500">Email</Text>
    <TextInput type="email" />
    <Alert.Root colorPalette="positive" variant="flat">
      <Alert.Description>
        Email is verified
      </Alert.Description>
    </Alert.Root>
  </Stack>
)
```

#### Width

The width is flexible, intended for many inline placements and page wide areas.

```jsx live
const App = () => (
  <Grid templateColumns="repeat(6, 1fr)" gap="400">
    <Grid.Item colSpan={2}>
      <Alert.Root colorPalette="critical" variant="subtle">
        <Alert.Description>
          Lorem ipsum dolor sit amet, consectetur adipiscing elit. <Link>Read more</Link>
        </Alert.Description>
      </Alert.Root>
    </Grid.Item>

    <Grid.Item colSpan={3}>
      <Alert.Root colorPalette="critical" variant="subtle">
        <Alert.Description>
          Lorem ipsum dolor sit amet, consectetur adipiscing elit. <Link>Read more</Link>
        </Alert.Description>
      </Alert.Root>
    </Grid.Item>

    <Grid.Item colSpan={6}>
      <Alert.Root colorPalette="critical" variant="subtle">
        <Alert.Description>
          Lorem ipsum dolor sit amet, consectetur adipiscing elit. <Link>Read more</Link>
        </Alert.Description>
        <Alert.Actions>
          <Button variant="outline">View Changes</Button>
        </Alert.Actions>
        <Alert.DismissButton />
      </Alert.Root>
    </Grid.Item>
  </Grid>
);
```

### Layout

The `layout` prop controls how the alert's parts are arranged. `stack` is the
default; use `inline` for compact, single-line confirmations (for example, an
agent-confirmation banner with an undo action); use `banner` for full-width,
edge-to-edge, page-level messages.

```jsx live
const App = () => (
  <Stack direction="column" gap="400">
    <Alert.Root colorPalette="info" layout="stack" variant="subtle">
      <Alert.Title>Stack (default)</Alert.Title>
      <Alert.Description>
        Icon and content stack vertically; actions and the dismiss button sit
        below the description.
      </Alert.Description>
    </Alert.Root>

    <Alert.Root
      colorPalette="positive"
      layout="inline"
      role="group"
      aria-label="Suggestion approved"
      hideIcon
    >
      <Alert.Title>Suggestion approved</Alert.Title>
      <Alert.Description>
        Applied the recommended discount to 3 products.
      </Alert.Description>
      <Alert.Actions>
        <Button variant="outline">Undo</Button>
      </Alert.Actions>
    </Alert.Root>

    <Alert.Root colorPalette="warning" layout="banner" variant="subtle">
      <Alert.Title>Scheduled maintenance</Alert.Title>
      <Alert.Description>
        The system will be unavailable tonight from 02:00 to 03:00 UTC.
      </Alert.Description>
    </Alert.Root>
  </Stack>
);
```

### Color palette

#### Positive

Positive alerts should be used for important updates, and product wide
announcements. They should not be used in place of toasts or confirmation
messages.

```jsx live
const App = () => (
  <Grid templateColumns="repeat(2, 1fr)" gap="400">
    <Alert.Root colorPalette="positive" variant="subtle">
      <Alert.Title>Success! Your Account is Verified</Alert.Title>
      <Alert.Description>
        Congratulations! Your account has been successfully verified. You now have full access to all features.
      </Alert.Description>
    </Alert.Root>

    <Alert.Root colorPalette="positive" variant="flat">
      <Alert.Title>Success! Your Account is Verified</Alert.Title>
      <Alert.Description>
        Congratulations! Your account has been successfully verified. You now have full access to all features.
      </Alert.Description>
    </Alert.Root>
  </Grid>
);
```

#### Info

Should be used when extra explanation is needed, this alert is meant to be more
temporary than using an info icon with a tooltip which declutters persistent
information for users.

```jsx live
const App = () => (
  <Grid templateColumns="repeat(2, 1fr)" gap="400">
    <Alert.Root colorPalette="info" variant="subtle">
      <Alert.Title>New Feature Available</Alert.Title>
      <Alert.Description>
        We've added a new 'Smart Suggestions' feature to help you find relevant content faster. <Link>Check it out in your settings.</Link>
      </Alert.Description>
    </Alert.Root>

    <Alert.Root colorPalette="info" variant="flat">
      <Alert.Title>New Feature Available</Alert.Title>
      <Alert.Description>
        We've added a new 'Smart Suggestions' feature to help you find relevant content faster. <Link>Check it out in your settings.</Link>
      </Alert.Description>
    </Alert.Root>
  </Grid>
)
```

#### Warning

Messages potential problems or situations that require user attention to prevent
issues. They require the user to take action to make needed corrections or
confirmation.

```jsx live
const App = () => (
  <Grid templateColumns="repeat(2, 1fr)" gap="400">
    <Alert.Root colorPalette="warning" variant="subtle">
      <Alert.Title>Low Storage Space</Alert.Title>
      <Alert.Description>
        Your device is running low on storage. Some features may be disabled until you free up space. <Link>Manage storage.</Link>
      </Alert.Description>
    </Alert.Root>

    <Alert.Root colorPalette="warning" variant="flat">
      <Alert.Title>Low Storage Space</Alert.Title>
      <Alert.Description>
        Your device is running low on storage. Some features may be disabled until you free up space. <Link>Manage storage.</Link>
      </Alert.Description>
    </Alert.Root>
  </Grid>
)
```

#### Critical

These are used to message immediate, severe problems that require urgent action.
They indicate that something has already gone wrong and needs to be addressed
immediately. Because these must interrupt the user, pass `role="alert"`
explicitly — the default `role` is the polite `"status"` (see the
Accessibility tab for the full announcement model).

```jsx live
const App = () => (
  <Grid templateColumns="repeat(2, 1fr)" gap="400">
    <Alert.Root colorPalette="critical" variant="subtle" role="alert">
      <Alert.Title>Payment Processing Error</Alert.Title>
      <Alert.Description>
        A payment processing error occurred for order #12345. Please <Link>verify your payment gateway settings</Link> immediately to prevent further issues.
      </Alert.Description>
    </Alert.Root>

    <Alert.Root colorPalette="critical" variant="flat" role="alert">
      <Alert.Title>Payment Processing Error</Alert.Title>
      <Alert.Description>
        A payment processing error occurred for order #12345. Please <Link>verify your payment gateway settings</Link> immediately to prevent further issues.
      </Alert.Description>
    </Alert.Root>
  </Grid>
)
```

#### Neutral

Use for messages that don't map to a status severity — general notices that
aren't positive, informational, a warning, or an error. Neutral has no
automatic icon; pair it with an explicit `Alert.Icon` or use `hideIcon`.

```jsx live
const App = () => (
  <Alert.Root colorPalette="neutral" variant="subtle" width="8000" hideIcon>
    <Alert.Title>Heads up</Alert.Title>
    <Alert.Description>
      This workspace will be archived at the end of the billing period.
    </Alert.Description>
  </Alert.Root>
)
```
