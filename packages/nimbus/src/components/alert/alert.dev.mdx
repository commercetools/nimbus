## Getting started

### Import

```tsx
import { Alert, type AlertProps } from '@commercetools/nimbus';
```

### Basic usage

The Alert component uses a compound component pattern. You must wrap all parts in `Alert.Root`. The component automatically handles layout and icon selection based on the `colorPalette`.

```jsx live-dev
const App = () => (
  <Alert.Root colorPalette="info">
    <Alert.Title>Information</Alert.Title>
    <Alert.Description>
      This is a basic alert message notifying you of a change.
    </Alert.Description>
  </Alert.Root>
)
```

## Usage examples

### Color palettes

The `colorPalette` prop determines the semantic color and the accompanying icon. The following examples demonstrate every palette available for this component (`positive`, `warning`, `critical`, `info`, `neutral`). `neutral` has no automatic icon — pair it with a custom `Alert.Icon` or `hideIcon`.

```jsx live-dev
const App = () => (
  <Stack direction="column" gap="400">
    <Alert.Root colorPalette="positive">
      <Alert.Title>Positive</Alert.Title>
      <Alert.Description>Operation completed successfully.</Alert.Description>
    </Alert.Root>

    <Alert.Root colorPalette="warning">
      <Alert.Title>Warning</Alert.Title>
      <Alert.Description>Please review your changes.</Alert.Description>
    </Alert.Root>

    <Alert.Root colorPalette="critical" role="alert">
      <Alert.Title>Critical</Alert.Title>
      <Alert.Description>An error occurred while saving.</Alert.Description>
    </Alert.Root>

    <Alert.Root colorPalette="info">
      <Alert.Title>Info</Alert.Title>
      <Alert.Description>New updates are available.</Alert.Description>
    </Alert.Root>

    <Alert.Root colorPalette="neutral" hideIcon>
      <Alert.Title>Neutral</Alert.Title>
      <Alert.Description>General notice with no severity.</Alert.Description>
    </Alert.Root>
  </Stack>
)
```

### Visual variants (emphasis)

The `variant` prop controls emphasis: `subtle` (default, tinted card), `outline` (border-only, minimal fill), `solid` (high-contrast fill), and `flat` (no chrome). `variant="outlined"` is a deprecated alias for `subtle` and renders identically — prefer `subtle`.

```jsx live-dev
const App = () => (
  <Stack direction="column" gap="400">
    <Alert.Root variant="subtle" colorPalette="info">
      <Alert.Title>Subtle (default)</Alert.Title>
      <Alert.Description>
        Tinted card. Best when the alert needs to stand out against surrounding content.
      </Alert.Description>
    </Alert.Root>

    <Alert.Root variant="outline" colorPalette="info">
      <Alert.Title>Outline</Alert.Title>
      <Alert.Description>
        Border-only, minimal fill. Lighter-weight than subtle.
      </Alert.Description>
    </Alert.Root>

    <Alert.Root variant="solid" colorPalette="info">
      <Alert.Title>Solid</Alert.Title>
      <Alert.Description>
        High-contrast fill. Use sparingly for the messages that must stand out most.
      </Alert.Description>
    </Alert.Root>

    <Alert.Root variant="flat" colorPalette="info">
      <Alert.Title>Flat</Alert.Title>
      <Alert.Description>
        No chrome. Suitable for dense interfaces or secondary notifications.
      </Alert.Description>
    </Alert.Root>
  </Stack>
)
```

### Layout

The `layout` prop controls how parts are arranged: `stack` (default — icon and content stack vertically), `inline` (content leading, actions trailing, wraps on narrow widths), or `banner` (full-width, edge-to-edge, page-level).

```jsx live-dev
const App = () => (
  <Stack direction="column" gap="400">
    <Alert.Root layout="stack" colorPalette="info">
      <Alert.Title>Stack (default)</Alert.Title>
      <Alert.Description>
        Icon and content stack vertically; actions and the dismiss button sit below the description.
      </Alert.Description>
    </Alert.Root>

    <Alert.Root layout="inline" colorPalette="info">
      <Alert.Title>Inline</Alert.Title>
      <Alert.Description>
        Content leads, actions trail on the same row, and wrap below on narrow widths.
      </Alert.Description>
      <Alert.Actions>
        <Button variant="outline">Action</Button>
      </Alert.Actions>
    </Alert.Root>

    <Alert.Root layout="banner" colorPalette="warning">
      <Alert.Title>Banner</Alert.Title>
      <Alert.Description>
        Full-width, edge-to-edge — no border radius. Intended for page-level messages.
      </Alert.Description>
    </Alert.Root>
  </Stack>
)
```

#### Silent inline confirmation (agent-confirmation pattern)

`layout="inline"` combined with `role="group"` + `aria-label` and `hideIcon` produces a compact confirmation banner that is not announced as a live region — useful for confirming a user-initiated action (e.g. an agent applying a suggestion) that the user is already watching, where an additional announcement would be redundant.

```jsx live-dev
const App = () => {
  const [applied, setApplied] = useState(true);

  if (!applied) {
    return <Text>Change undone.</Text>;
  }

  return (
    <Alert.Root
      layout="inline"
      colorPalette="positive"
      role="group"
      aria-label="Suggestion approved"
      hideIcon
    >
      <Alert.Title>Suggestion approved</Alert.Title>
      <Alert.Description>
        Applied the recommended discount to 3 products.
      </Alert.Description>
      <Alert.Actions>
        <Button variant="outline" onPress={() => setApplied(false)}>
          Undo
        </Button>
      </Alert.Actions>
    </Alert.Root>
  );
}
```

### Icon customization

Omit `Alert.Icon` for the automatic status icon (based on `colorPalette`). Provide `Alert.Icon` with any icon to override it, or use `hideIcon` on `Alert.Root` to suppress the icon entirely (it takes precedence over both the automatic icon and an explicit `Alert.Icon`).

```jsx live-dev
const App = () => (
  <Stack direction="column" gap="400">
    <Alert.Root colorPalette="info">
      <Alert.Icon><Icons.StarOutline /></Alert.Icon>
      <Alert.Title>Custom icon</Alert.Title>
      <Alert.Description>Replaces the automatic status icon.</Alert.Description>
    </Alert.Root>

    <Alert.Root colorPalette="info" hideIcon>
      <Alert.Title>No icon</Alert.Title>
      <Alert.Description>hideIcon suppresses the icon entirely.</Alert.Description>
    </Alert.Root>
  </Stack>
)
```

### Flexible composition

The component parts (`Title`, `Description`, `Actions`, `DismissButton`) can be composed as needed. The `Alert.Root` handles the layout regardless of the order in JSX.

**Title only:**

```jsx live-dev
const App = () => (
  <Alert.Root colorPalette="positive">
    <Alert.Title>Success: Item created.</Alert.Title>
  </Alert.Root>
)
```

**Description only:**

```jsx live-dev
const App = () => (
  <Alert.Root colorPalette="info">
    <Alert.Description>
      System maintenance scheduled for tonight at 02:00 UTC.
    </Alert.Description>
  </Alert.Root>
)
```

### Adding actions

Use `Alert.Actions` to group action buttons. These will be positioned correctly within the alert layout.

```jsx live-dev
const App = () => (
  <Alert.Root colorPalette="warning">
    <Alert.Title>Storage Limit Reached</Alert.Title>
    <Alert.Description>
      You have used 95% of your available storage.
    </Alert.Description>
    <Alert.Actions>
      <Stack direction="row" gap="200">
        <Button variant="solid">Upgrade Plan</Button>
        <Button variant="ghost">Dismiss</Button>
      </Stack>
    </Alert.Actions>
  </Alert.Root>
)
```

### Dismissible alerts

There are two ways to add a close action. Either way, **you** must handle the dismissal logic (e.g., removing the alert from the DOM) — Alert has no internal open/closed state.

**`dismissible` + `onDismiss` (convenience):** set `dismissible` and Alert renders a localized, correctly-positioned dismiss button for you; `onDismiss` fires when it's pressed.

```jsx live-dev
const App = () => {
  const [isVisible, setIsVisible] = useState(true);

  if (!isVisible) {
    return (
      <Button onPress={() => setIsVisible(true)}>
        Reset Example
      </Button>
    );
  }

  return (
    <Alert.Root
      colorPalette="info"
      dismissible
      onDismiss={() => setIsVisible(false)}
    >
      <Alert.Title>Dismissible Alert</Alert.Title>
      <Alert.Description>
        Click the close button to dismiss this message.
      </Alert.Description>
    </Alert.Root>
  );
}
```

**Composable `Alert.DismissButton` (advanced):** render it explicitly when you need to customize its placement, `variant`, `size`, or other button props. If both `dismissible` and a manual `Alert.DismissButton` child are present, the manual slot wins — the automatic button is not rendered.

```jsx live-dev
const App = () => {
  const [isVisible, setIsVisible] = useState(true);

  if (!isVisible) {
    return (
      <Button onPress={() => setIsVisible(true)}>
        Reset Example
      </Button>
    );
  }

  return (
    <Alert.Root colorPalette="info" variant="subtle">
      <Alert.Title>Dismissible Alert</Alert.Title>
      <Alert.Description>
        Click the close button to dismiss this message.
      </Alert.Description>
      <Alert.DismissButton onPress={() => setIsVisible(false)} />
    </Alert.Root>
  );
}
```

## Component requirements

## Accessibility

The `Alert` component has built-in accessibility features:

- **Role**: `role` is a native, overridable prop on `Alert.Root`. The default is `role="status"` — a polite live region that doesn't interrupt the user. For critical/interrupting messages, pass `role="alert"` explicitly. For silent inline confirmations that shouldn't be announced as a live region at all, pass `role="group"` with an `aria-label`. See the Accessibility tab for the full announcement model, including live-region priming guidance.
- **Icons**: The automatic status icon is decorative and automatically hidden from screen readers to avoid redundancy. A custom `Alert.Icon` is rendered as-is — mark it `aria-hidden` yourself if it's purely decorative, or give it an accessible label if it conveys meaning on its own.
- **Dismiss Button**: The `Alert.DismissButton` (composable or auto-rendered via `dismissible`) comes with a default accessible label ("Dismiss") which can be localized or overridden via `aria-label`.

If your use case requires tracking and analytics for this component, it is good practice to add a **persistent**, **unique** id to the component:

```tsx
const PERSISTENT_ID = "example-alert";

export const Example = () => (
  <Alert.Root id={PERSISTENT_ID}>
    <Alert.Title>Notification</Alert.Title>
  </Alert.Root>
);
```

### Composition constraints

The `Alert` component manages its own layout grid. To ensure correct placement:

- Keep sub-components (`Title`, `Description`, `Actions`, `DismissButton`) as direct children of `Alert.Root`.
- Do not wrap sub-components in other elements (like `div` or `Stack`), as these wrappers will be rendered as part of the layout and may break the visual structure.
- You *can* use layout components *inside* `Alert.Actions` or `Alert.Description` to organize content within those slots.
