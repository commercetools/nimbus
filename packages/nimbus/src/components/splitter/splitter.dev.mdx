## Getting started

### Import

```tsx
import { Splitter, type SplitterRootProps } from "@commercetools/nimbus";
```

### Basic usage

`Splitter.Root` wraps exactly two `Splitter.Pane` children with one
`Splitter.Handle` between them. Each pane carries a string `id`; the initial
split is set with `defaultSizes` (percentages, keyed by id).

```jsx live-dev
const App = () => (
  <Box height="320px" width="100%" borderWidth="25" borderColor="neutral.6">
    <Splitter.Root
      defaultSizes={{ nav: 30, main: 70 }}
      panes={{ nav: { minSize: 15 }, main: { minSize: 25 } }}
    >
      <Splitter.Pane id="nav">
        <Box p="400" bg="indigo.3" h="100%">
          <Text>
            Drag the handle to resize. Tab to it, then use the arrow keys.
          </Text>
        </Box>
      </Splitter.Pane>
      <Splitter.Handle />
      <Splitter.Pane id="main">
        <Box p="400" bg="amber.3" h="100%">
          <Text>Main</Text>
        </Box>
      </Splitter.Pane>
    </Splitter.Root>
  </Box>
);
```

## Usage examples

### Orientation

`orientation="vertical"` stacks the panes and makes ArrowUp / ArrowDown the
active keys. The handle's `aria-orientation` reflects this (W3C separator
semantics describe the boundary axis, not the layout axis).

```jsx live-dev
const App = () => (
  <Box height="360px" width="100%" borderWidth="25" borderColor="neutral.6">
    <Splitter.Root
      orientation="vertical"
      defaultSizes={{ top: 40, bottom: 60 }}
    >
      <Splitter.Pane id="top">
        <Box p="400" bg="teal.3" h="100%">
          <Text>Top</Text>
        </Box>
      </Splitter.Pane>
      <Splitter.Handle />
      <Splitter.Pane id="bottom">
        <Box p="400" bg="rose.3" h="100%">
          <Text>Bottom</Text>
        </Box>
      </Splitter.Pane>
    </Splitter.Root>
  </Box>
);
```

### Per-pane constraints

Sizing is split across two props by lifecycle, not by accident:

- **`defaultSizes`** holds the _dynamic_ proportions — the live state that
  drags, collapses, and persistence change. It shares its `Record<id, number>`
  shape with `onSizesChange` / `onSizesChangeEnd` so a stored value flows
  straight back in (see [Persistence](#persistence)).
- **`panes`** holds the _static_ per-pane configuration — `minSize`,
  `collapsible`, `collapsedSize` — that never changes at runtime.

Both are keyed by the same pane ids. Keeping the persisted proportions in one
flat prop is what makes hydration a one-liner.

Each entry in the `panes` map accepts a `minSize`. A pane's upper bound is the
complement of its partner's `minSize` (`max = 100 − partner.minSize`), so a
single value per side fully describes the boundary — there is no separate
`maxSize`.

```jsx live-dev
const App = () => (
  <Box height="320px" width="100%" borderWidth="25" borderColor="neutral.6">
    <Splitter.Root
      defaultSizes={{ nav: 30, main: 70 }}
      panes={{ nav: { minSize: 15 }, main: { minSize: 25 } }}
    >
      <Splitter.Pane id="nav">
        <Box p="400" bg="indigo.3" h="100%">
          <Text>Nav (min 15 → max 75)</Text>
        </Box>
      </Splitter.Pane>
      <Splitter.Handle />
      <Splitter.Pane id="main">
        <Box p="400" bg="amber.3" h="100%">
          <Text>Main (min 25)</Text>
        </Box>
      </Splitter.Pane>
    </Splitter.Root>
  </Box>
);
```

### Collapsible panes

Mark a pane `collapsible` and control which pane is collapsed with
`collapsedPane` (a single pane id or `null` — only one collapses at a time).
Because collapse is plain controlled state, any control can drive it; Enter on
the focused handle toggles it too. `onCollapsedPaneChange` fires on every
transition.

```jsx live-dev
const App = () => {
  const [collapsed, setCollapsed] = useState(null);

  return (
    <Stack direction="column" gap="300">
      <Button
        alignSelf="start"
        onPress={() => setCollapsed((c) => (c === "nav" ? null : "nav"))}
      >
        {collapsed === "nav" ? "Show nav" : "Hide nav"}
      </Button>
      <Box height="280px" width="100%" borderWidth="25" borderColor="neutral.6">
        <Splitter.Root
          defaultSizes={{ nav: 30, main: 70 }}
          panes={{
            nav: { minSize: 10, collapsible: true },
            main: { minSize: 20 },
          }}
          collapsedPane={collapsed}
          onCollapsedPaneChange={setCollapsed}
        >
          <Splitter.Pane id="nav">
            <Box p="400" bg="indigo.3" h="100%">
              <Text>Nav</Text>
            </Box>
          </Splitter.Pane>
          <Splitter.Handle />
          <Splitter.Pane id="main">
            <Box p="400" bg="amber.3" h="100%">
              <Text>Main</Text>
            </Box>
          </Splitter.Pane>
        </Splitter.Root>
      </Box>
    </Stack>
  );
};
```

Double-click on the handle restores the boundary to the sizes resolved on mount;
set `isDoubleClickDisabled` to suppress it.

### Persistence

Sizes are uncontrolled for drag performance, so persist them in your app with
any storage: hydrate `defaultSizes` from stored state and write back in
`onSizesChangeEnd` (it fires once per settled interaction, so no debouncing is
needed). Collapse persists through its controlled state.

```tsx
// `useLocalStorage` is any storage hook of your choosing.
const [sizes, setSizes] = useLocalStorage("ide-layout", { nav: 30, main: 70 });
const [collapsed, setCollapsed] = useLocalStorage("ide-collapsed", null);

<Splitter.Root
  defaultSizes={sizes}
  onSizesChangeEnd={setSizes}
  collapsedPane={collapsed}
  onCollapsedPaneChange={setCollapsed}
  panes={{ nav: { minSize: 10, collapsible: true }, main: { minSize: 20 } }}
>
  <Splitter.Pane id="nav">…</Splitter.Pane>
  <Splitter.Handle />
  <Splitter.Pane id="main">…</Splitter.Pane>
</Splitter.Root>;
```

`onSizesChange` (live, every drag tick) is also available when you need a
real-time read-out; prefer `onSizesChangeEnd` for persistence.

### Disabled state

`isDisabled` makes the whole splitter non-interactive: the handle leaves the tab
order, gets `aria-disabled`, and ignores drag, keyboard, and collapse input.

```jsx live-dev
const App = () => (
  <Box height="240px" width="100%" borderWidth="25" borderColor="neutral.6">
    <Splitter.Root isDisabled defaultSizes={{ nav: 30, main: 70 }}>
      <Splitter.Pane id="nav">
        <Box p="400" bg="indigo.3" h="100%">
          <Text>Nav</Text>
        </Box>
      </Splitter.Pane>
      <Splitter.Handle />
      <Splitter.Pane id="main">
        <Box p="400" bg="amber.3" h="100%">
          <Text>Main</Text>
        </Box>
      </Splitter.Pane>
    </Splitter.Root>
  </Box>
);
```

### Pixel-precise sizes

Percentages carry full float precision end-to-end (nothing in the size pipeline
is rounded — only the handle's `aria-valuenow` is rounded, for assistive
technology). To land on an exact pixel at a known container width, pass the
computed float — `250px` in an `800px` container is `31.25`.

```jsx live-dev
const App = () => (
  <Box height="240px" width="100%" borderWidth="25" borderColor="neutral.6">
    <Splitter.Root defaultSizes={{ nav: 31.25, main: 68.75 }}>
      <Splitter.Pane id="nav">
        <Box p="400" bg="indigo.3" h="100%">
          <Text>31.25%</Text>
        </Box>
      </Splitter.Pane>
      <Splitter.Handle />
      <Splitter.Pane id="main">
        <Box p="400" bg="amber.3" h="100%">
          <Text>68.75%</Text>
        </Box>
      </Splitter.Pane>
    </Splitter.Root>
  </Box>
);
```

### Three or more regions

The Splitter is two-pane by design. For additional regions, nest a Splitter
inside a pane's children; each splitter owns its own state independently.

```jsx live-dev
const App = () => (
  <Box height="360px" width="100%" borderWidth="25" borderColor="neutral.6">
    <Splitter.Root defaultSizes={{ nav: 25, rest: 75 }}>
      <Splitter.Pane id="nav">
        <Box p="400" bg="indigo.3" h="100%">
          <Text>Nav</Text>
        </Box>
      </Splitter.Pane>
      <Splitter.Handle />
      <Splitter.Pane id="rest">
        <Splitter.Root defaultSizes={{ main: 65, aside: 35 }}>
          <Splitter.Pane id="main">
            <Box p="400" bg="amber.3" h="100%">
              <Text>Main</Text>
            </Box>
          </Splitter.Pane>
          <Splitter.Handle />
          <Splitter.Pane id="aside">
            <Box p="400" bg="rose.3" h="100%">
              <Text>Aside</Text>
            </Box>
          </Splitter.Pane>
        </Splitter.Root>
      </Splitter.Pane>
    </Splitter.Root>
  </Box>
);
```

## Component requirements

### Structure

- `Splitter.Root` must contain exactly two `Splitter.Pane` children with one
  `Splitter.Handle` between them; a development-time warning is emitted
  otherwise.
- Every `Splitter.Pane` requires a unique `id`. Per-pane configuration
  (`minSize`, `collapsible`, `collapsedSize`) lives on `Splitter.Root` in the
  `panes` map keyed by that id — nothing is configured on the pane itself.

## Accessibility

The Splitter handles the W3C window-splitter semantics internally: the handle is
`role="separator"` with `aria-orientation`, `aria-valuenow` / `aria-valuemin` /
`aria-valuemax`, `aria-valuetext`, and `aria-controls` pointing at the previous
pane. It ships a localized default `aria-label` ("Resize panes"); you can
override it per handle for a more specific label:

```tsx
<Splitter.Handle aria-label={msg.format("resizeNavigation")} />
```

If your use case requires tracking and analytics, it is good practice to add a
**persistent**, **unique** id to the handle:

```tsx
const PERSISTENT_ID = "ide-layout-splitter-handle";

export const Example = () => (
  <Splitter.Root defaultSizes={{ nav: 30, main: 70 }}>
    <Splitter.Pane id="nav">…</Splitter.Pane>
    <Splitter.Handle id={PERSISTENT_ID} />
    <Splitter.Pane id="main">…</Splitter.Pane>
  </Splitter.Root>
);
```

#### Keyboard navigation

When the handle has focus:

- `Tab` / `Shift+Tab`: Move focus to/from the handle (in DOM order).
- `ArrowLeft` / `ArrowRight` (horizontal) or `ArrowUp` / `ArrowDown` (vertical):
  Move the boundary by `keyboardStep` percentage points.
- `Home` / `End`: Jump the boundary to the previous pane's minimum / maximum.
- `Enter`: Toggle collapse of the adjacent collapsible pane.
