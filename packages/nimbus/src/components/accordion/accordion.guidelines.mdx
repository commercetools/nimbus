## Guidelines

Accordion guidelines offer a set of standardized rules and recommendations that
ensure consistency, usability, and accessibility across all design elements and
interactions.

### Best practices

- **Label clarity:** Headers must be clear and descriptive so users understand
  what information is contained within each section.
- **Visual cues:** Use clear visual cues to indicate that a section can be
  expanded and its current state. If other icons and controls are present,
  ensure that their use is clear and explained.
- **Accessibility:** Ensure accordions are accessible to users with
  disabilities, including keyboard navigation and screen reader compatibility.
- **Avoid nesting too deeply:** Overly nested accordions can become confusing
  and difficult to navigate.

### Common accordion controls

These are some common patterns for accordions controls. These are included in
the figma library components to make designing common shared uses easier.

#### Revert and save

This action allows users to discard any unsaved changes they've made to the
controls since the last successful save or the default settings were loaded.

```jsx live
const App = () => (
  <Stack direction="horizontal" alignItems="center">
    <Button colorPalette="primary" size="xs" variant="outline">
      <Icons.Undo />
      Revert
    </Button>
    <Button colorPalette="primary" size="xs" variant="solid">
      Save changes
    </Button>
  </Stack>
);
```

#### Cancel and save

Similar to revert and save, there is a difference in persistence and scope.
"Revert" often implies going back to a saved state, affecting changes made over
a longer period, while "Cancel" typically discards unsaved changes from the
current interaction.

```jsx live
const App = () => (
  <Stack direction="horizontal" alignItems="center">
    <Button colorPalette="primary" size="xs" variant="outline">
      Cancel
    </Button>
    <Button colorPalette="primary" size="xs" variant="solid">
      Save
    </Button>
  </Stack>
);
```

#### Four icon buttons

Use up to four icons to allow your user to control the information within the
associated accordion.

```jsx live
const App = () => (
  <Stack direction="horizontal" alignItems="center">
    <IconButton colorPalette="primary" variant="ghost">
      <Icons.Edit />
    </IconButton>
    <IconButton colorPalette="primary" variant="ghost">
      <Icons.PushPin />
    </IconButton>
    <IconButton colorPalette="primary" variant="ghost">
      <Icons.FileCopy />
    </IconButton>
    <IconButton colorPalette="critical" variant="ghost">
      <Icons.Delete />
    </IconButton>
  </Stack>
);
```

#### Delete

Single icon button, this is a common used control, used to delete the entire
accordion and content.

```jsx live
const App = () => (
  <IconButton colorPalette="critical" variant="ghost">
    <Icons.Delete />
  </IconButton>
);
```

#### Switch

The use of switch often signals the effect of the settings being applied, or
activating the content within to be edited by the user.

```jsx live
const App = () => <Switch size="md">Inactive</Switch>;
```

#### Select input

Allowing the users to select from a list. This control often controls the type
of form that will be shown for user completion.

```jsx live
const App = () => (
  <Select.Root
    aria-label="Select an option"
    data-testid="select"
    selectedKey="1"
  >
    <Select.Options>
      <Select.Option id="1">Input value</Select.Option>
    </Select.Options>
  </Select.Root>
);
```

#### Custom controls

If these more typical controls do not suit the needs of your feature, create
controls that best suit your needs.

### Usage

Accordions are a versatile component that can effectively manage and present
information in a concise and organized way, improving usability and saving
screen space.

> [!TIP]\
> When to use

- **Lengthy content:** Use an accordion when you have a significant amount of
  content to display.
- **Logically grouped content:** Use an accordion when the content can be
  logically grouped under headings.
- **Limited vertical space:** Use an accordion when vertical space is limited.
- **Non-essential information:** Use an accordion when users don't need to see
  all the information at once.
- **Simplifying complex forms/settings:** Use an accordion when you want to
  simplify complex forms or settings.

> [!CAUTION]\
> When not to use

- **Brief content:** Do not use an accordion when content is brief and easily
  fits on the screen. Hiding small amounts of information behind clicks is
  unnecessary.
- **Deeply nested content:** Do not use an accordion if the content is deeply
  nested. Multiple layers of accordions can become confusing and difficult to
  navigate.
- **Outgrowing component:** Do not use an accordion when controls and content
  are outgrowing the component. If the content becomes too complex, consider an
  alternative UI component.

### Accordion nesting

Avoid nesting accordions because they hide information and make it hard for
users to predict the content of deeper nested components. Instead, create
simple, well-organized pages that use accordions to hold information in
meaningful ways for the user.

> [!TIP]\
> **Do**
>
> - Do organize the right information within accordions, if all closed, users
>   should be able to navigate information easily.

```jsx live
const App = () => (
  <Accordion.Root size="sm">
    <Accordion.Item>
      <Accordion.Header>General information</Accordion.Header>
      <Accordion.Content>
        <Stack gap="400">
          <FormField.Root>
            <FormField.Label>Product name</FormField.Label>
            <FormField.Input>
              <TextInput size="sm" value="Cozy coffee mug" w="100%" />
            </FormField.Input>
          </FormField.Root>
          <FormField.Root>
            <FormField.Label>Product description</FormField.Label>
            <FormField.Input>
              <TextInput
                size="sm"
                value="A 15oz hand-thrown mug made by artisans."
                w="100%"
              />
            </FormField.Input>
          </FormField.Root>
          <FormField.Root>
            <FormField.Label>Product key</FormField.Label>
            <FormField.Input>
              <TextInput size="sm" value="Cozy-coffee-mug" w="100%" />
            </FormField.Input>
          </FormField.Root>
        </Stack>
      </Accordion.Content>
    </Accordion.Item>
  </Accordion.Root>
);
```

> [!CAUTION]\
> **Don't**
>
> - Do not nest accordions excessively.
> - Do not use accordion controls when nested to reduce complexity.

### Accordion controls

Make it easy for users to understand what the attached controls do, and how to
interact with them when needed.

> [!TIP]\
> **Do**
>
> - Use tools that the user expects to use in the area to complete their tasks.
>   There should be no confusion on what the tools are for and what their
>   effects should be.
> - Include tooltips to describe icon buttons when used.

    <Accordion.Item>
      <Accordion.Header>
        General information
        <Accordion.HeaderRightContent>

### Content

Whether content is custom or using typical forms, think of basic hierarchy to
keep content usable and easy to understand.

> [!TIP]\
> **Do**
>
> - Use left alignment and consistent styling for any content within accordions.
> - Combine like topics and content together well to help the user understand
>   what is expected within each accordion.

```jsx live
const App = () => (
  <Accordion.Root size="sm" w="3/4">
    <Accordion.Item>
      <Accordion.Header>Review the B2C sample data feature</Accordion.Header>
      <Accordion.Content>
        <Stack gap="400">
          <Image
            src="/images/accordion/sample-data.png"
            alt="B2C sample data"
            width="100%"
          />
          <Text fontWeight="700">Sample data imported</Text>

            <Icons.RemoveRedEye />
            Let's get started
          </Button>
        </Stack>
      </Accordion.Content>
    </Accordion.Item>
  </Accordion.Root>
);
```

> [!CAUTION]\
> **Don't**
>
> - Do not use anything other than left alignment of text and forms when
>   possible.
> - Do not overstuff accordions with unrelated information or odds and ends.
>   Accordions are not junk drawers.
> - Do not use controls that are undefined and confusing to use.
