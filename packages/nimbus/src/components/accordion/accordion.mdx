## Overview

The primary difference between accordion types lies in whether they use
additional controls for accessing information or offer a simple click-to-expand
interaction.

### Resources

Deep dive into implementation details and access the Nimbus design library.

## Variables

Get familiar with the features.

### Size

#### Medium

The default size for forms.

```jsx live
const App = () => (
  <Accordion.Root>
    <Accordion.Item>
      <Accordion.Header>
        Headline
        <Accordion.HeaderRightContent>
          <IconButton colorPalette="primary" variant="ghost">
            <Icons.Edit />
          </IconButton>
          <IconButton colorPalette="primary" variant="ghost">
            <Icons.PushPin />
          </IconButton>
          <IconButton colorPalette="primary" variant="ghost">
            <Icons.FileCopy />
          </IconButton>
          <IconButton colorPalette="critical" variant="ghost">
            <Icons.Delete />
          </IconButton>
        </Accordion.HeaderRightContent>
      </Accordion.Header>
      <Accordion.Content>Content goes here</Accordion.Content>
    </Accordion.Item>
  </Accordion.Root>
);
```

#### Small

Use for small condensed areas, or if there is a user preference for more
condensed forms.

```jsx live
const App = () => (
  <Accordion.Root size="sm">
    <Accordion.Item>
      <Accordion.Header>
        Headline
        <Accordion.HeaderRightContent >
          <Stack direction="horizontal" alignItems="center">
          <IconButton colorPalette="primary" variant="ghost" size="sm">
            <Icons.Edit />
          </IconButton>
          <IconButton colorPalette="primary" variant="ghost" size="sm">
            <Icons.PushPin />
          </IconButton>
          <IconButton colorPalette="primary" variant="ghost" size="sm">
            <Icons.FileCopy />
          </IconButton>
          <IconButton colorPalette="critical" variant="ghost" size="sm">
            <Icons.Delete />
          </IconButton>
          </Stack>
        </Accordion.HeaderRightContent>
      </Accordion.Header>
      <Accordion.Content>Content goes here</Accordion.Content>
    </Accordion.Item>
  </Accordion.Root>
);
```

### Content

The main use for accordions is for forms and filters. Custom content can be
created for the body of the accordion.

```jsx live
const App = () => (
  <Accordion.Root size="sm">
    <Accordion.Item>
      <Accordion.Header>
        Details
        <Accordion.HeaderRightContent>
          <Stack direction="horizontal" alignItems="center">
            <Button colorPalette="primary" size="xs" variant="outline">
              <Icons.Undo />
              Revert
            </Button>
            <Button colorPalette="primary" size="xs" variant="solid">
              Save changes
            </Button>
          </Stack>
        </Accordion.HeaderRightContent>
      </Accordion.Header>
      <Accordion.Content>
        <Stack direction="vertical" gap="200">
          <FormField.Root>
            <FormField.Label>First name</FormField.Label>
            <FormField.Input>
              <TextInput size="sm" value="john" w="100%"/>
            </FormField.Input>
          </FormField.Root>
          <FormField.Root>
            <FormField.Label>Last name</FormField.Label>
            <FormField.Input>
              <TextInput size="sm" value="doe" w="100%"/>
            </FormField.Input>
          </FormField.Root>
          <FormField.Root>
            <FormField.Label>Birthday</FormField.Label>
            <FormField.Input>
              <TextInput size="sm" value="10.05.1982" w="100%"/>
            </FormField.Input>
          </FormField.Root>
        </Stack>
      </Accordion.Content>
    </Accordion.Item>
  </Accordion.Root>
);
```
