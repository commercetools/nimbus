# DateInput

The DateInput component allows users to enter and select a date value using a segmented input field. It provides internationalization support, validation, and accessibility features.

## Basic Usage

Use the DateInput component to capture date values from users.

```jsx-live
const App = () => <DateInput aria-label="Enter a date" />
```

### Controlled vs Uncontrolled

The DateInput component can be used in both controlled and uncontrolled modes.

```jsx-live
const App = () => {
  // Uncontrolled with default value
  const [date, setDate] = useState(new CalendarDate(2024, 6, 15));
  
  // Handler for controlled input
  const handleDateChange = (value) => {
    if (value) {
      setDate(value);
    }
  };
  
  return (
    <Stack direction="column" gap="400" alignItems="start">
      <Text>Uncontrolled with defaultValue</Text>
      <DateInput defaultValue={new CalendarDate(2024, 1, 15)} aria-label="Uncontrolled input" />
      
      <Text>Controlled input (current value: {date.toString()})</Text>
      <DateInput 
        value={date} 
        onChange={handleDateChange} 
        aria-label="Controlled input" 
      />
      
      <Button onPress={() => setDate(new CalendarDate(2024, 6, 15))}>Reset to 2024-06-15</Button>
    </Stack>
  );
}
```

### Granularity

Control which date segments are displayed using the `granularity` prop.

```jsx-live
const App = () => {
  return (
    <Stack direction="column" gap="400" alignItems="start">
      <Text>Day granularity (default)</Text>
      <DateInput defaultValue={new CalendarDate(2024, 6, 15)} granularity="day" aria-label="Day granularity" />
      
      <Text>Month granularity</Text>
      <DateInput defaultValue={new CalendarDate(2024, 6, 15)} granularity="month" aria-label="Month granularity" />
      
      <Text>Year granularity</Text>
      <DateInput defaultValue={new CalendarDate(2024, 6, 15)} granularity="year" aria-label="Year granularity" />
    </Stack>
  );
}
```

### Validation

Use `minValue` and `maxValue` to restrict input to a specific date range.

```jsx-live
const App = () => {
  return (
    <Stack direction="column" gap="400" alignItems="start">
      <Text>Date must be between 2024-01-01 and 2024-12-31</Text>
      <DateInput 
        minValue={new CalendarDate(2024, 1, 1)} 
        maxValue={new CalendarDate(2024, 12, 31)} 
        defaultValue={new CalendarDate(2024, 6, 15)} 
        aria-label="Date with validation" 
      />
    </Stack>
  );
}
```

### With FormField

DateInput integrates seamlessly with the FormField component.

```jsx-live
const App = () => {
  const [date, setDate] = useState(new CalendarDate(2024, 6, 15));
  const [isInvalid, setIsInvalid] = useState(false);

  const handleDateChange = (value) => {
    setDate(value);
    
    // Validate date - for this example we'll consider dates before 2024 invalid
    if (value) {
      setIsInvalid(value.year < 2024);
    } else {
      setIsInvalid(true); // Empty value is invalid
    }
  };

  return (
    <FormField.Root isInvalid={isInvalid} isRequired>
      <FormField.Label>Event Date</FormField.Label>
      <FormField.Input>
        <DateInput 
          value={date} 
          onChange={handleDateChange} 
          aria-label="Event date"
        />
      </FormField.Input>
      <FormField.Description>
        Select the date for your event (must be in 2024 or later)
      </FormField.Description>
      <FormField.Error>
        Please select a valid date in 2024 or later
      </FormField.Error>
    </FormField.Root>
  );
}
```

## API Reference

### DateInput Props

The DateInput component accepts all standard date field props from React Aria, plus additional styling props.

#### Core Props

- `value?: DateValue | null` - The current date value (controlled)
- `defaultValue?: DateValue | null` - The default date value (uncontrolled)
- `onChange?: (value: DateValue | null) => void` - Handler called when the date changes
- `minValue?: DateValue` - The minimum allowed date
- `maxValue?: DateValue` - The maximum allowed date
- `granularity?: 'day' | 'month' | 'year'` - Which date segments to display

#### State Props

- `isDisabled?: boolean` - Whether the input is disabled
- `isReadOnly?: boolean` - Whether the input is read-only
- `isRequired?: boolean` - Whether the input is required
- `isInvalid?: boolean` - Whether the input has a validation error

#### Styling Props

- `size?: 'sm' | 'md'` - The size of the input (default: 'md')
- `variant?: 'solid' | 'ghost'` - The visual style variant (default: 'solid')

#### Accessibility Props

- `aria-label?: string` - Accessible label for the input
- `aria-labelledby?: string` - ID of element that labels the input
- `aria-describedby?: string` - ID of element that describes the input

### Usage Notes

- Always provide an accessible label using `aria-label` or `aria-labelledby`
- Use with FormField component for complete form integration
- The component automatically handles internationalization based on the user's locale
- Date values use the `@internationalized/date` library for proper date handling
