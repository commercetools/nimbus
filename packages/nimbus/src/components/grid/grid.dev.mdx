## Getting started

### Import

```tsx
import { Grid, type GridProps } from "@commercetools/nimbus";
```

### Basic usage

```jsx live-dev
const App = () => (
  <Grid gap="100">
    <Grid.Item>
      <Box p="400" bg="neutral.7">
        Item 1
      </Box>
    </Grid.Item>
    <Grid.Item>
      <Box p="400" bg="neutral.7">
        Item 2
      </Box>
    </Grid.Item>
    <Grid.Item>
      <Box p="400" bg="neutral.7">
        Item 3
      </Box>
    </Grid.Item>
  </Grid>
);
```

## Usage examples

### Column-based layout

Use the `templateColumns` prop to define the number and size of columns in your grid.

```jsx live-dev

      <Box p="400" bg="neutral.7">
        Item 1

    </Grid.Item>
    <Grid.Item>

    <Grid.Item>
      <Box p="400" bg="neutral.7">
        Item 4

```

### Row and column spanning

Grid items can span multiple rows or columns using the `rowSpan` and `colSpan` props on `Grid.Item`.

```jsx live-dev
const App = () => (
  <Grid
    h="200px"
    templateRows="repeat(2, 1fr)"
    templateColumns="repeat(5, 1fr)"
    gap="100"
  >
    <Grid.Item rowSpan={2} colSpan={1}>
      <Box p="400" bg="primary.7" h="full">
        Sidebar
      </Box>
    </Grid.Item>
    <Grid.Item colSpan={2}>
      <Box p="400" bg="neutral.7" h="full">
        Top Left
      </Box>
    </Grid.Item>
    <Grid.Item colSpan={2}>
      <Box p="400" bg="neutral.7" h="full">
        Top Right
      </Box>
    </Grid.Item>
    <Grid.Item colSpan={4}>
      <Box p="400" bg="neutral.7" h="full">
        Bottom Section
      </Box>
    </Grid.Item>
  </Grid>
);
```

### Template areas

Use named grid areas to create semantic layouts. This approach is particularly useful for common page layouts.

```jsx live-dev
const App = () => (
  <Grid
    h="400px"
    templateAreas={`
      "header header"
      "nav main"
      "footer footer"
    `}
    templateColumns="200px 1fr"
    gap="400"
  >
    <Grid.Item gridArea="header">
      <Box p="400" bg="primary.7" h="full">
        Header
      </Box>
    </Grid.Item>
    <Grid.Item gridArea="nav">
```

## Component requirements

### Usage context

**Grid vs SimpleGrid**: Use Grid when you need fine-grained control over layout with template areas, custom spanning, or complex grid configurations. Use SimpleGrid for simpler, uniform grid layouts where items have equal size.

**Grid.Item wrapper**: Always wrap child elements in `Grid.Item` components to ensure proper grid behavior and enable spanning props.

## Accessibility

**Role**: Grid renders as a `<div>` with `display: grid`. It has no implicit ARIA role.

**Labeling**: For grids that represent data tables or require context, use `aria-label` or `aria-labelledby`:

```tsx
<Grid aria-label="Product catalog">
  {/* Grid items */}
</Grid>
```

**Keyboard navigation**: Grid itself doesn't provide keyboard navigation. If your grid contains interactive elements, ensure they maintain logical tab order.

### Persistent ID

If your use case requires tracking and analytics for this component, it is good practice to add a **persistent**, **unique** id to the component:

```tsx
const PERSISTENT_ID = "product-grid";

export const ProductGrid = () => (
  <Grid id={PERSISTENT_ID} templateColumns="repeat(3, 1fr)" gap="400">
    <Grid.Item>
      <ProductCard />
    </Grid.Item>
    {/* More items */}
  </Grid>
);
```
