## Getting started

### Import

```tsx
import { DetailPage, type DetailPageProps } from '@commercetools/nimbus';
```

### Basic usage

The DetailPage component uses a compound pattern to structure detail views. Wrap all parts within `DetailPage.Root`.

```jsx live-dev
const App = () => (
  <DetailPage.Root>
    <DetailPage.Header>
      <DetailPage.BackLink href="/products">Back to products</DetailPage.BackLink>
      <DetailPage.Title>Product Details</DetailPage.Title>
      <DetailPage.Subtitle>SKU-12345</DetailPage.Subtitle>
    </DetailPage.Header>
    <DetailPage.Content>
      <Text>Product information goes here.</Text>
    </DetailPage.Content>
  </DetailPage.Root>
)
```

## Usage examples

### Info detail page (read-only)

For pages that display information without form actions, omit the `DetailPage.Footer`.

```jsx live-dev
const App = () => (
  <DetailPage.Root>
    <DetailPage.Header>
      <DetailPage.BackLink href="/products">Back to products</DetailPage.BackLink>
      <DetailPage.Title>Product Details</DetailPage.Title>
      <DetailPage.Subtitle>SKU-12345</DetailPage.Subtitle>
    </DetailPage.Header>
    <DetailPage.Content>
      <Stack gap="400">
        <Text>This is a read-only detail page with product information.</Text>
      </Stack>
    </DetailPage.Content>
  </DetailPage.Root>
)
```

### Form detail page (with footer)

For pages with form actions, add a `DetailPage.Footer` containing action buttons.

```jsx live-dev
const App = () => (
  <DetailPage.Root>
    <DetailPage.Header>
      <DetailPage.BackLink href="/orders">Back to orders</DetailPage.BackLink>
      <DetailPage.Title>Edit Order</DetailPage.Title>
      <DetailPage.Subtitle>Order #ORD-2024-001</DetailPage.Subtitle>
    </DetailPage.Header>
    <DetailPage.Content>
      <Stack gap="400">
        <Text>Form fields would go here.</Text>
      </Stack>
    </DetailPage.Content>
    <DetailPage.Footer>
      <Stack direction="row" gap="200">
        <Button>Save</Button>
        <Button variant="ghost">Cancel</Button>
      </Stack>
    </DetailPage.Footer>
  </DetailPage.Root>
)
```

### Detail page with tab navigation

For pages with route-based tab navigation, compose `TabNav` inside `DetailPage.Header`. Unlike `Tabs` (which switches content panels), `TabNav` renders navigation links — in a real app, the router handles content rendering.

```jsx live-dev
const App = () => (
  <DetailPage.Root>
    <DetailPage.Header paddingBottom="0">
      <DetailPage.BackLink href="/customers">Back to customers</DetailPage.BackLink>
      <DetailPage.Title>Customer Details</DetailPage.Title>
      <DetailPage.Subtitle>customer@example.com</DetailPage.Subtitle>
      <DetailPage.TabNav>
        <TabNav.Root aria-label="Customer sections">
          <TabNav.Item href="/customers/123/general" isCurrent>General</TabNav.Item>
          <TabNav.Item href="/customers/123/addresses">Addresses</TabNav.Item>
          <TabNav.Item href="/customers/123/orders">Orders</TabNav.Item>
        </TabNav.Root>
      </DetailPage.TabNav>
    </DetailPage.Header>
    <DetailPage.Content>
      <Box py="400">
        <Text>General information content</Text>
      </Box>
    </DetailPage.Content>
  </DetailPage.Root>
)
```

## Component parts

| Part | Description |
|------|-------------|
| `DetailPage.Root` | Page container providing layout context |
| `DetailPage.Header` | Header section for back link, title, and subtitle |
| `DetailPage.BackLink` | Back navigation link (requires `href` prop) |
| `DetailPage.Title` | Page title (renders as `h1`) |
| `DetailPage.Subtitle` | Optional subtitle text |
| `DetailPage.Content` | Main content area |
| `DetailPage.HeaderActions` | Optional action buttons alongside the title |
| `DetailPage.TabNav` | Tab-styled navigation for route-based page sections (place in Header) |
| `DetailPage.Footer` | Optional footer for form actions |

## Accessibility

The DetailPage component follows WCAG 2.1 AA guidelines by using semantic HTML landmarks and proper heading hierarchy.

#### Landmark roles

- `DetailPage.Header` uses the `<header>` landmark element
- `DetailPage.Content` uses the `<main>` landmark element
- `DetailPage.Footer` uses the `<footer>` landmark (`contentinfo`) element
- `DetailPage.Title` renders as an `<h1>` element, ensuring proper heading hierarchy
- `DetailPage.BackLink` renders as a semantic `<a>` element with an `aria-label` defaulting to "Go back"

#### Labeling

- `DetailPage.BackLink` receives a default `aria-label` of "Go back" via i18n. Override with a custom `aria-label` when the destination context is important (e.g. `aria-label="Back to products"`).
- When composing with `TabNav`, provide an `aria-label` on `TabNav.Root` to describe the navigation group for screen readers.

#### Keyboard navigation

| Key | Action |
|-----|--------|
| `Tab` | Move focus between interactive elements (back link, tabs, content, footer actions) |
| `Shift + Tab` | Move focus in reverse order |
| `Enter` / `Space` | Activate the focused link or button |

#### Screen reader behavior

| Element | Role | Announced as |
|---------|------|--------------|
| `DetailPage.Header` | `banner` | "banner" landmark |
| `DetailPage.BackLink` | `link` | Link with aria-label (e.g. "Go back" or custom label) |
| `DetailPage.Title` | `heading` (level 1) | "Product Details, heading level 1" |
| `DetailPage.Content` | `main` | "main" landmark |
| `DetailPage.Footer` | `contentinfo` | "contentinfo" landmark |
