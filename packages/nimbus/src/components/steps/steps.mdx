# Steps

A display-only progress indicator that visually communicates the user's position
within a multi-step process such as forms, wizards, and onboarding flows.

## Overview

The Steps component helps users understand where they are in a multi-step
process. It provides clear visual feedback about completed, current, and
upcoming steps, making complex workflows feel more manageable and predictable.

### Key Features

- **Three size variants**: Extra small (xs), small (sm), and medium (md) to fit
  different interface densities
- **Two orientations**: Horizontal for wide layouts and vertical for sidebar or
  narrow contexts
- **Flexible indicators**: Numeric step numbers with optional checkmarks, or
  custom icons for semantic meaning
- **Display-only**: Focuses purely on visual representation; navigation is
  handled by the parent component

### Step states

Each step visually indicates its state based on the current progress.

```jsx-live
const App = () => {
  const [currentStep, setCurrentStep] = React.useState(1);

  return (
    <Stack direction="column" gap="600">
      <Steps.Root step={currentStep} count={4}>
        <Steps.List>
          <Steps.Item index={0}>
            <Steps.Indicator type="numeric" />
            <Steps.Content>
              <Steps.Label>Step 1</Steps.Label>
            </Steps.Content>
          </Steps.Item>
          <Steps.Separator />
          <Steps.Item index={1}>
            <Steps.Indicator type="numeric" />
            <Steps.Content>
              <Steps.Label>Step 2</Steps.Label>
            </Steps.Content>
          </Steps.Item>
          <Steps.Separator />
          <Steps.Item index={2}>
            <Steps.Indicator type="numeric" />
            <Steps.Content>
              <Steps.Label>Step 3</Steps.Label>
            </Steps.Content>
          </Steps.Item>
          <Steps.Separator />
          <Steps.Item index={3}>
            <Steps.Indicator type="numeric" />
            <Steps.Content>
              <Steps.Label>Step 4</Steps.Label>
            </Steps.Content>
          </Steps.Item>
        </Steps.List>
      </Steps.Root>

      <Stack direction="row" gap="400">
        <Button
          variant="outline"
          size="sm"
          onClick={() => setCurrentStep(Math.max(0, currentStep - 1))}
          isDisabled={currentStep === 0}
        >
          Previous
        </Button>
        <Button
          variant="solid"
          size="sm"
          onClick={() => setCurrentStep(Math.min(4, currentStep + 1))}
          isDisabled={currentStep === 4}
        >
          Next
        </Button>
      </Stack>
    </Stack>
  );
}
```

## Guidelines

Use the Steps component strategically to enhance user workflow.

### Best practices

- **Keep labels concise**: Use short, clear step labels that describe the action
  or content
- **Limit step count**: Avoid more than 5-7 steps to prevent overwhelming users
- **Provide descriptions**: Use descriptions for complex steps that need
  additional context
- **Maintain consistency**: Use the same indicator type throughout a flow

> [!TIP]\
> When to use
>
> - **Multi-step forms**: Registration, checkout, surveys
> - **Wizards**: Setup flows, configuration, onboarding
> - **Progress tracking**: Task completion, status indicators

> [!CAUTION]\
> When not to use
>
> - **Linear navigation**: Use tabs or pagination instead
> - **Single actions**: Steps are meant for multi-step processes
> - **Unknown step count**: Use progress bars for indeterminate progress

## Accessibility

Accessibility ensures that digital content and functionality are usable by
everyone, including people with disabilities, by addressing visual, auditory,
cognitive, and physical limitations.

### Accessibility standards

- **Semantic structure**: Uses list semantics (`role="list"` and
  `role="listitem"`) for proper screen reader navigation
- **Current step indication**: Uses `aria-current="step"` to announce the
  active step
- **Step labeling**: Each step includes an accessible label describing its
  position and state (e.g., "Step 1 of 3: complete")
- **Decorative elements hidden**: Separators and indicators are marked as
  decorative (`aria-hidden="true"`) since their meaning is communicated through
  accessible labels
- **Color independence**: State is communicated through both color AND visual
  indicators (checkmarks, styling) to support users with color vision deficiency
