## Getting started

### Import

Import the Steps compound component and its TypeScript types:

```tsx
import { Steps, type StepsRootProps } from "@commercetools/nimbus";
```

### Basic usage

The Steps component is a compound component with multiple parts that work together. At
minimum, you need `Steps.Root`, `Steps.List`, `Steps.Item`, `Steps.Trigger`, and `Steps.Indicator`:

```jsx-live-dev
const App = () => {
  return (
    <Steps.Root defaultStep={1} count={3}>
      <Steps.List>
        <Steps.Item index={0}>
          <Steps.Trigger>
            <Steps.Indicator>
              <Steps.Status
                complete={<Icons.Check />}
                incomplete={<Steps.Number />}
              />
            </Steps.Indicator>
            <Steps.Title>Account</Steps.Title>
          </Steps.Trigger>
          <Steps.Separator />
        </Steps.Item>

        <Steps.Item index={1}>
          <Steps.Trigger>
            <Steps.Indicator>
              <Steps.Status
                complete={<Icons.Check />}
                incomplete={<Steps.Number />}
              />
            </Steps.Indicator>
            <Steps.Title>Profile</Steps.Title>
          </Steps.Trigger>
          <Steps.Separator />
        </Steps.Item>

        <Steps.Item index={2}>
          <Steps.Trigger>
            <Steps.Indicator>
              <Steps.Status
                complete={<Icons.Check />}
                incomplete={<Steps.Number />}
              />
            </Steps.Indicator>
            <Steps.Title>Review</Steps.Title>
          </Steps.Trigger>
        </Steps.Item>
      </Steps.List>
    </Steps.Root>
  );
};
```

## Usage examples

### Controlled vs Uncontrolled

The Steps component supports both controlled and uncontrolled usage patterns:

**Uncontrolled** - The component manages state internally using `defaultStep`:

```tsx
<Steps.Root defaultStep={0} count={3}>...</Steps.Root>
```

**Controlled** - You manage state externally using `step` and `onStepChange`:

```jsx-live-dev
const App = () => {
  const [currentStep, setCurrentStep] = React.useState(0);

  return (
    <Stack direction="column" gap="600">
      <Steps.Root
        step={currentStep}
        onStepChange={(details) => setCurrentStep(details.step)}
        count={3}
      >
        <Steps.List>
          <Steps.Item index={0}>
            <Steps.Trigger>
              <Steps.Indicator>
                <Steps.Status
                  complete={<Icons.Check />}
                  incomplete={<Steps.Number />}
                />
              </Steps.Indicator>
              <Steps.Title>Account</Steps.Title>
            </Steps.Trigger>
            <Steps.Separator />
          </Steps.Item>

          <Steps.Item index={1}>
            <Steps.Trigger>
              <Steps.Indicator>
                <Steps.Status
                  complete={<Icons.Check />}
                  incomplete={<Steps.Number />}
                />
              </Steps.Indicator>
              <Steps.Title>Profile</Steps.Title>
            </Steps.Trigger>
            <Steps.Separator />
          </Steps.Item>

          <Steps.Item index={2}>
            <Steps.Trigger>
              <Steps.Indicator>
                <Steps.Status
                  complete={<Icons.Check />}
                  incomplete={<Steps.Number />}
                />
              </Steps.Indicator>
              <Steps.Title>Review</Steps.Title>
            </Steps.Trigger>
          </Steps.Item>
        </Steps.List>
      </Steps.Root>

      <Text color="neutral.11" fontSize="350">
        Current step: {currentStep}
      </Text>
    </Stack>
  );
};
```

## Component structure

The Steps component is composed of the following parts:

| Component | Description |
|-----------|-------------|
| `Steps.Root` | Container that manages step state and provides context |
| `Steps.List` | Container for grouping step items |
| `Steps.Item` | Container for a single step (trigger + separator) |
| `Steps.Trigger` | Clickable element within each step for direct navigation |
| `Steps.Indicator` | Visual marker showing step status |
| `Steps.Number` | Displays the step number (1-indexed) |
| `Steps.Status` | Renders different content based on step state |
| `Steps.Title` | Displays the step title |
| `Steps.Description` | Displays optional hint text |
| `Steps.Separator` | Visual line connecting step indicators |
| `Steps.Content` | Content container that auto-shows/hides based on current step |
| `Steps.CompletedContent` | Content displayed when all steps are complete |
| `Steps.PrevTrigger` | Navigation button to go to previous step |
| `Steps.NextTrigger` | Navigation button to go to next step |
| `Steps.Context` | Render prop for accessing step state |
| `Steps.ItemContext` | Render prop for accessing current item state |

## Accessibility

The Steps component implements accessibility features following WAI-ARIA patterns:

**Roles:**

- `Steps.List` has `role="tablist"`
- Each `Steps.Trigger` has `role="tab"`
- `Steps.Content` panels have `role="tabpanel"`

**State communication:**

- Current step trigger has `aria-selected="true"`
- Triggers reference their content panels via `aria-controls`
- Orientation is communicated via `aria-orientation`

**Keyboard navigation:**

- Tab moves focus between triggers
- Click or Enter activates a trigger (unless `linear` mode restricts it)
- Arrow keys can navigate between triggers

If your use case requires tracking and analytics for this component, it is good practice to add a **persistent**, **unique** id to the component:

```tsx
const PERSISTENT_ID = "checkout-progress-steps";

export const CheckoutProgress = () => (
  <Steps.Root id={PERSISTENT_ID} defaultStep={0} count={4}>
    <Steps.List>
      {/* ... */}
    </Steps.List>
  </Steps.Root>
);
```
