## Getting started

### Import

```tsx
import { MainPage, type MainPageProps } from '@commercetools/nimbus';
```

### Basic usage

MainPage provides a page skeleton with header, content, and optional footer. At minimum, you need Root, Header with Title, and Content.

```jsx live-dev
const App = () => (
  <MainPage.Root>
    <MainPage.Header>
      <MainPage.Title>Page Title</MainPage.Title>
    </MainPage.Header>
    <MainPage.Content>
      <Text>Your page content goes here.</Text>
    </MainPage.Content>
  </MainPage.Root>
)
```

## Sub-components

### MainPage.Header

The header section uses a two-column grid layout. Title, Subtitle, and any custom content flow into the left column (column 1). Actions spans all rows on the right (column 2) and collapses when not present.

```jsx live-dev
const App = () => (
  <MainPage.Root>
    <MainPage.Header>
      <MainPage.Title>Title on the left</MainPage.Title>
      <MainPage.Actions>
        <Button variant="ghost">Secondary</Button>
        <Button>Primary</Button>
      </MainPage.Actions>
    </MainPage.Header>
    <MainPage.Content>
      <Text>Content area</Text>
    </MainPage.Content>
  </MainPage.Root>
)
```

#### MainPage.Title

Renders a styled `<h1>` element pinned to grid column 1. Accepts children for the title text.

```jsx live-dev
const App = () => (
  <MainPage.Root>
    <MainPage.Header>
      <MainPage.Title>Products</MainPage.Title>
    </MainPage.Header>
    <MainPage.Content>
      <Text>Content area</Text>
    </MainPage.Content>
  </MainPage.Root>
)
```

#### MainPage.Subtitle

Optional subtitle rendered as a `<p>` element pinned to grid column 1, below the title. Place it as a sibling of Title inside the Header.

```jsx live-dev
const App = () => (
  <MainPage.Root>
    <MainPage.Header>
      <MainPage.Title>Products</MainPage.Title>
      <MainPage.Subtitle>Manage your product catalog</MainPage.Subtitle>
    </MainPage.Header>
    <MainPage.Content>
      <Text>Content area</Text>
    </MainPage.Content>
  </MainPage.Root>
)
```

#### MainPage.Actions

A flex container with gap for action buttons. Pinned to grid column 2 (right), spanning all rows.

```jsx live-dev
const App = () => (
  <MainPage.Root>
    <MainPage.Header>
      <MainPage.Title>Products</MainPage.Title>
      <MainPage.Actions>
        <Button variant="ghost">Export</Button>
        <Button>Add Product</Button>
      </MainPage.Actions>
    </MainPage.Header>
    <MainPage.Content>
      <Text>Content area</Text>
    </MainPage.Content>
  </MainPage.Root>
)
```

#### Custom elements in the header

Any additional children placed in the Header default to column 2 (right). To place a custom element on the left side, set `gridColumn="1"`.

```jsx live-dev
const App = () => (
  <MainPage.Root>
    <MainPage.Header>
      <MainPage.Title mb='100'>Page Title</MainPage.Title>
      <Badge 
        gridColumn="1"
        justifySelf="flex-start"
        colorPalette="blue"
        size="xs"
      >
        Custom in column 1
      </Badge>
      <Badge 
        colorPalette="mint"
        justifySelf="flex-end" 
        size='xs'
      >
        Custom in column 2
      </Badge>
    </MainPage.Header>
    <MainPage.Content>
      <Text>Content area</Text>
    </MainPage.Content>
  </MainPage.Root>
)
```

The Header also accepts any content in place of the standard sub-components for full customization.

```jsx live-dev
const App = () => (
  <MainPage.Root>
    <MainPage.Header>
      <Stack direction="row" gap="200" align="center">
        <Heading as="h1">Custom Title</Heading>
        <Badge colorPalette="info">Draft</Badge>
      </Stack>
    </MainPage.Header>
    <MainPage.Content>
      <Text>Content area</Text>
    </MainPage.Content>
  </MainPage.Root>
)
```

### MainPage.Content

The main content area with scrollable overflow. Uses margin-based spacing to preserve sticky positioning for child components like DataTable.

```jsx live-dev
const App = () => (
  <MainPage.Root>
    <MainPage.Header>
      <MainPage.Title>Page Title</MainPage.Title>
    </MainPage.Header>
    <MainPage.Content>
      <Box bg="blue.3" padding="400" borderRadius="200">
        <Text color="blue.12">Content area with automatic margin spacing</Text>
      </Box>
    </MainPage.Content>
  </MainPage.Root>
)
```

### MainPage.Footer

Optional footer for page-level actions. Omit it entirely when the page has no form actions. The footer grid row collapses automatically when absent.

```jsx live-dev
const App = () => (
  <MainPage.Root>
    <MainPage.Header>
      <MainPage.Title>With Footer</MainPage.Title>
    </MainPage.Header>
    <MainPage.Content>
      <Box bg="neutral.3" padding="400" borderRadius="200">
        <Text>Form content</Text>
      </Box>
    </MainPage.Content>
    <MainPage.Footer>
      <Stack direction="row" gap="200" justify="flex-end" width="100%">
        <Button variant="ghost">Cancel</Button>
        <Button>Save</Button>
      </Stack>
    </MainPage.Footer>
  </MainPage.Root>
)
```

## Usage examples

### Info main page (read-only)

For pages that display information without form actions, omit the MainPage.Footer.

```jsx live-dev
const App = () => (
  <MainPage.Root>
    <MainPage.Header borderBottom='none' paddingBottom='0'>
      <MainPage.Title>Products</MainPage.Title>
      <MainPage.Actions>
        <Button variant="ghost">Export</Button>
        <Button>Add Product</Button>
      </MainPage.Actions>
    </MainPage.Header>
    <MainPage.Content>
      <Text>Product list content goes here.</Text>
    </MainPage.Content>
  </MainPage.Root>
)
```

### Form main page (with footer)

For pages with form actions, add a MainPage.Footer containing action buttons.

```jsx live-dev
const App = () => (
  <MainPage.Root>
    <MainPage.Header>
      <MainPage.Title>Project Settings</MainPage.Title>
    </MainPage.Header>
    <MainPage.Content>
      <Stack gap="400">
        <FormField.Root>
          <FormField.Label>Project name</FormField.Label>
          <FormField.Input>
            <TextInput placeholder="e.g. My Store" />
          </FormField.Input>
        </FormField.Root>
        <FormField.Root>
          <FormField.Label>Description</FormField.Label>
          <FormField.Input>
            <TextInput placeholder="Describe the project" />
          </FormField.Input>
        </FormField.Root>
      </Stack>
    </MainPage.Content>
    <MainPage.Footer>
      <Stack direction="row" gap="200" justify="flex-end" width="100%">
        <Button variant="ghost">Cancel</Button>
        <Button>Save</Button>
      </Stack>
    </MainPage.Footer>
  </MainPage.Root>
)
```

### Main page with tab navigation

For pages with route-based tab navigation, compose `TabNav` inside `MainPage.Header`. Unlike `Tabs` (which switches content panels), `TabNav` renders navigation links — in a real app, the router handles content rendering.

```jsx live-dev
const App = () => (
  <MainPage.Root>
    <MainPage.Header paddingBottom="0">
      <MainPage.Title>Product Details</MainPage.Title>
      <MainPage.Actions>
        <Button>Publish</Button>
      </MainPage.Actions>
      <MainPage.TabNav>
        <TabNav.Root aria-label="Product sections">
          <TabNav.Item href="/products/123/general" isCurrent>General</TabNav.Item>
          <TabNav.Item href="/products/123/variants">Variants</TabNav.Item>
          <TabNav.Item href="/products/123/images">Images</TabNav.Item>
        </TabNav.Root>
      </MainPage.TabNav>
    </MainPage.Header>
    <MainPage.Content>
      <Text>General information content</Text>
    </MainPage.Content>
  </MainPage.Root>
)
```

## Component requirements

### Accessibility

MainPage uses semantic HTML landmarks for assistive technology navigation:

- **Header** renders as `<header>` (banner landmark)
- **Title** renders as `<h1>` for the page heading
- **Content** renders as `<main>` (main landmark)
- **Footer** renders as `<footer>` (contentinfo landmark)

Since `<main>` should be unique per page, avoid nesting multiple MainPage components.

### Keyboard navigation

MainPage is a layout skeleton with no interactive behavior of its own. Focus moves naturally through landmark regions via the `Tab` key, allowing screen reader users to jump between header, main content, and footer using landmark navigation shortcuts.

## Migrating from Merchant Center Application Kit

MainPage replaces the four main page components from
[`@commercetools-frontend/application-components`](https://docs.commercetools.com/merchant-center-customizations/tooling-and-configuration/commercetools-frontend-application-components#components-for-main-pages):

| Application Kit Component | Nimbus Equivalent |
|---|---|
| `<InfoMainPage title="...">` | `<MainPage.Root>` + Header/Title + Content (no Footer) |
| `<FormMainPage title="..." onPrimaryButtonClick={...}>` | `<MainPage.Root>` + Header/Title + Content + Footer with buttons |
| `<CustomFormMainPage formControls={...}>` | Same as Form — Footer accepts any children |
| `<TabularMainPage tabControls={...}>` | `<MainPage.Root>` + Header/Title with `<TabNav.Root>` + Content |

Key differences:

- **Composition over configuration** — Instead of 10+ props for button labels, handlers, and visibility, compose your own buttons in Footer or Actions
- **No built-in tab routing** — Use the TabNav component inside the Header for route-based navigation
- **Title and subtitle as children** — `<MainPage.Title>Products</MainPage.Title>` and `<MainPage.Subtitle>...</MainPage.Subtitle>` replace `title` and `subtitle` props on the AppKit page components
- **No `customTitleRow`** — The Header accepts any content, so replace Title with your own elements
- **No `Intl` static property** — Use your own i18n solution for button labels
