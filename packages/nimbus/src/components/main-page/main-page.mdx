# MainPage

A page skeleton compound component that provides the structural layout for
top-level views. Combines a header (with title, subtitle, and actions), a
content area, and an optional footer.

## Overview

MainPage is a compound component that structures top-level views into three
zones: a header, a content area, and an optional footer.

### Key features

- **MainPage.Header** — Two-column grid that positions Title/Subtitle on the left and Actions on the right
- **MainPage.Title** — Styled `<h1>` element accepting children for the page title
- **MainPage.Subtitle** — Optional `<p>` element for secondary descriptive text below the title
- **MainPage.Content** — Scrollable content area with margin-based spacing
- **MainPage.Footer** — Optional action bar for form pages; omit it and the space collapses automatically
- **Composable** — Arrange sub-components freely to build info pages, form pages, or tabular pages from the same primitives

## Variables

Get familiar with the sub-components.

### Header with title, subtitle, and actions

The header uses a two-column grid to position title and subtitle on the left and actions on the right.

```jsx live
const App = () => (
  <MainPage.Root border='solid-25' borderColor='neutral.6' borderRadius="200">
    <MainPage.Header>
      <MainPage.Title>Page Title</MainPage.Title>
      <MainPage.Subtitle>Optional subtitle text</MainPage.Subtitle>
      <MainPage.Actions>
        <Button variant="ghost">Secondary</Button>
        <Button>Primary Action</Button>
      </MainPage.Actions>
    </MainPage.Header>
    <MainPage.Content>
      <Box bg="neutral.3" padding="600" borderRadius="200">
        <Text>Content area</Text>
      </Box>
    </MainPage.Content>
  </MainPage.Root>
)
```

### Content area

The Content area provides scrollable overflow with margin-based spacing that preserves sticky positioning for child components like DataTable.

```jsx live
const App = () => (
  <MainPage.Root border='solid-25' borderColor='blue.6' borderRadius="200">
    <MainPage.Header>
      <MainPage.Title>Content Example</MainPage.Title>
    </MainPage.Header>
    <MainPage.Content>
      <Box bg="blue.3" padding="400" borderRadius="200">
        <Text color="blue.12">Content with automatic margin spacing</Text>
      </Box>
    </MainPage.Content>
  </MainPage.Root>
)
```

### Optional footer

Include Footer for form-like pages with save/cancel actions. Omit it for read-only pages — the grid row collapses automatically.

```jsx live
const App = () => (
  <MainPage.Root border='solid-25' borderColor='neutral.6' borderRadius="200">
    <MainPage.Header>
      <MainPage.Title>With Footer</MainPage.Title>
    </MainPage.Header>
    <MainPage.Content>
      <Box bg="neutral.3" padding="400" borderRadius="200">
        <Text>Form content area</Text>
      </Box>
    </MainPage.Content>
    <MainPage.Footer>
      <Stack direction="row" gap="200" justify="flex-end" width="100%">
        <Button variant="ghost">Cancel</Button>
        <Button>Save</Button>
      </Stack>
    </MainPage.Footer>
  </MainPage.Root>
)
```

## Accessibility

MainPage uses semantic HTML landmarks to provide a clear document structure for
assistive technologies:

- **Header** renders as `<header>` (banner landmark)
- **Title** renders as `<h1>` for the page heading
- **Content** renders as `<main>` (main landmark)
- **Footer** renders as `<footer>` (contentinfo landmark)

These landmarks allow screen reader users to navigate directly between page
sections. Since `<main>` should be unique per page, avoid nesting multiple
MainPage components.
