## Getting started

### Import

```tsx
import { Group, type GroupProps } from '@commercetools/nimbus';
```

### Basic usage

```jsx-live-dev
const App = () => {
  return (
    <Group aria-label="Action buttons">
      <Button>Save</Button>
      <Button>Cancel</Button>
      <Button>Delete</Button>
    </Group>
  );
};
```

## Usage examples

### Grouping with spacing

Use Chakra UI style props like `gap` to control spacing between grouped elements:

```jsx-live-dev
const App = () => {
  return (
    <Stack direction="column" gap="400">
      <Group aria-label="Compact button group" gap="100">
        <Button size="xs">Edit</Button>
        <Button size="xs">Copy</Button>
        <Button size="xs">Delete</Button>
      </Group>

      <Group aria-label="Default button group" gap="300">
        <Button>Edit</Button>
        <Button>Copy</Button>
        <Button>Delete</Button>
      </Group>

        <Button size="2xs">Edit</Button>
        <Button size="2xs">Copy</Button>

```

### With hover handlers

Group provides hover event handlers for tracking user interactions:

```jsx-live-dev
const App = () => {
  const [isHovering, setIsHovering] = useState(false);
  const [hoverCount, setHoverCount] = useState(0);

  return (
    <Stack direction="column" gap="400" alignItems="flex-start">
      <Group
        aria-label="Interactive button group"
        gap="300"
        onHoverStart={() => {
          setIsHovering(true);
          setHoverCount((prev) => prev + 1);
        }}
        onHoverEnd={() => setIsHovering(false)}
        padding="400"
        borderRadius="200"
        backgroundColor={isHovering ? 'primary.2' : 'transparent'}
        transition="background-color 0.2s"
      >
        <Button>Action 1</Button>
        <Button>Action 2</Button>
        <Button>Action 3</Button>
      </Group>

      <Stack direction="column" gap="100">
        <Text fontSize="350">
          Status: {isHovering ? 'Hovering' : 'Not hovering'}
        </Text>
        <Text fontSize="350" color="fg.muted">
          Hover count: {hoverCount}
        </Text>
      </Stack>
    </Stack>
  );
};
```

## Component requirements

### Accessibility

The Group component follows WCAG 2.1 AA guidelines and implements proper semantic grouping:

#### Role

- Automatically applies `role="group"` to identify related elements
- Groups should have a descriptive `aria-label` or `aria-labelledby` to provide context

#### Labeling

Always provide an accessible name for groups:

```tsx
// Using aria-label
<Group aria-label="Text formatting controls">
  {/* controls */}
</Group>

// Using aria-labelledby
<>
  <Heading id="toolbar-heading">Editing Tools</Heading>
  <Group aria-labelledby="toolbar-heading">
    {/* tools */}
  </Group>
</>
```

#### Keyboard Navigation

Group is a semantic container that does not interfere with keyboard navigation. Each child element maintains its own keyboard interaction patterns.

#### Persistent ID

If your use case requires tracking and analytics for this component, it is good practice to add a **persistent**, **unique** id to the component:

```tsx
const PERSISTENT_ID = "primary-action-group";

export const ActionToolbar = () => (
  <Group id={PERSISTENT_ID} aria-label="Primary actions">
    <Button>Save</Button>
    <Button>Cancel</Button>
  </Group>
);
```

## Common patterns

### Button groups

Group related actions together for better visual hierarchy:

```jsx-live-dev
const App = () => {
  return (
    <Stack direction="column" gap="400" alignItems="flex-start">
      <Stack direction="column" gap="200">
        <Text fontWeight="600">Primary Actions</Text>
        <Group aria-label="Primary actions" gap="300">
          <Button variant="solid">Save</Button>
          <Button variant="outline">Save Draft</Button>
          <Button variant="ghost">Cancel</Button>
        </Group>
      </Stack>

      <Stack direction="column" gap="200">
        <Text fontWeight="600">Destructive Actions</Text>
        <Group aria-label="Destructive actions" gap="300">
          <Button colorPalette="critical">Delete</Button>
          <Button variant="outline" colorPalette="critical">
            Archive
          </Button>
        </Group>
      </Stack>

      <Stack direction="column" gap="200">
        <Text fontWeight="600">Icon Button Group</Text>
        <Group aria-label="Text alignment controls" gap="100">
          <IconButton aria-label="Align left">
            <Icons.FormatAlignLeft />
          </IconButton>
          <IconButton aria-label="Align center">
            <Icons.FormatAlignCenter />
          </IconButton>
          <IconButton aria-label="Align right">
            <Icons.FormatAlignRight />
          </IconButton>
          <IconButton aria-label="Align justify">
            <Icons.FormatAlignJustify />
          </IconButton>
        </Group>
      </Stack>
    </Stack>
  );
};
```

### Form field grouping

Use Group to organize related form controls:

```jsx-live-dev
const App = () => {
  const [firstName, setFirstName] = useState('');
  const [lastName, setLastName] = useState('');

  return (
    <Stack direction="column" gap="400" maxWidth="600px">
      <Stack direction="column" gap="200">
        <Text fontWeight="600">Name</Text>
        <Group aria-label="Name fields" gap="300">
          <TextInput
            placeholder="First name"
            value={firstName}
            onChange={setFirstName}
            flex="1"
          />
          <TextInput
            placeholder="Last name"
            value={lastName}
            onChange={setLastName}
            flex="1"
          />
        </Group>
      </Stack>

      <Stack direction="column" gap="200">
        <Text fontWeight="600">Date Range</Text>
        <Group aria-label="Date range fields" gap="300">
          <TextInput placeholder="Start date" flex="1" />
          <Text alignSelf="center">to</Text>
          <TextInput placeholder="End date" flex="1" />
        </Group>
      </Stack>
    </Stack>
  );
};
```
