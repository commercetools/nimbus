## Overview

These specifications define the standard styles and variations for icon buttons
within our design system, covering different visual treatments for diverse use
cases.

### Resources

Deep dive into implementation details and access the Nimbus design library.

[Figma library](https://www.figma.com/design/AvtPX6g7OGGCRvNlatGOIY/NIMBUS-design-system?node-id=346-2570&m=dev)

## Hierarchy

#### Most common: Ghost

Ghost icon buttons are used most often, the represent familiar actions that
don't compete with a primary action on the page.

```jsx live
const App = () => {
  return (
    <IconButton variant="ghost" colorPalette="primary">
      <Icons.Edit/>
    </IconButton>
  );
};
```

#### Use sparingly: Solid

Solid icon buttons are sparingly used, but represent commonly understood primary
actions by the user.

```jsx live
const App = () => {
  return (
    <IconButton variant="solid" colorPalette="primary">
      <Icons.Edit/>
    </IconButton>
  );
};
```

#### Least used: Outline

Outline icon buttons are used as secondary buttons, used often in button groups
to represent icon toggles and within data tables.

```jsx live
const App = () => {
  return (
    <IconButton variant="outline" colorPalette="primary">
      <Icons.Edit/>
    </IconButton>
  );
};
```

## Variables

Get familiar with the features.

### Size

#### Medium (40px)

These are the default and preferred size for icon buttons.

```jsx live
const App = () => {
  return (
    <IconButton variant="solid" colorPalette="primary" size="md">
      <Icons.Edit/>
    </IconButton>
  );
};
```

#### X-Small (32px)

Placed in a small dense content areas, e.g., in pop-over or inside a data table.

```jsx live
const App = () => {
  return (
    <IconButton variant="solid" colorPalette="primary" size="xs">
      <Icons.Edit/>
    </IconButton>
  );
};
```

#### XX-Small (24px)

2XS is used sparingly mostly inside other components e.g, inside inputs or
filter badges.

```jsx live
const App = () => {
  return (
    <IconButton variant="solid" colorPalette="primary" size="2xs">
      <Icons.Edit/>
    </IconButton>
  );
};
```

### Color palette

#### Primary

This is the default color palette for the icon button. This styling should be
used most commonly for all actions.

```jsx live
const App = () => {
  return (
    <IconButton variant="ghost" colorPalette="primary">
      <Icons.ViewColumn/>
    </IconButton>
  );
};
```

#### Critical

Use this icon button when the user is approaching a permanent destructive
action. See guidelines for more information.

```jsx live
const App = () => {
  return (
    <IconButton variant="ghost" colorPalette="critical">
      <Icons.Delete/>
    </IconButton>
  );
};
```

#### Tooltip

For accessibility compliance (and good user experience for all) icon buttons
that do not have a label must have a tooltip. Go to tooltips for more
information about the component.

```jsx live
const App = () => {
  return (
    <Tooltip.Root delay={0} closeDelay={0}>
      <IconButton variant="ghost" colorPalette="primary">
        <Icons.Edit/>
      </IconButton>
      <Tooltip.Content>
        Edit
      </Tooltip.Content>
    </Tooltip.Root>
  );
};
```
