# ToggleButton

A toggle button allows users to switch between two states (selected/unselected) by clicking. It's particularly useful for binary choices like favorites, bookmarks, or feature toggles.

## Overview

Toggle buttons provide clear visual feedback about their current state and allow users to easily switch between on/off states. They combine the familiar interaction patterns of buttons with the state persistence of toggles.

### Basic Usage

#### Controlled Toggle Button

When you need to control the toggle state externally, use the `isSelected` and `onChange` props.

```jsx-live
const App = () => {
  const [isSelected, setIsSelected] = React.useState(false);

  return (
    <Stack direction="column" align="start" gap="400">
      <ToggleButton isSelected={isSelected} onChange={setIsSelected}>
        Controlled Toggle (selected: {isSelected.toString()})
      </ToggleButton>
      <Button onClick={() => setIsSelected(!isSelected)}>
        Toggle externally
      </Button>
    </Stack>
  );
};
```

#### Uncontrolled Toggle Button

For simple cases where you don't need to control the state, use `defaultSelected` to set the initial state.

```jsx-live
const App = () => (
  <ToggleButton defaultSelected={true}>
    Uncontrolled Toggle (starts selected)
  </ToggleButton>
);
```

## Guidelines

Toggle buttons should be used when you need to represent a binary state that users can control. They provide immediate visual feedback and are particularly effective for actions like favorites, bookmarks, or feature toggles.

### Writing Guidelines

- Use clear, action-oriented labels that describe what will be toggled
- Keep labels concise and descriptive
- Use sentence case (capitalize the first word, lowercase for the rest)
- Consider how the label reads in both selected and unselected states

### Preferred Usage Patterns

| **Use Case** | **Example** | **Description** |
| --- | --- | --- |
| **Favorites** | "Favorite", "Like", "Star" | Actions that add/remove items from a collection |
| **Bookmarks** | "Save", "Bookmark", "Remember" | Actions that save content for later |
| **Features** | "Dark Mode", "Notifications", "Auto-save" | Settings that can be enabled/disabled |
| **Visibility** | "Show Details", "Hide Advanced", "Collapse" | Controls that show/hide content |

### Usage

> [!TIP]\
> When to use

- **Binary states:** When users need to switch between two clear states
- **Persistent actions:** When the state should be remembered (favorites, settings)
- **Feature toggles:** For enabling/disabling features or options
- **Collections:** Adding/removing items from lists or collections
- **Visibility controls:** Showing/hiding content or details

> [!CAUTION]\
> When not to use

- **Navigation:** Use links or navigation components instead
- **Form submission:** Use regular buttons for submitting forms
- **Multiple options:** Use radio buttons or select components for multiple choices
- **Temporary actions:** Use regular buttons for one-time actions
- **Complex workflows:** Use multi-step processes or wizards

### Icons in Toggle Buttons

Icons help users quickly understand the toggle action and provide visual context.

> [!TIP]\
> **Do**

- Use icons that clearly represent the action being toggled
- Place icons to the left of text for better readability
- Use consistent icons for similar actions across your application
- Choose icons that work well in both selected and unselected states

```jsx-live
const App = () => (
  <Stack direction="horizontal" gap="400">
    <ToggleButton>
      <Icons.ThumbUp />
      Like
    </ToggleButton>
    <ToggleButton>
      <Icons.Bookmark />
      Save
    </ToggleButton>
    <ToggleButton>
      <Icons.Star />
      Favorite
    </ToggleButton>
  </Stack>
);
```

> [!CAUTION]\
> **Don't**

- Don't use misleading icons that don't match the action
- Avoid placing icons after text unless there's a specific reason
- Don't use multiple icons for a single toggle action
- Avoid icons that are unclear in their selected state

```jsx-live
const App = () => (
  <Stack direction="horizontal" gap="400">
    <ToggleButton>
      <Icons.Delete />
      Like
    </ToggleButton>
    <ToggleButton>
      Save
      <Icons.Bookmark />
    </ToggleButton>
    <ToggleButton>
      <Icons.Add />
      Favorite
      <Icons.Remove />
    </ToggleButton>
  </Stack>
);
```

### Accessibility

The ToggleButton component is built on React Aria Components and provides comprehensive keyboard and screen reader support. It uses proper ARIA attributes to communicate the toggle state to assistive technologies.

#### Keyboard Interaction

- **Tab:** Moves focus to the toggle button
- **Enter/Space:** Toggles the button state
- **Escape:** Moves focus away from the button (when focused)

```jsx-live
const App = () => (
  <ToggleButton onPress={() => alert("Toggled! Try using Enter or Space keys.")}>
    <Icons.Star />
    Keyboard Accessible Toggle
  </ToggleButton>
);
```

### Accessibility Standards

#### 1. Perceivable:

- **Clear State Indication:** The selected state is clearly visible through color, typography, and background changes
- **Sufficient Color Contrast:** All text maintains proper contrast ratios in both selected and unselected states
- **Focus Indicator:** A clear focus ring appears when the toggle button is focused via keyboard

#### 2. Operable:

- **Keyboard Accessible:** Full keyboard support with Tab, Enter, and Space key interactions
- **Target Size:** Minimum 24x24 pixel target size for easy clicking/tapping
- **Touch-Friendly:** Appropriate sizing and spacing for touch interfaces

#### 3. Understandable:

- **Clear Labels:** Labels clearly describe what will be toggled
- **Consistent Behavior:** Toggle behavior is consistent across the application
- **State Communication:** The current state is clearly communicated visually and to screen readers

#### 4. Robust:

- **Screen Reader Support:** Proper ARIA attributes communicate the toggle state and changes
- **Cross-Platform:** Works consistently across different browsers and assistive technologies
- **Semantic HTML:** Uses proper button semantics with toggle state information
