## Overview

`ChatMessage` renders **one message** in an AI chat conversation — an avatar
paired with a message bubble, plus optional actions, a meta row, and a typing
indicator for streaming replies. It does not compose the surrounding feed:
stacking messages, scrolling, and the live region that announces new content are
the transcript's job (see `ChatMessageList`).

The `sender` prop drives which participant the message is from and controls both
the layout direction and the bubble/avatar styling:

- `user` — the human; avatar trails the bubble.
- `agent` (default) — the assistant; avatar leads the bubble.

`sender` denotes only _who_ sent the message. Non-message interjections (system
notices, dividers) are the separate `ChatNotice` component, and tool/function
output is content _inside_ an `agent` message — neither is a `sender` value.

`tone="error"` is independent of `sender` — it tints the bubble to flag a failed
generation, since an agent message can still fail.

## Variables

Get familiar with the features.

### Senders

Each sender has its own layout direction and surface: `user` places the bubble
before the avatar, `agent` places the avatar before the bubble. A `ChatNotice`
(a peer, not a sender) can sit between turns as a centered interjection.

```jsx live
const App = () => (
  <Stack gap="600">
    <ChatMessage.Root sender="user">
      <ChatMessage.Avatar firstName="Ada" lastName="Lovelace" />
      <ChatMessage.Bubble>
        <Text>Can you summarise last week's orders?</Text>
      </ChatMessage.Bubble>
    </ChatMessage.Root>
    <ChatNotice>Conversation history was cleared.</ChatNotice>
    <ChatMessage.Root sender="agent">
      <ChatMessage.Avatar>
        <Icons.AutoAwesome />
      </ChatMessage.Avatar>
      <ChatMessage.Bubble>
        <Text>Revenue is up 12% week-over-week.</Text>
      </ChatMessage.Bubble>
    </ChatMessage.Root>
  </Stack>
);
```

### Tone

`tone="error"` tints the bubble with the critical palette to flag a failed
generation, on top of whatever `sender` styling already applies.

```jsx live
const App = () => (
  <ChatMessage.Root sender="agent" tone="error">
    <ChatMessage.Avatar>
      <Icons.AutoAwesome />
    </ChatMessage.Avatar>
    <ChatMessage.Bubble>
      <Text>Something went wrong generating this response.</Text>
      <ChatMessage.Actions>
        <Button variant="outline" colorPalette="primary">
          Retry
        </Button>
      </ChatMessage.Actions>
    </ChatMessage.Bubble>
  </ChatMessage.Root>
);
```

### Actions

`ChatMessage.Actions` is a right-aligned row of buttons rendered inside the
bubble. Place it last so the actions pin to the bottom of the card.

```jsx live
const App = () => (
  <ChatMessage.Root sender="agent">
    <ChatMessage.Avatar>
      <Icons.AutoAwesome />
    </ChatMessage.Avatar>
    <ChatMessage.Bubble>
      <Text>I drafted the date range for your report. Approve to apply.</Text>
      <ChatMessage.Actions>
        <Button variant="outline" colorPalette="primary">
          Save as draft
        </Button>
        <Button variant="solid" colorPalette="primary">
          Approve
        </Button>
      </ChatMessage.Actions>
    </ChatMessage.Bubble>
  </ChatMessage.Root>
);
```

### Meta

`ChatMessage.Meta` is a `space-between` row rendered below the bubble, for a
timestamp, a trust/provenance link, or reaction icons.

```jsx live
const App = () => (
  <ChatMessage.Root sender="agent">
    <ChatMessage.Avatar>
      <Icons.AutoAwesome />
    </ChatMessage.Avatar>
    <ChatMessage.Bubble>
      <Text>Here is the summary you asked for.</Text>
    </ChatMessage.Bubble>
    <ChatMessage.Meta>
      <Link href="#">How was this generated?</Link>
      <Stack direction="row" alignItems="center" gap="100">
        <Text color="neutral.11">Apr 13, 11:56pm</Text>
        <IconButton aria-label="Good response" variant="ghost" size="xs">
          <Icons.ThumbUp />
        </IconButton>
        <IconButton aria-label="Bad response" variant="ghost" size="xs">
          <Icons.ThumbDown />
        </IconButton>
        <IconButton aria-label="Copy" variant="ghost" size="xs">
          <Icons.ContentCopy />
        </IconButton>
      </Stack>
    </ChatMessage.Meta>
  </ChatMessage.Root>
);
```

### Typing / streaming

`ChatMessage.Typing` shows an animated "generating…" indicator as the bubble's
payload while a reply streams. Pair it with `isStreaming` on the root, which
sets `aria-busy`.

```jsx live
const App = () => (
  <ChatMessage.Root sender="agent" isStreaming>
    <ChatMessage.Avatar>
      <Icons.AutoAwesome />
    </ChatMessage.Avatar>
    <ChatMessage.Bubble>
      <ChatMessage.Typing>
        <Text color="neutral.11">Assistant is typing…</Text>
      </ChatMessage.Typing>
    </ChatMessage.Bubble>
  </ChatMessage.Root>
);
```
