## Accessibility

Accessibility ensures that digital content and functionality are usable by
everyone, including people with disabilities, by addressing visual, auditory,
cognitive, and physical limitations.

```jsx-live
const App = () => {
  const [isOpen, setIsOpen] = useState(false);

  return (
    <>
      <Button onPress={() => setIsOpen(true)}>Open Dialog</Button>
      <Dialog.Root open={isOpen} onOpenChange={setIsOpen}>
        <Dialog.Content>
          <Dialog.Header>
            <Dialog.Title>Dialog Title</Dialog.Title>
          </Dialog.Header>
          <Dialog.Body>
            <Text>Dialog content goes here.</Text>
          </Dialog.Body>
          <Dialog.Footer>
            <Button variant="outline" onPress={() => setIsOpen(false)}>
              Cancel
            </Button>
            <Button variant="solid" onPress={() => setIsOpen(false)}>
              Confirm
            </Button>
          </Dialog.Footer>
        </Dialog.Content>
      </Dialog.Root>
    </>
  );
};
```

### Accessibility standards

- **Keyboard navigation:** Tab moves focus to next focusable element within
  dialog, Shift + Tab moves to previous element, Escape closes the dialog
  (unless disabled), Enter/Space activates focused button or trigger.
- **Role identification:** Dialog has `dialog` role with proper labeling.
- **Focus management:** Focus moves to dialog on open and returns to trigger on
  close.
- **Focus containment:** Tab navigation stays within dialog boundaries.
- **Accessible names:** Title and description properly associate with dialog.
- **State announcements:** Screen readers announce when dialog opens/closes.
- **Info and relationships:** Proper semantic structure with header, body, and
  footer.
- **Contrast:** All text meets minimum contrast requirements.
- **Non-text contrast:** Focus indicators and UI components meet contrast
  standards.
- **No keyboard trap:** Focus contained but escapable via Escape key.
- **Focus order:** Logical focus progression through dialog elements.
- **Focus visible:** Clear focus indicators on all interactive elements.
- **On focus:** No unexpected context changes when focusing elements.
- **On input:** Predictable behavior for all user interactions.
- **Labels or instructions:** Clear labeling and instructions where needed.
- **Name, role, value:** Proper semantic markup and ARIA attributes.
- **Status messages:** Appropriate announcements for state changes.
