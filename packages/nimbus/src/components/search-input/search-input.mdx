# SearchInput

A search input component that allows users to enter search queries with built-in clear functionality.

## Overview

The SearchInput component provides a specialized text input for search operations, featuring a search icon, clear button, and keyboard shortcuts. Built on React Aria's SearchField for accessibility, it handles common search patterns like clear on Escape and submit on Enter.

### Key Features

- **Built-in search icon**: Visual indicator that this field is for searching
- **Clear button**: Appears when text is entered, allows quick clearing of input
- **Keyboard shortcuts**: Escape to clear, Enter to submit
- **Accessible**: WCAG 2.1 AA compliant with proper ARIA attributes

The component supports controlled and uncontrolled modes with event handlers.

```jsx-live

  const [submitted, setSubmitted] = useState("");
```

## Guidelines

SearchInput guidelines focus on creating clear, accessible, and user-friendly search experiences by emphasizing proper placeholder text, clear visual feedback, and keyboard accessibility.

### Best practices

- **Clear placeholder text**: Use concise, descriptive placeholder text that indicates what users can search for (e.g., "Search products...", "Search by name or ID")
- **Visible clear button**: The clear button should appear when text is entered and be easily clickable
- **Keyboard accessibility**: Ensure users can tab to the input, type, press Escape to clear, and Enter to submit
- **Loading states**: Consider showing loading indicators when search is in progress
- **Immediate feedback**: Show search results as users type (debounced) or on submission
- **Error handling**: Display clear error messages when searches fail

### When to use

> [!TIP]\
> When to use

- **Search functionality**: When users need to search through a list, database, or collection
- **Quick filtering**: For filtering large datasets or tables in real-time
- **Navigation**: As a site-wide or section-wide search tool
- **Autocomplete**: When combined with suggestions or autocomplete functionality

> [!CAUTION]\
> When not to use

- **Simple filtering**: If filtering is better handled by dropdowns or checkboxes
- **Form submission**: Don't use for general text inputs that aren't search-related
- **Complex queries**: When users need advanced search features like boolean operators or multiple fields

### Placeholder text

Good placeholder text helps users understand what they can search for and provides examples when appropriate.

> [!TIP]\
> **Do**

- Use action-oriented text: "Search products...", "Find a customer..."
- Provide examples when helpful: "Search by name, SKU, or ID"
- Keep it concise and clear

```jsx-live
const App = () => (
  <SearchInput placeholder="Search products by name or SKU..." />
)
```

> [!CAUTION]\
> **Don't**

- Use vague text like "Search..." without context
- Make placeholder text too long or detailed
- Rely solely on placeholder for instructions (use labels when needed)

```jsx-live
const App = () => (
  <SearchInput placeholder="Type here to search" />
)
```

### Clear button behavior

The clear button appears automatically when text is entered and disappears when the field is empty.

> [!TIP]\
> **Best practices**

- The clear button is always present in the DOM but hidden when empty (prevents layout shift)
- Clicking the clear button clears the input and triggers the `onClear` event
- The clear button is not in the tab order (tabindex="-1") to streamline keyboard navigation
- Users can press Escape to clear the input

## Accessibility

This component is built with React Aria's SearchField and follows WCAG 2.1 AA guidelines.

### Keyboard Navigation

| Key | Action |
|-----|--------|
| `Tab` | Move focus to/from the search input |
| `Enter` | Submit the search query (triggers `onSubmit`) |
| `Escape` | Clear the input (triggers `onClear`) |

### Screen Reader Support

- Properly announces as a search field (`role="searchbox"`)
- Clear button announces its purpose
- State changes are announced (disabled, invalid, read-only)
- Value changes are conveyed to assistive technologies

### ARIA Attributes

- `role="searchbox"` - Identifies the input as a search field
- `aria-label` or `aria-labelledby` - Provides accessible name
- `aria-invalid` - Indicates invalid state
- `aria-disabled` - Announces disabled state
- `aria-readonly` - Announces read-only state
