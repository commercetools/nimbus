## Getting started

### Import

```tsx
import { Image, type ImageProps } from '@commercetools/nimbus';
```

### Basic usage

The Image component displays images with built-in support for accessibility, lazy loading, and responsive behavior.

```jsx-live-dev
const App = () => (
  <Image 
    src="https://images.unsplash.com/photo-1506905925346-21bda4d32df4?w=400&h=300&fit=crop"
    alt="Mountain landscape with blue sky"
    rounded="md"
    maxWidth="400px"
  />
)
```

### Responsive images

Use `srcSet` and `sizes` to serve different image resolutions based on the device's screen size and pixel density:

```jsx-live-dev
const App = () => (
  <Image 
    src="https://images.unsplash.com/photo-1426604966848-d7adac402bff?w=800&fit=crop"
    srcSet="
      https://images.unsplash.com/photo-1426604966848-d7adac402bff?w=400&fit=crop 400w,
      https://images.unsplash.com/photo-1426604966848-d7adac402bff?w=800&fit=crop 800w,
      https://images.unsplash.com/photo-1426604966848-d7adac402bff?w=1200&fit=crop 1200w
    "
    sizes="(max-width: 600px) 400px, (max-width: 1200px) 800px, 1200px"
    alt="Responsive nature image"
    rounded="md"
    maxWidth="full"
  />
)
```

## Component requirements

### Usage context

The Image component is a styled wrapper around the native HTML `<img>` element with enhanced features:

**When to use Image:**
- When you need Chakra UI styling capabilities (responsive props, theme tokens)
- For consistent styling across your application
- When using features like `fit` or `align` props for easier object-fit control
- For integration with Nimbus theme and design system

**When to use native `<img>`:**
- In simple cases where no styling or special features are needed
- When optimizing for minimal bundle size
- For static content that doesn't require theme integration

### Accessibility

The Image component provides accessibility through proper HTML image semantics and ARIA support.

#### Mandatory alt text

Every image must have an `alt` attribute. The alt text should:
- Describe the image content and context for screen reader users
- Be empty (`alt=""`) for purely decorative images
- Describe the action if the image is interactive (e.g., part of a link or button)
- Provide meaningful information, not just "image" or the file name

```jsx-live-dev
const App = () => (
  <Stack gap="400">
    <Box>
      <Text fontWeight="semibold" marginBottom="200">Informative image</Text>
      <Image
        src="https://images.unsplash.com/photo-1506905925346-21bda4d32df4?w=400&h=300&fit=crop"
        alt="Snow-capped mountain peaks at sunrise with pink and orange sky"
        rounded="md"
        maxWidth="400px"
      />
    </Box>
    <Box>
      <Text fontWeight="semibold" marginBottom="200">Decorative image (empty alt)</Text>
      <Image
        src="https://images.unsplash.com/photo-1470071459604-3b5ec3a7fe05?w=400&h=300&fit=crop"
        alt=""
        rounded="md"
        maxWidth="400px"
      />
    </Box>
  </Stack>
)
```

#### Accessibility standards

- **Role**: The Image component renders as an `<img>` element with implicit `img` role
- **Labeling**: Always provide descriptive `alt` text. For complex images (charts, diagrams), consider providing additional context through captions or aria-describedby

#### Keyboard Navigation

Images are not focusable by default. Keyboard navigation only applies when images are wrapped in interactive elements such as buttons or links. In those cases, ensure the wrapping element has proper focus indicators and keyboard event handlers.

#### Persistent ID tracking

If your use case requires tracking and analytics for this component, it is good practice to add a **persistent**, **unique** id to the component:

```tsx
const PERSISTENT_ID = "product-hero-image";

export const Example = () => (
  <Image 
    id={PERSISTENT_ID}
    src="https://images.unsplash.com/photo-1506905925346-21bda4d32df4?w=800&fit=crop"
    alt="Featured product image"
  />
);
```

## API reference

<PropsTable id="Image" />

The Image component accepts all standard HTML `<img>` attributes including:
- `src`: Image source URL
- `alt`: Alternative text for accessibility
- `srcSet`: Responsive image sources
- `sizes`: Size hints for responsive images
- `loading`: "lazy" or "eager" loading behavior
- `decoding`: "async", "sync", or "auto" for image decoding
- `crossOrigin`: CORS settings
- `referrerPolicy`: Referrer policy for image requests

Plus Chakra UI style props and the following enhanced props:
- `fit`: Object fit behavior (maps to `object-fit` CSS property)
- `align`: Object position (maps to `object-position` CSS property)

### Avatar with fallback

Use Image within custom avatar implementations or as profile pictures:

```jsx-live-dev
const App = () => {
  const [imageError, setImageError] = React.useState(false);
  
  const handleError = () => {
    setImageError(true);
  };
  
  return (
    <Stack direction="row" gap="400" alignItems="center">
      <Box position="relative">
        {!imageError ? (
          <Image 
            src="https://i.pravatar.cc/150?img=8"
            alt="John Doe profile picture"
            boxSize="80px"
            borderRadius="full"
            fit="cover"
            onError={handleError}
          />
        ) : (
          <Box 
            boxSize="80px"
            borderRadius="full"
            bg="bg.muted"
            display="flex"
            alignItems="center"
            justifyContent="center"
            fontWeight="semibold"
            fontSize="xl"
          >
            JD
          </Box>
        )}
      </Box>
      <Box>
        <Text fontWeight="semibold">John Doe</Text>
        <Text color="fg.muted" fontSize="sm">Product Designer</Text>
      </Box>
    </Stack>
  );
}
```
