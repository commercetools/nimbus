# List Component

List is a compound component that organizes related items or data into a vertical, scannable format. It provides semantic HTML list elements (`<ul>`, `<ol>`, `<li>`) with flexible styling options and custom indicators.

## Getting started

### Import

```tsx
import { List } from '@commercetools/nimbus';
```

### Basic usage

The simplest implementation uses semantic HTML with the `asChild` prop:

```jsx-live-dev
const App = () => (
  <List.Root asChild>
    <ul>
      <List.Item>First item</List.Item>
      <List.Item>Second item</List.Item>
      <List.Item>Third item</List.Item>
    </ul>
  </List.Root>
)
```

## Usage examples

### Visual variants

The List component supports two visual variants: `marker` (default) and `plain`. The `marker` variant displays native list markers (bullets or numbers), while `plain` removes markers and is ideal for custom indicators.

```jsx-live-dev
const App = () => (
  <Stack direction="row" gap="600" alignItems="flex-start">
    <Stack gap="200" flex="1">
      <Text fontWeight="600">Marker variant (default)</Text>
      <List.Root asChild variant="marker">
        <ul>
          <List.Item>Red pandas</List.Item>
          <List.Item>Pangolins</List.Item>
          <List.Item>Opossums</List.Item>
        </ul>
      </List.Root>
    </Stack>
    
    <Stack gap="200" flex="1">
      <Text fontWeight="600">Plain variant</Text>
      <List.Root variant="plain">
        <List.Item>
          <List.Indicator>
            <Icon color="positive.9"><Icons.Circle /></Icon>
          </List.Indicator>
          Reading a book
        </List.Item>
        <List.Item>
          <List.Indicator>
            <Icon color="positive.9"><Icons.Circle /></Icon>
          </List.Indicator>
          Going for a walk
        </List.Item>
      </List.Root>
    </Stack>
  </Stack>
)
```

### Custom indicators

Use `List.Indicator` with the `plain` variant to add custom markers, icons, or other visual elements before list items:

```jsx-live-dev
const App = () => (
  <List.Root variant="plain">
    <List.Item>
      <List.Indicator>
        <Icon color="positive.9"><Icons.Check /></Icon>
      </List.Indicator>
      Completed task
    </List.Item>
    <List.Item>
      <List.Indicator>
        <Icon color="warning.9"><Icons.Warning /></Icon>

      Task with warning

    <List.Item>
      <List.Indicator>
        <Icon color="critical.11"><Icons.Close /></Icon>
      </List.Indicator>

```

## Component requirements

### Usage context

List should be used when you need to:
- Display a series of related items in a vertical format
- Organize structured content with clear hierarchy
- Present navigation menus or option lists
- Show task lists, metadata, or brief information

For image-heavy content or complex data comparison, consider using other components like Card or DataTable instead.

### Accessibility

The List component uses semantic HTML (`<ul>`, `<ol>`, `<li>`) to convey structure to assistive technologies. When using the `asChild` prop, ensure you render as the appropriate semantic element:

```tsx
// ✅ Correct - uses semantic <ul> element
<List.Root asChild>
  <ul>
    <List.Item>Item 1</List.Item>
  </ul>
</List.Root>

// ✅ Correct - uses semantic <ol> element
<List.Root asChild>
  <ol>
    <List.Item>Item 1</List.Item>
  </ol>
</List.Root>
```

For custom indicators, ensure they have appropriate accessibility attributes:

```tsx
<List.Root variant="plain">
  <List.Item>
    <List.Indicator>
      <Icon aria-label="Completed" color="positive.9">
        <Icons.Check />
      </Icon>
    </List.Indicator>
    Task completed
  </List.Item>
</List.Root>
```

If your use case requires tracking and analytics for this component, it is good practice to add a **persistent**, **unique** id to the component:

```tsx
const PERSISTENT_ID = "example-list";

export const Example = () => (
  <List.Root asChild id={PERSISTENT_ID}>
    <ul>
      <List.Item>Item 1</List.Item>
    </ul>
  </List.Root>
);
```

#### Keyboard navigation

The List component itself doesn't handle keyboard interactions, as it's a presentational component. However, when list items are interactive (e.g., clickable), ensure they are keyboard accessible:

- `Tab` / `Shift+Tab`: Navigate between interactive list items
- `Enter` / `Space`: Activate the focused list item
- Ensure interactive items have proper focus indicators

## Common patterns

### Task list with status indicators

Create a task list with visual status indicators:

```jsx-live-dev
const App = () => {
  const tasks = [
    { id: 1, text: "Review design mockups", completed: true },
    { id: 2, text: "Write component documentation", completed: true },
    { id: 3, text: "Fix accessibility issues", completed: false },
    { id: 4, text: "Update test coverage", completed: false },
  ];

  return (
    <List.Root variant="plain">
      {tasks.map((task) => (
        <List.Item key={task.id}>
          <List.Indicator>
            <Icon 
              color={task.completed ? "positive.9" : "neutral.7"}
            >
              {task.completed ? <Icons.Check /> : <Icons.Circle />}
            </Icon>
          </List.Indicator>
          <Text 
            textDecoration={task.completed ? "line-through" : "none"}
            color={task.completed ? "neutral.9" : "fg.default"}
          >
            {task.text}
          </Text>
        </List.Item>
      ))}
    </List.Root>
  );
}
```

### Navigation menu

Use List to create navigation menus:

```jsx-live-dev
const App = () => {
  const menuItems = [
    "Home",
    "Products",
    "About",
    "Contact",
  ];

  return (
    <Box p="400" bg="neutral.2" borderRadius="200" width="fit-content">
      <List.Root asChild variant="plain">
        <ul>
          {menuItems.map((item, index) => (
            <List.Item key={index}>
              <Link href={`#${item.toLowerCase()}`}>
                {item}
              </Link>
            </List.Item>
          ))}
        </ul>
      </List.Root>
    </Box>
  );
}
```
