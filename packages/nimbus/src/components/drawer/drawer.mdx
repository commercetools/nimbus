# Drawer

A foundational drawer component for overlays that require user attention and interaction. Built with React Aria Components for accessibility and WCAG 2.1 AA compliance.

## Overview

Drawers are overlay windows that appear on top of the main content to display important information, gather user input, or require user decisions. They temporarily disable the main interface until the user completes an action or dismisses the drawer.

### Key features

- **Accessibility First**: Built with React Aria Components for WCAG 2.1 AA compliance
- **Flexible Positioning**: Support for left, right, top, and bottom placements  
- **Responsive Sizing**: Multiple size variants from xs to full-screen
- **Smooth Animations**: Customizable motion presets for entrance/exit
- **Focus Management**: Automatic focus trapping and restoration
- **Keyboard Navigation**: Full keyboard support including Escape to close
- **Scroll Handling**: Options for inside or outside scrolling behavior

## Variables

Get familiar with the features.

### Basic Usage

```jsx-live
const App = () => (
  <Drawer.Root>
    <Drawer.Trigger>
      Anything can be a trigger
    </Drawer.Trigger>
    <Drawer.Content>
      <Drawer.Header>
        <Drawer.Title>Drawer Title</Drawer.Title>
        <Drawer.CloseTrigger />
      </Drawer.Header>
      <Drawer.Body>
          This is a basic drawer with default settings. It includes a backdrop overlay
          (enabled by default), title, and close button for a complete experience.
      </Drawer.Body>
      <Drawer.Footer>
        <Button variant="outline" slot="close">Cancel</Button>
        <Button>Save</Button>
      </Drawer.Footer>
    </Drawer.Content>
  </Drawer.Root>
)
```

### Size

Control the drawer width by applying  `width` prop on the Content component,
it's a regular style-prop, so you can use all size-tokens but also custom values.

```jsx-live
const App = () => (
  <Drawer.Root >
    <Drawer.Trigger asChild>
      <Button>Open Large Drawer</Button>
    </Drawer.Trigger>
    <Drawer.Content width="3xl">
      <Drawer.Header>
        <Drawer.Title>Large Drawer</Drawer.Title>
        <Drawer.CloseTrigger />
      </Drawer.Header>
      <Drawer.Body>
        This drawer uses a custom width. You can change the width by
        adjusting the width prop on Drawer.Root to values like "xs", "sm",
        "md", "lg", "xl", or "full", but also raw css values.
      </Drawer.Body>
    </Drawer.Content>
  </Drawer.Root>
)
```

### Placement Options

Position the drawer at different locations using the `placement` prop. You can also control the backdrop overlay with the `showBackdrop` prop (enabled by default):

```jsx-live
const App = () => {
  const [showBackdrop, setShowBackdrop] = useState(true);

  const placements = [
    { value: "left", label: "Left", description: "Slides in from the left side, ideal for navigation panels or filters" },
    { value: "right", label: "Right (default)", description: "Slides in from the right side, commonly used for details panels and forms" },
    { value: "top", label: "Top", description: "Slides in from the top, useful for notifications or alerts" },
    { value: "bottom", label: "Bottom", description: "Slides in from the bottom, good for mobile-friendly sheets or action panels" }
  ];

  return (
    <Stack direction="column" gap="400">
      <Checkbox isSelected={showBackdrop} onChange={setShowBackdrop}>Show backdrop</Checkbox>

      <Stack direction="row" gap="400" wrap="wrap">
        {placements.map((placement) => (
          <Drawer.Root key={placement.value} placement={placement.value} showBackdrop={showBackdrop}>
            <Drawer.Trigger asChild>
              <Button>{placement.label}</Button>
            </Drawer.Trigger>
            <Drawer.Content>
              <Drawer.Header>
                <Drawer.Title>{placement.label} Placement</Drawer.Title>
                <Drawer.CloseTrigger />
              </Drawer.Header>
              <Drawer.Body>
                {placement.description}
              </Drawer.Body>
            </Drawer.Content>
          </Drawer.Root>
        ))}
      </Stack>
    </Stack>
  );
}
```


### Controlled State

Use external state to control the dialog programmatically:

```jsx-live
const App = () => {
  const [isOpen, setIsOpen] = React.useState(false);
  
  return (
    <Stack>
      <Button onClick={() => setIsOpen(true)}>
        Open Controlled Drawer
      </Button>
      
      <Text>Drawer is currently: {isOpen ? "open" : "closed"}</Text>
      
      <Drawer.Root isOpen={isOpen} onOpenChange={setIsOpen}>
        <Drawer.Content>
          <Drawer.Header>
            <Drawer.Title>Controlled Drawer</Drawer.Title>
            <Drawer.CloseTrigger />
          </Drawer.Header>
          <Drawer.Body>
              This drawer's state is controlled by the parent component.
              You can open/close it programmatically or through user interaction.
          </Drawer.Body>
          <Drawer.Footer>
            <Button variant="outline" onClick={() => setIsOpen(false)}>
              Cancel
            </Button>
            <Button onClick={() => setIsOpen(false)}>
              Save
            </Button>
          </Drawer.Footer>
        </Drawer.Content>
      </Drawer.Root>
    </Stack>
  );
}
```


## Accessibility

The Drawer component follows WCAG 2.1 AA guidelines and implements proper modal dialog patterns.

### Screen reader support


- **Role identification**: Drawer has `dialog` role with proper labeling
- **Focus management**: Focus moves to drawer on open, returns to trigger on close
- **Focus containment**: Tab navigation stays within drawer boundaries
- **Accessible names**: Title and description properly associate with drawer
- **State announcements**: Screen readers announce when drawer opens/closes

### WCAG Compliance

#### 1. Perceivable
- **1.3.1 Info and Relationships**: Proper semantic structure with header, body, footer
- **1.4.3 Contrast**: All text meets minimum contrast requirements  
- **1.4.11 Non-text Contrast**: Focus indicators and UI components meet contrast standards

#### 2. Operable  
- **2.1.1 Keyboard**: Full keyboard navigation support
- **2.1.2 No Keyboard Trap**: Focus contained but escapable via Escape key
- **2.4.3 Focus Order**: Logical focus progression through dialog elements
- **2.4.7 Focus Visible**: Clear focus indicators on all interactive elements

#### 3. Understandable
- **3.2.1 On Focus**: No unexpected context changes when focusing elements
- **3.2.2 On Input**: Predictable behavior for all user interactions
- **3.3.2 Labels or Instructions**: Clear labeling and instructions where needed

#### 4. Robust
- **4.1.2 Name, Role, Value**: Proper semantic markup and ARIA attributes
- **4.1.3 Status Messages**: Appropriate announcements for state changes

The Drawer component ensures that all users, regardless of their abilities or assistive technologies, can effectively interact with drawer content.
