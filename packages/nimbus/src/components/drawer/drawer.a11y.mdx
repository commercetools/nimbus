## Accessibility

Accessibility ensures that digital content and functionality are usable by
everyone, including people with disabilities, by addressing visual, auditory,
cognitive, and physical limitations.

```jsx-live
const App = () => {
  const [isOpen, setIsOpen] = useState(false);

  return (
    <>
      <Button onPress={() => setIsOpen(true)}>Open Drawer</Button>
      <Drawer.Root open={isOpen} onOpenChange={setIsOpen}>
        <Drawer.Content>
          <Drawer.Header>
            <Drawer.Title>Drawer Title</Drawer.Title>
          </Drawer.Header>
          <Drawer.Body>
            <Text>Drawer content goes here.</Text>
          </Drawer.Body>
          <Drawer.Footer>
            <Button variant="outline" onPress={() => setIsOpen(false)}>
              Cancel
            </Button>
            <Button variant="solid" onPress={() => setIsOpen(false)}>
              Confirm
            </Button>
          </Drawer.Footer>
        </Drawer.Content>
      </Drawer.Root>
    </>
  );
};
```

### Accessibility standards

- **Role identification:** Drawer has `dialog` role with proper labeling.
- **Focus management:** Focus moves to drawer on open and returns to trigger on
  close.
- **Focus containment:** Tab navigation stays within drawer boundaries.
- **Accessible names:** Title and description properly associate with drawer.
- **State announcements:** Screen readers announce when drawer opens/closes.
- **Info and relationships:** Proper semantic structure with header, body, and
  footer.
- **Contrast:** All text meets minimum contrast requirements.
- **Non-text contrast:** Focus indicators and UI components meet contrast
  standards.
- **Keyboard navigation:** Full keyboard navigation support.
- **No keyboard trap:** Focus contained but escapable via Escape key.
- **Focus order:** Logical focus progression through drawer elements.
- **Focus visible:** Clear focus indicators on all interactive elements.
- **On focus:** No unexpected context changes when focusing elements.
- **On input:** Predictable behavior for all user interactions.
- **Labels or instructions:** Clear labeling and instructions where needed.
- **Name, role, value:** Proper semantic markup and ARIA attributes.
- **Status messages:** Appropriate announcements for state changes.
