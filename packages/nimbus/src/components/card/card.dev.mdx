## Getting started

### Import

```tsx
import { Button, Card, Link, type CardProps } from '@commercetools/nimbus';
```

### Basic usage

The Card component uses a compound pattern to structure content. Wrap `Card.Header`, `Card.Body`, and `Card.Footer` within `Card.Root`.

```jsx live-dev
const App = () => (
  <Card.Root>
    <Card.Header>Card Title</Card.Header>
    <Card.Body>
      <Text>This is the main card content.</Text>
    </Card.Body>
  </Card.Root>
)
```

## Usage examples

### Size options

The `size` prop controls the internal padding and gap of the card. Available sizes are `sm`, `md`, and `lg`.

```jsx live-dev
const App = () => (
  <Stack direction="column" gap="400">
    <Card.Root size="sm">
      <Card.Header>Small</Card.Header>
      <Card.Body>Compact content spacing.</Card.Body>
    </Card.Root>

    <Card.Root size="md">
      <Card.Header>Medium</Card.Header>
      <Card.Body>Standard content spacing (default).</Card.Body>
    </Card.Root>

    <Card.Root size="lg">
      <Card.Header>Large</Card.Header>
      <Card.Body>Spacious content layout.</Card.Body>
    </Card.Root>
  </Stack>
)
```

### Visual variants

The `variant` prop controls the card's visual treatment. Eight variants
expose every combination of three independent axes — `outlined` (border),
`elevated` (shadow), and `muted` (background) — joined with hyphens.
Variant names omit any axis that is off.

| variant                       | border | shadow | background |
| ----------------------------- | :----: | :----: | :--------: |
| `plain`                       |        |        |  default   |
| `outlined` _(default)_        |   ✓    |        |  default   |
| `elevated`                    |        |   ✓    |  default   |
| `outlined-elevated`           |   ✓    |   ✓    |  default   |
| `muted`                       |        |        |   muted    |
| `outlined-muted`              |   ✓    |        |   muted    |
| `elevated-muted`              |        |   ✓    |   muted    |
| `outlined-elevated-muted`     |   ✓    |   ✓    |   muted    |

```jsx live-dev
const App = () => (
  <Stack direction="column" gap="400">
    <Card.Root variant="outlined">
      <Card.Header>Outlined (Default)</Card.Header>
      <Card.Body>Standard bordered look.</Card.Body>
    </Card.Root>

    <Card.Root variant="elevated">
      <Card.Header>Elevated</Card.Header>
      <Card.Body>Shadow depth without border.</Card.Body>
    </Card.Root>

    <Card.Root variant="outlined-elevated">
      <Card.Header>Outlined + elevated</Card.Header>
      <Card.Body>Both border and shadow.</Card.Body>
    </Card.Root>

    <Card.Root variant="muted">
      <Card.Header>Muted</Card.Header>
      <Card.Body>Muted background for differentiation.</Card.Body>
    </Card.Root>

    <Card.Root variant="plain">
      <Card.Header>Plain</Card.Header>
      <Card.Body>Minimal visual treatment.</Card.Body>
    </Card.Root>
  </Stack>
)
```

### Card with footer

Use `Card.Footer` for actions and metadata placed below the body content.

```jsx live-dev
const App = () => (
  <Card.Root variant="outlined" size="md">
    <Card.Header>Project Summary</Card.Header>
    <Card.Body>
      <Text>The project is on track for the next milestone.</Text>
    </Card.Body>
    <Card.Footer>
      <Button variant="solid" colorPalette="primary" size="sm">View Details</Button>
    </Card.Footer>
  </Card.Root>
)
```

### Accessible cards

Use `<Heading slot="title">` and `<Text slot="description">` to automatically wire `aria-labelledby` and `aria-describedby` on the card. The card itself stays a plain `<div>` — set `role` explicitly on the consumer side if a landmark or interactive role is needed.

```jsx live-dev
const App = () => (
  <Card.Root variant="outlined" size="md">
    <Card.Header>
      <Heading slot="title" as="h3">Sprint Summary</Heading>
    </Card.Header>
    <Card.Body>
      <Text slot="description">
        The team completed 85% of planned story points this sprint.
      </Text>
    </Card.Body>
    <Card.Footer>
      <Button variant="solid" colorPalette="primary" size="sm">View Sprint</Button>
    </Card.Footer>
  </Card.Root>
)
```

### Interactive cards

When making a card interactive, avoid placing `onClick` directly on the `Card.Root`. Instead, wrap the card in an anchor tag or use proper ARIA roles to ensure accessibility.

```jsx live-dev
const App = () => (
  <Stack direction="column" gap="400">
    {/* Option 1: Wrapped in a Nimbus Link (Recommended for navigation) */}
    <Link 
      href="#" 
      textDecoration="none" 
      color="inherit" 
      display="block"
      _hover={{ textDecoration: 'none' }}
      aria-label="Navigate to details"
    >
      <Card.Root variant="outlined">
        <Card.Header>Navigational Card</Card.Header>
        <Card.Body>
          This entire card acts as a link using the system Link component.
        </Card.Body>
      </Card.Root>
    </Link>

    {/* Option 2: Wrapped in a native <button> (Recommended for actions) */}
    <Button
      variant="ghost"
      onPress={() => alert("Card clicked")}
      aria-label="Trigger action"
      p="0"
    >
      <Card.Root variant="outlined">
        <Card.Header>Action Card</Card.Header>
        <Card.Body>
          Native button gives you Tab focus, Enter/Space activation, and a focus
          ring for free.
        </Card.Body>
      </Card.Root>
    </Button>
  </Stack>
)
```

## Component requirements

## Accessibility

The `Card` component supports automatic ARIA wiring via React Aria's slot system.

- **Automatic ARIA wiring**: Place `<Heading slot="title">` inside the card for automatic `aria-labelledby`. Place `<Text slot="description">` for `aria-describedby`. The card itself stays a plain `<div>` — no implicit `role` is applied.
- **Manual labeling**: Pass `aria-label` directly to `Card.Root` when slot-based labeling is not suitable.
- **Semantic role**: If you need a landmark role (e.g., `role="article"` or `role="region"`), set it explicitly on `Card.Root` or on a wrapping element.
- **Headings**: Ensure the content within `Card.Header` follows your page's heading hierarchy (e.g., use `Heading` component with appropriate `as` prop).
- **Interactive Cards**: If a card is interactive (clickable), do not attach `onClick` directly to the card div. Instead, wrap the card in a native link or button, which provides focus, keyboard activation, and the correct accessible name automatically.

If your use case requires tracking and analytics for this component, it is good practice to add a **persistent**, **unique** id to the component:

```tsx
const PERSISTENT_ID = "project-summary-card";

export const Example = () => (
  <Card.Root id={PERSISTENT_ID}>
    <Card.Header>Summary</Card.Header>
    <Card.Body>...</Card.Body>
  </Card.Root>
);
```

#### Keyboard navigation

Standard Cards are not focusable. Interactive content inside the card (buttons, links) follows the default focus order.
