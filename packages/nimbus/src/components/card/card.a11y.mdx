## Accessibility

Accessibility ensures that digital content and functionality are usable by
everyone, including people with disabilities, by addressing visual, auditory,
cognitive, and physical limitations.

### Automatic ARIA wiring

Card supports automatic ARIA wiring via React Aria's slot system. When you place
a `<Heading slot="title">` or `<Text slot="description">` inside a Card, the
root element automatically gains `role="article"` along with `aria-labelledby`
and/or `aria-describedby` pointing to those elements.

```jsx live
const App = () => (
  <Card.Root variant="outlined" size="md">
    <Card.Header>
      <Heading slot="title" as="h3">Sprint Summary</Heading>
    </Card.Header>
    <Card.Body>
      <Text slot="description">
        The team completed 85% of planned story points this sprint.
      </Text>
    </Card.Body>
    <Card.Footer>
      <Button variant="solid" colorPalette="primary" size="sm">View Sprint</Button>
    </Card.Footer>
  </Card.Root>
)
```

### Slot behavior

| Slots present | `role` | `aria-labelledby` | `aria-describedby` |
|---|---|---|---|
| `Heading slot="title"` + `Text slot="description"` | `article` | points to Heading | points to Text |
| `Heading slot="title"` only | `article` | points to Heading | — |
| `Text slot="description"` only | `article` | — | points to Text |
| No slots | — | — | — |

Cards without slot-prop children remain plain `<div>` elements with no ARIA role
or attributes. A `<Heading>` or `<Text>` without a `slot` prop is not affected.

### Manual labeling

If slot-based labeling is not suitable, pass `aria-label` directly to
`Card.Root`:

```jsx live
const App = () => (
  <Card.Root variant="outlined" size="md" aria-label="Product summary card">
    <Card.Body>
      <Text>Content with manual aria-label.</Text>
    </Card.Body>
  </Card.Root>
)
```

### Accessibility standards

- **Non-text content:** Provide descriptive alternative text (`alt` attribute)
  for images within cards.
- **Info and relationships:** Use semantic HTML to structure the card content
  logically (e.g., headings, lists).
- **Color contrast:** Ensure sufficient color contrast between text and
  background within the card.
- **Keyboard accessibility:** Ensure the entire card or its primary interactive
  elements are focusable and operable using only a keyboard.
- **Focus visible:** Provide a clear visual focus indicator when the card or its
  interactive elements are focused.
- **Target size:** If the card is interactive, ensure touch target sizes are
  adequate.
- **Meaningful sequence:** Ensure the content within the card is presented in a
  logical order.
- **Semantic HTML:** Use semantic HTML to ensure compatibility with assistive
  technologies.
- **Avoid complex interactive cards:** Avoid making the entire card a single
  interactive element if it contains multiple distinct actions, to ensure proper
  interaction for assistive technology users.
