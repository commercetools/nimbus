# Pages

Page layout patterns provide pre-built, opinionated layouts for common full-page structures. Use them to avoid hand-composing flex and stack primitives for pages that follow a well-established shape.

## When to use page patterns

Page patterns are ideal for:
- Public-facing pages (login, registration, password reset) with a centered, single-column layout
- Reducing layout boilerplate for pages that appear across multiple apps
- Ensuring consistent landmark structure and accessibility across page types

For pages with bespoke chrome, sidebars, or multi-column structures, compose `Flex` and `Stack` directly.
