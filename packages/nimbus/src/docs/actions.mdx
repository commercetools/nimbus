# Actions

Action patterns provide pre-composed sets of controls for common form and page interactions. Use them to avoid hand-composing button groups and wiring up save, cancel, and delete behavior from scratch.

## When to use action patterns

Action patterns are ideal for:
- Standard form footers that need save, cancel, and optional delete controls
- Reducing boilerplate for repeated action bar shapes across an app
- Ensuring consistent button ordering and keyboard behavior

For action bars with non-standard button sets or custom layouts, compose `Button` and `Group` directly.
