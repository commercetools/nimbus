# Installation

How to add Nimbus to your react frontend.

## 1. Install the nimbus packages

You can install Nimbus using your preferred package manager.

### Core Installation

Install the core Nimbus packages along with required peer dependencies:

#### pnpm

```bash
pnpm add @commercetools/nimbus @commercetools/nimbus-icons @chakra-ui/react
```

#### npm

```bash
npm install @commercetools/nimbus @commercetools/nimbus-icons @chakra-ui/react
```

#### yarn

```bash
yarn add @commercetools/nimbus @commercetools/nimbus-icons @chakra-ui/react
```

### Rich Text Editor (Optional)

If you're using the Rich Text Input component, also install the Slate dependencies:

#### pnpm

```bash
pnpm add slate slate-history slate-hyperscript slate-react
```

#### npm

```bash
npm install slate slate-history slate-hyperscript slate-react
```

#### yarn

```bash
yarn add slate slate-history slate-hyperscript slate-react
```

> [!TIP]
> Your package manager will automatically resolve compatible versions based on Nimbus's peer dependency requirements.

> [!NOTE]  
> Nimbus requires React 19.x. If you're on an earlier version, upgrade with `pnpm add react@19 react-dom@19`

### TypeScript Support

TypeScript types for component variants, sizes, and design tokens are generated
via a postinstall hook. Modern package managers (pnpm v8+, yarn v2+) block
postinstall scripts by default. To enable automatic type generation:

#### pnpm

Add to your `package.json`:

```json
{
  "pnpm": {
    "onlyBuiltDependencies": ["@commercetools/nimbus"]
  }
}
```

Or run `pnpm approve-builds` after installation.

#### Manual generation

If the postinstall doesn't run, generate types manually:

```bash
npx @chakra-ui/cli typegen node_modules/@commercetools/nimbus/dist/index.es.js
```

### Design Token Autocomplete (Optional)

Install the design token TypeScript plugin to see resolved CSS values next to token names in autocomplete. For example, when you type `gap="`, the dropdown shows `600 = 24px` instead of just `600`.

#### pnpm

```bash
pnpm add -D @commercetools/nimbus-design-token-ts-plugin
```

#### npm

```bash
npm install -D @commercetools/nimbus-design-token-ts-plugin
```

#### yarn

```bash
yarn add -D @commercetools/nimbus-design-token-ts-plugin
```

Then add the plugin to your `tsconfig.json` (or `tsconfig.app.json` if using
project references):

```json
{
  "compilerOptions": {
    "plugins": [
      { "name": "@commercetools/nimbus-design-token-ts-plugin" }
    ]
  }
}
```

In VS Code, you must use the workspace TypeScript version for the plugin to
activate:

1. Open the command palette (`Cmd+Shift+P` / `Ctrl+Shift+P`)
2. Run **"TypeScript: Select TypeScript Version"**
3. Select **"Use Workspace Version"**
4. Restart the TS server: **"TypeScript: Restart TS Server"**

The plugin supports spacing, colors, font sizes, border radius, font weights,
shadows, opacity, z-index, and more.

## 2. Set up NimbusProvider

Wrap your application with the `NimbusProvider` at the root level. This provider
is essential for Nimbus components to function correctly, as it supplies the
necessary theme and context.

```tsx
// App.tsx or main.tsx
import React from "react";
import ReactDOM from "react-dom/client";
import { NimbusProvider } from "@commercetools/nimbus";
import App from "./App";

ReactDOM.createRoot(document.getElementById("root")!).render(
  <React.StrictMode>
    <NimbusProvider>
      <App />
    </NimbusProvider>
  </React.StrictMode>
);
```

> [!NOTE]\
> Checkout the [NimbusProvider documentation](/components/utilities/nimbusprovider)
> for more details on how to set up **i18n** and **routing**.

## 3. Use Nimbus Components

Once the provider is set up, you can start using Nimbus components in your
application.

```jsx
// App.tsx
import { Flex, Box, Button } from "@commercetools/nimbus";
import { ArrowForward } from "@commercetools/nimbus-icons";

const App = () => {
  return (
    <Flex width="100vw" height="100vh">
      <Box margin="auto">
        <Button
          variant="solid"
          colorPalette="primary"
          onPress={() => alert("It just works!")}
        >
          <ArrowForward /> Click me
        </Button>
      </Box>
    </Flex>
  );
};

export default App;
```

That's it! You've successfully installed and set up Nimbus in your project. You
can now explore and use the wide range of components offered by the library.

## 4. Set up Testing (Optional)

If you're using Jest or Vitest to test your application, you'll need to add Nimbus polyfills to your test configuration:

### Jest

```javascript
// jest.config.js
module.exports = {
  setupFiles: [
    '@commercetools/nimbus/setup-jsdom-polyfills'
  ]
};
```

### Vitest

```typescript
// vitest.config.ts
export default defineConfig({
  test: {
    setupFiles: ['@commercetools/nimbus/setup-jsdom-polyfills']
  }
});
```

For complete testing setup instructions, see the **[Testing Setup](/home/getting-started/testing-setup)** guide.

## Troubleshooting

### TypeScript types not working

If props like `variant` or `size` show as `any`:

1. Run the manual type generation command (see TypeScript Support section above)
2. Restart your TypeScript server (VS Code: `Cmd+Shift+P` → "TypeScript: Restart TS Server")
