# useScrollableRegion

Make scrollable containers accessible to keyboard and screen reader users.
Detects content overflow via `ResizeObserver` and dynamically manages `tabIndex`,
ARIA roles, accessible name, and a keyboard-only focus ring.

> **TIP:** If you just need a standalone scrollable container and don't need to
> attach the behavior to an existing element, use the
> [`ScrollableRegion`](/components/scrollable-region) component instead. It
> wraps this hook and renders the appropriate element automatically.

## Basic usage

Import the hook from the package and attach the returned `ref` and spread
`containerProps` onto your scrollable element.

The hook is designed for cases where you need to apply scrollable-region
accessibility to an existing element — for example, a compound component slot
where you cannot introduce a wrapper component.

## Reference

```ts
const { ref, isOverflowing, containerProps } = useScrollableRegion(options);
```

### Options

| Option | Type | Default | Description |
|---|---|---|---|
| `role` | `"region"` \| `"group"` | `"group"` | Landmark role applied when overflowing. |
| `aria-label` | `string` | &mdash; | Accessible name for the container. Required when `role="region"` (either this or `aria-labelledby` must be provided). |
| `aria-labelledby` | `string` | &mdash; | ID of the element labeling this region. Required when `role="region"` (either this or `aria-label` must be provided). |
| `debounceMs` | `number` | `100` | Debounce interval for overflow evaluation. |
| `overflow` | `"auto"` \| `"scroll"` | `"auto"` | CSS overflow value. |

### Return value

- `ref` &mdash; ref callback to attach to the scrollable DOM element.
- `isOverflowing` &mdash; boolean indicating whether content overflows.
- `containerProps` &mdash; spread-ready props object including `tabIndex`, `role`,
  ARIA attributes, overflow style, and focus ring styles. Only includes
  semantic attributes when the element is overflowing.
