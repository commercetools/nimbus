# Accessibility Documentation (`{component}.a11y.mdx`)

> Reference file for the **writing-consumer-documentation** skill, loaded for
> the Accessibility tab. The skill owns mode detection and argument parsing;
> this file is the per-type detail.

WCAG 2.1 AA conformance and assistive-technology behavior, written for **all
roles** — a consumer needs to know what to preserve, a designer needs to know
what to validate. Implementation-only ARIA wiring a consumer must keep can also
appear in the `.dev.mdx` Implementation tab; this tab is the user-facing a11y
contract.

Canonical structure rules live in
[documentation.md](../../../../docs/file-type-guidelines/documentation.md); this
file is the per-type quick reference.

## Frontmatter (minimal — view file)

```yaml
---
tab-title: Accessibility
tab-order: 4
---
```

Inherits id/title/menu/tags from the main `{component}.mdx`. No top-level
`# Title` heading in the body.

## Content structure

1. **Accessibility standards** — the WCAG criteria met: contrast ratios (4.5:1
   text, 3:1 graphics/UI), semantic structure, focus management, target size, no
   unexpected context changes.
2. **Keyboard navigation** (interactive components) — a key/action table:

   ```markdown
   | Key             | Action                |
   | --------------- | --------------------- |
   | `Tab`           | Move focus to/from it |
   | `Enter`/`Space` | Activate              |
   | `Escape`        | Close/Cancel          |
   | `Arrow` keys    | Navigate options      |
   ```

3. **Screen reader support** — role, state, and live-region announcements; label
   associations.
4. **ARIA attributes** — roles/attributes used and when each is needed (for
   custom implementations).

## Authoring rules

- Use `jsx live` blocks (**not** `jsx live-dev`) for any examples.
- State conformance precisely; don't claim criteria the component doesn't meet.
- Prose follows the [writing style](../../../../docs/writing-style.md)
  **designer overlay** — plain language, but `must`/`should`/`may` carry the
  obligation.

## Validation checklist

- [ ] Frontmatter is exactly `tab-title: Accessibility` + `tab-order: 4`.
- [ ] No top-level `# Title` heading in the body.
- [ ] WCAG conformance statement present.
- [ ] Keyboard navigation table for interactive components.
- [ ] Screen reader / ARIA notes present.
- [ ] Any examples use `jsx live`, define `App`, and need no imports.
