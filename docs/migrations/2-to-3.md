# Migrating from Nimbus 2.x to 3.x

The only breaking change in 3.0.0 is a rework of the **`Card`** component. Its
body slot was renamed, and its four ad-hoc styling props were replaced by the
standard Nimbus `variant` and `size` props so Card matches every other
component. Everything else in 2.x is forward-compatible — if you don't use
`Card`, upgrading to 3.x requires no changes.

[← Back to API Evolution](../api-evolution.md) ·
[All migration guides](./README.md)

## At a glance

| 2.x                                               | 3.x                                             |
| ------------------------------------------------- | ----------------------------------------------- |
| `Card.Content`                                    | `Card.Body`                                     |
| `cardPadding="sm \| md \| lg"`                    | `size="sm \| md \| lg"`                         |
| `borderStyle`, `elevation`, `backgroundStyle`     | folded into `variant`                           |
| `Card.Root` is `inline-flex` (shrinks to content) | `Card.Root` is block-level `flex` (fills width) |

A `Card.Root` left at its 2.x defaults (`borderStyle="outlined"`,
`elevation="none"`, `backgroundStyle="default"`, `cardPadding="md"`) maps
exactly to the 3.x defaults (`variant="outlined"`, `size="md"`). If you never
set those props, the only required edit is `Card.Content` → `Card.Body`.

## 1. `Card.Content` → `Card.Body`

```tsx
// Before (2.x)
<Card.Root>
  <Card.Header>Title</Card.Header>
  <Card.Content>Body text</Card.Content>
</Card.Root>

// After (3.x)
<Card.Root>
  <Card.Header>Title</Card.Header>
  <Card.Body>Body text</Card.Body>
</Card.Root>
```

## 2. `cardPadding` → `size`

A straight rename — the accepted values (`sm` / `md` / `lg`) and the default
(`md`) are unchanged.

```tsx
// Before (2.x)
<Card.Root cardPadding="lg">…</Card.Root>

// After (3.x)
<Card.Root size="lg">…</Card.Root>
```

## 3. `borderStyle` + `elevation` + `backgroundStyle` → `variant`

The three independent boolean-ish axes are now a single `variant` enum. Each
axis that was "on" contributes its name, in the fixed order
`outlined`-`elevated`-`muted`, joined with `-`. If none are on, the variant is
`plain`.

| 2.x prop          | "On" value | Contributes to `variant` |
| ----------------- | ---------- | ------------------------ |
| `borderStyle`     | `outlined` | `outlined`               |
| `elevation`       | `elevated` | `elevated`               |
| `backgroundStyle` | `muted`    | `muted`                  |

Build the variant by listing the on-axes in order:

| 2.x props                                                         | 3.x `variant`              |
| ----------------------------------------------------------------- | -------------------------- |
| _(all defaults / none set)_                                       | `outlined` _(the default)_ |
| `borderStyle="none"`                                              | `plain`                    |
| `elevation="elevated"`                                            | `outlined-elevated`        |
| `borderStyle="none" elevation="elevated"`                         | `elevated`                 |
| `backgroundStyle="muted"`                                         | `outlined-muted`           |
| `borderStyle="none" backgroundStyle="muted"`                      | `muted`                    |
| `elevation="elevated" backgroundStyle="muted"`                    | `outlined-elevated-muted`  |
| `borderStyle="none" elevation="elevated" backgroundStyle="muted"` | `elevated-muted`           |

Full variant set:
`"plain" | "outlined" | "elevated" | "outlined-elevated" | "muted" | "outlined-muted" | "elevated-muted" | "outlined-elevated-muted"`.

```tsx
// Before (2.x)
<Card.Root borderStyle="outlined" elevation="elevated" backgroundStyle="muted">…</Card.Root>

// After (3.x)
<Card.Root variant="outlined-elevated-muted">…</Card.Root>
```

## 4. `Card.Root` now fills its container

`Card.Root` changed from `inline-flex` to a block-level `flex` container, so it
now fills the available width by default instead of shrinking to its content. If
you relied on the old shrink-to-fit behavior, constrain it explicitly:

```tsx
// Restore 2.x sizing behavior if you depended on it
<Card.Root width="fit-content">…</Card.Root>
```

## New in 3.x (optional, non-breaking)

- **`Card.Footer`** — a slot for actions/metadata below the body.
- **Free-form `Card.Root`** — used without
  `Card.Header`/`Card.Body`/`Card.Footer`, padding now lives on the Root, so
  arbitrary children no longer render flush against the border.
- **Slot-based ARIA labelling** — a `<Heading slot="title">` or
  `<Text slot="description">` inside a Card auto-wires `aria-labelledby` /
  `aria-describedby`. The Card stays a plain `<div>`; set `role` explicitly if
  you need a landmark.

## Checklist

- [ ] Replace every `<Card.Content>` with `<Card.Body>`.
- [ ] Rename `cardPadding` → `size` on every `Card.Root`.
- [ ] Replace `borderStyle` / `elevation` / `backgroundStyle` with the single
      `variant` value from the table above; delete the old props.
- [ ] Audit layouts that assumed a `Card` shrinks to its content — add
      `width="fit-content"` (or an explicit width) where needed.
